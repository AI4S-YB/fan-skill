# 基因组注释分析引导

## 这个分析解决什么问题

用户有基因组组装，需要预测基因位置和功能。典型表述：
- "我刚组装完基因组，怎么注释基因"
- "我的物种没有参考注释"
- "怎么评估注释质量"

## 你需要问什么

1. **基因组组装质量**: N50？染色体水平/scaffold/contig？
2. **基因组大小**: 影响运行时间和计算资源需求
3. **RNA-seq 证据**: 有吗？哪些组织？深度够吗？
4. **近缘物种**: 有高质量注释的近缘种吗？
5. **多倍体**: 是否分亚基因组注释？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **重复序列**: 植物基因组重复序列高达 30-85%。RepeatMasker/RepeatModeler 是必需步骤，屏蔽不足 → TE 序列被误预测为基因
- **多组织 RNA-seq**: 植物基因表达具有很强的组织特异性。单组织 RNA-seq 会遗漏 20-40% 的基因。建议至少有叶、根、花三个组织
- **BUSCO 评估**: 植物使用 embryophyta_odb10 或更特异的 lineage（eudicots_odb10、liliopsida_odb10）。多倍体中 BUSCO Duplicated 偏高是正常的
- **非模式植物**: 大多数植物没有现成的 repeat library 或 species-specific AUGUSTUS model。需要额外的从头训练步骤

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"基因组组装(染色体水平)、多组织RNA-seq证据、近缘物种注释可用"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有基因组组装，缺乏RNA-seq证据；或组装质量较低(scaffold级别)"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"基因组组装尚未完成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
