# 多组学整合分析引导

## 这个分析解决什么问题

用户想整合多种组学数据发现跨层次的生物学规律。典型表述：
- "怎么把转录组和代谢组关联起来"
- "我想看 mRNA-蛋白的相关性"
- "多组学数据中哪些因子是共享的"

## 你需要问什么

1. **有哪些组学**: 转录组/代谢组/蛋白组/表观组？几种？
2. **样本量**: ≥15 用 MOFA2；≥20 用 DIABLO；n<15 仅两组学 PLS
3. **有分类标签吗**: 处理/对照？品种分类？无标签→无监督(MOFA2)
4. **数据匹配**: 所有样本都有全部组学吗？有缺失吗？
5. **多倍体**: 转录组中同源基因如何处理？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **样本匹配**: 多组学整合要求同一批样本（或至少同一基因型的同批种植材料）的不同组学数据。不同批次种植的植物可能因环境差异产生系统性偏差
- **组学层选择**: 植物中常见组合：基因组+转录组+代谢组（基因→代谢通路），或转录组+蛋白组（mRNA-蛋白相关性弱，r=0.4-0.6）
- **多倍体**: 多倍体物种的多组学整合需要能在亚基因组水平解析各层数据（如区分 A、B、D 亚基因组的转录本和蛋白）
- **环境响应**: 植物代谢组和转录组对环境高度敏感。建议在受控环境（生长室/温室）进行多组学实验以减少环境噪声

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
