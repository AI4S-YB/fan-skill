# 植物多组学整合：分析笔记

这份笔记帮助你像一个有经验的植物系统生物学家一样思考和做决策。

## 在开始之前：理解你的组学数据

### 你有哪几种组学？

植物多组学最常见的组合：

- **转录组 + 代谢组**：基因表达→代谢物积累，最经典。适合研究次生代谢途径（类黄酮、萜类、生物碱）。
- **转录组 + 蛋白质组**：mRNA-蛋白质相关性在植物中通常 0.4-0.6，低于微生物。翻译后调控和蛋白质降解是重要因素。
- **转录组 + 表观组 (甲基化/DNase/ATAC)**：表观调控→基因表达。适合发育过程和环境响应。
- **转录组 + 代谢组 + 蛋白质组**：三组学。需要足够样本量（>= 20），用 MOFA2。
- **转录组 + 代谢组 + 微生物组**：宿主-微生物互作。植物根际研究中常见，需注意微生物组与植物组学的数据尺度差异。

### 样本量足够吗？

多组学整合对样本量的要求比单组学高：

- **n < 15**：仅适合两组学 PLS。多组学方法结果不稳定。
- **n = 15-30**：MOFA2 可用，但需限制因子数（建议 <= 5）。
- **n >= 30**：MOFA2 和 DIABLO 均适用。因子估计稳定。
- **n >= 50**：充分。可以考虑更复杂的模型结构和交叉验证。

### 有分类标签（表型/处理组）吗？

- **有明确分组**（如：抗病 vs 感病、干旱 vs 对照、不同品种）：
  → 用 DIABLO。它能找到最大化组间差异的多组学特征。
  → 但注意：DIABLO 需要每个组有足够样本（至少 5-8 个）。

- **无明确分组**（如：自然群体的多组学图谱）：
  → 用 MOFA2。无监督学习，从数据本身发现结构。
  → 或两组学 PLS，如果只有两种组学。

- **有连续性状**（如：产量、株高、含油量）：
  → 可以将其编码为 DIABLO 的连续型 outcome。
  → 或先用 MOFA2 找因子，再与性状做关联分析。

## 选择你的整合方法

### MOFA2 — 无监督多组学因子分析

从多种组学数据中学习隐藏的"因子"，每个因子捕获跨组学的协变信号。

**适合**：
- 3 种及以上组学
- 探索性分析：不知道期待什么样的结果
- 想发现"数据自己说的故事"
- 自然群体、多样性面板、多品种比较

**不适合**：
- 样本量 < 15（因子不稳定）
- 你已有明确的分类标签且分类是你的首要目标（用 DIABLO）

**MOFA2 输出的解读**：
- 每个因子解释了各 omics 中多少方差（variance decomposition plot）
- 每个因子在哪些特征上权重最大（top weights）→ 理解因子的生物学意义
- 因子得分在每个样本上的值 → 看样本分组/排序

### DIABLO — 有监督多组学判别分析

同时分析多种组学，找到最能区分预设类别的特征组合。

**适合**：
- 有明确的分类标签（处理组、表型级别、品种类别等）
- 目标是"哪些组学特征共同决定类别差异"
- 你需要分类器的生物学可解释性

**不适合**：
- 无分类标签
- 分类标签的生物学基础不明确（DIABLO 总会找到差异，但可能没有生物学意义）
- 样本量分配极不均衡

**DIABLO 的关键决策**：
- **design matrix**：组学间的关系权重。通常设 0.1（弱关联），不要设太高否则过拟合。
- **ncomp**：保留的成分数。通过 perf() 函数的 error rate 选择。
- **keepX**：每个成分保留的特征数。用 tune() 函数调优。

### mixOmics PLS/sPLS — 两组学关联

两组学之间的关联分析，找到协变结构。

**适合**：
- 只有两种组学（如：转录组 + 代谢组）
- 想看"基因表达与代谢物积累的关系"
- 想识别关键调控基因-代谢物对

**不适合**：
- 三种及以上组学（用 MOFA2 或 DIABLO）
- 需要多重比较校正（sPLS 本身已做变量选择）

## 数据预处理的植物学考量

### 转录组

- 过滤低表达基因：至少在 20% 样本中 count >= 10
- 标准化：VST (DESeq2) 或 TMM (edgeR)，然后 log2 转换
- 批量效应：如果样本来自不同批次，用 ComBat 或 RUVseq 校正
- **植物特别注意**：叶绿体和线粒体基因可以保留，但在解释富集结果时加以说明

### 代谢组

- 通常已经过峰对齐和归一化（如 XCMS 或 MS-DIAL 处理）
- 进一步处理：log2 转换 + Pareto scaling（均值中心化 + 除以标准差平方根）
- 缺失值：如果 > 50% 样本缺失 → 删除该代谢物；如果 < 20% → 用最小值的 1/2 填充
- **植物特别注意**：次生代谢物在植物中高度多样化，许多代谢物无法鉴定。保留"未知"代谢物——它们可能携带有意义的信息

### 蛋白质组

- 常用 label-free quantification (LFQ) 强度值
- log2 转换（蛋白质组数据通常已经是 log2 的）
- 缺失值：如果 < 30% → 用最小值插补；如果 > 30% → 删除
- **植物特别注意**：Rubisco 等高丰度蛋白可能主导信号。考虑移除 Rubisco 相关蛋白后再标准化，或使用中位数中心化

### 表观组 (甲基化/DNase/ATAC-seq)

- 甲基化：β 值 (0-1)，用 M 值 (logit 转换) 进行统计分析
- ATAC-seq/DNase-seq：count-based，用 DESeq2 标准化
- **植物特别注意**：植物 CG/CHG/CHH 三种甲基化上下文需分别处理

### 微生物组

- 通常用相对丰度 (relative abundance) 数据
- CLR (centered log-ratio) 转换处理组分性
- 过滤：至少在 10% 样本中相对丰度 > 0.1%
- **植物特别注意**：根际 vs 根内微生物群差异极大，不要混合分析

## 常见陷阱

### 批次效应掩盖生物学信号

多组学数据常来自不同实验、不同时间、不同人员。如果 MOFA2 的第一个因子就完美区分了批次——你的数据存在严重的批次效应。

**怎么办**：
1. 在预处理阶段用 ComBat 等工具校正
2. 如果无法校正，在 MOFA2 模型中包含批次协变量
3. 在报告中诚实地讨论批次效应对结果的影响

### 组学间的样本名称不一致

转录组的样本叫 "Leaf_Control_1"，代谢组的同一份样本叫 "LC1"——这会把所有样本都当唯一。**在进行任何分析之前，先确认样本名称统一。**

### 多组学整合不等于提高功效

错误认知："我的转录组 DEG 不显著，加上代谢组数据，用 MOFA2 就能找到信号了。"

组学整合不能弥补单个组学的统计功效不足。它能做的是：发现**跨组学协调变化**的信号，这些信号在单组学中可能被噪声掩盖但**并非不可能被发现**。

### 植物多倍体的同源基因问题

小麦 (6x)、棉花 (4x)、油菜 (4x) 等多倍体作物：
- 同源基因 (homeologs) 在转录组中可能各有不同表达模式
- MOFA2 无法自动区分同源基因——如果它们被当作独立特征，可能拉低单个因子的方差解释
- 建议：先按亚基因组分组做 MOFA2，再比较因子结构的异同

### 相关性不等于因果性

MOFA2 发现的共变因子不代表因果调控关系。比如一个因子同时驱动了某个转录因子和某个代谢物——这不意味着该转录因子直接调控该代谢物的合成。需要额外的实验验证（如酵母单杂/ChIP-qPCR 等）。

### 富集分析需注意背景基因集

对 MOFA2 因子的 top 基因做 GO/KEGG 富集分析时，背景基因集应该是"该组学中存在的所有基因"，而不是"全基因组的基因"。植物参考注释的质量因物种而异——拟南芥和水稻的注释远优于大多数作物。
