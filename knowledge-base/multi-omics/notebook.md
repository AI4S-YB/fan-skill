# 植物多组学整合：分析笔记

这份笔记帮助你像一个有经验的植物系统生物学家一样思考和做决策。

## 在开始之前：理解你的组学数据

### 你有哪几种组学？

植物多组学最常见的组合：

- **转录组 + 代谢组**：基因表达→代谢物积累，最经典。适合研究次生代谢途径（类黄酮、萜类、生物碱）。
- **转录组 + 蛋白质组**：mRNA-蛋白质相关性在植物中通常 0.4-0.6，低于微生物。翻译后调控和蛋白质降解是重要因素。
- **转录组 + 表观组 (甲基化/DNase/ATAC)**：表观调控→基因表达。适合发育过程和环境响应。
- **转录组 + 代谢组 + 蛋白质组**：三组学。需要足够样本量（>= 20），用 MOFA2。
- **转录组 + 代谢组 + 微生物组**：宿主-微生物互作。植物根际研究中常见，需注意微生物组与植物组学的数据尺度差异。

### MOFA vs DIABLO vs mixOmics：完整决策指南

三个工具在不同场景下各有所长。选择取决于你的核心问题类型和数据特征：

| 维度 | MOFA2 | DIABLO | mixOmics (s)PLS |
|------|-------|--------|-----------------|
| 学习类型 | 无监督 | 有监督（分类/回归） | 无监督或弱监督 |
| 组学数量 | 2-6 种 | 2-5 种 | 2 种为主，可扩展到 3+ |
| 样本量需求 | >= 15（推荐 >= 30） | >= 20（每组 >= 5） | >= 10（两组学） |
| 缺失数据容忍度 | 高（每个组学允许不同缺失模式） | 中（每组学样本应基本一致） | 低（建议完整匹配） |
| 输出类型 | 低维因子 + 因子-特征权重 + 方差分解 | 判别模型 + 特征选择 + 分类器 | 投影/loading + 相关图 |
| 因子数选择 | 自动或手动设定 | 通过交叉验证选择 | 通过交叉验证或手动设定 |
| 植物领域使用 | 上升趋势（泛组学因子分析） | 稳定（分类/诊断研究） | 很广泛（代谢-转录关联） |
| 多倍体适用性 | 好（因子模型不要求亚基因组信息） | 一般（特征选择时可能出现亚基因组偏差） | 好（变量选择可以控制） |

**MOFA2 的最佳使用时机**：
1. 你不知道期望什么样的结果（探索性分析）
2. 你有 3 种及以上组学，且想同时分析所有组学
3. 你有缺失数据（不是所有样本都有所有组学的数据）
4. 你想发现"跨组学协调变化"的隐藏因子
5. 你想了解每个组学对方差的总贡献

**DIABLO 的最佳使用时机**：
1. 你有明确的分类标签（处理 vs 对照、高产品种 vs 低产品种）
2. 目标是跨组学的分类/诊断模型
3. 你需要特征选择（哪些基因/代谢物最具判别力）
4. 你想得到高可视化直观结果（用 plotIndiv、plotVar、circosPlot）

**mixOmics (s)PLS 的最佳使用时机**：
1. 你只有两种组学（最常见：转录组 + 代谢组）
2. 你想找两组学之间的关联结构（哪些基因与哪些代谢物共变）
3. 你需要网络式输出（关联网络），而非因子/分类器

**工具选择的实战决策**：
- 如果你的研究问题是"哪些生物学过程跨组学协调变化？"→ MOFA2
- 如果你的研究问题是"哪种组学特征最能区分我的处理组？"→ DIABLO
- 如果你的研究问题是"这 500 个基因和这 300 个代谢物有什么关联？"→ sPLS
- 如果你同时面临以上三个问题 → MOFA2 探索 + DIABLO 验证（两组结果互补）

### 样本量足够吗？

多组学整合对样本量的要求比单组学高：

- **n < 15**：仅适合两组学 PLS。多组学方法结果不稳定。
- **n = 15-30**：MOFA2 可用，但需限制因子数（建议 <= 5）。
- **n >= 30**：MOFA2 和 DIABLO 均适用。因子估计稳定。
- **n >= 50**：充分。可以考虑更复杂的模型结构和交叉验证。

### 有分类标签（表型/处理组）吗？

- **有明确分组**（如：抗病 vs 感病、干旱 vs 对照、不同品种）：
  → 用 DIABLO。它能找到最大化组间差异的多组学特征。
  → 但注意：DIABLO 需要每个组有足够样本（至少 5-8 个）。

- **无明确分组**（如：自然群体的多组学图谱）：
  → 用 MOFA2。无监督学习，从数据本身发现结构。
  → 或两组学 PLS，如果只有两种组学。

- **有连续性状**（如：产量、株高、含油量）：
  → 可以将其编码为 DIABLO 的连续型 outcome。
  → 或先用 MOFA2 找因子，再与性状做关联分析。

## 选择你的整合方法

### MOFA2 — 无监督多组学因子分析

从多种组学数据中学习隐藏的"因子"，每个因子捕获跨组学的协变信号。

**适合**：
- 3 种及以上组学
- 探索性分析：不知道期待什么样的结果
- 想发现"数据自己说的故事"
- 自然群体、多样性面板、多品种比较

**不适合**：
- 样本量 < 15（因子不稳定）
- 你已有明确的分类标签且分类是你的首要目标（用 DIABLO）

**MOFA2 输出的解读**：
- 每个因子解释了各 omics 中多少方差（variance decomposition plot）
- 每个因子在哪些特征上权重最大（top weights）→ 理解因子的生物学意义
- 因子得分在每个样本上的值 → 看样本分组/排序

### DIABLO — 有监督多组学判别分析

同时分析多种组学，找到最能区分预设类别的特征组合。

**适合**：
- 有明确的分类标签（处理组、表型级别、品种类别等）
- 目标是"哪些组学特征共同决定类别差异"
- 你需要分类器的生物学可解释性

**不适合**：
- 无分类标签
- 分类标签的生物学基础不明确（DIABLO 总会找到差异，但可能没有生物学意义）
- 样本量分配极不均衡

**DIABLO 的关键决策**：
- **design matrix**：组学间的关系权重。通常设 0.1（弱关联），不要设太高否则过拟合。
- **ncomp**：保留的成分数。通过 perf() 函数的 error rate 选择。
- **keepX**：每个成分保留的特征数。用 tune() 函数调优。

### mixOmics PLS/sPLS — 两组学关联

两组学之间的关联分析，找到协变结构。

**适合**：
- 只有两种组学（如：转录组 + 代谢组）
- 想看"基因表达与代谢物积累的关系"
- 想识别关键调控基因-代谢物对

**不适合**：
- 三种及以上组学（用 MOFA2 或 DIABLO）
- 需要多重比较校正（sPLS 本身已做变量选择）

### MOFA2 因子的生物学解读和验证

MOFA2 输出因子后，最大的挑战不是技术性的，而是生物学解读。因子的"数字"本身没有生物学含义——需要你赋予它含义：

**因子解读的系统方法**：
1. **检查因子-特征权重 (weights)**：对于每个因子，提取所有特征的权重，按|weight|降序排列。取 top 50-100 个特征。
2. **按组学分类**：这些 top 特征中，来自哪个组学的特征最多？如果因子 1 在转录组中解释了 30% 方差、在代谢组中解释了 45% 方差，但在蛋白质组中解释了 5% 方差——该因子主要代表转录-代谢协变。
3. **按生物学功能分类**：对 top 基因做 GO 富集分析。如果富集的 GO terms 是一致的生物学主题（如"光合作用"、"线粒体呼吸"、"茉莉酸信号"），解释了因子代表的生物学过程。
4. **按代谢物类别分类**：对 top 代谢物做通路富集。如果一个代谢物是某条通路的一部分，且该通路中的多个代谢物都权重高，说明该通路作为一个整体在跨组学协调变化。
5. **检查因子得分 (factor scores) 的模式**：在样本空间中可视化因子得分。样本是否按处理组/品种/时间点分组？哪些样本的因子得分极端（> 2 SD）？极端样本的极端值可能是生物学上有趣的特例。

**因子验证的额外步骤**：
- 如果一个因子的得分与已知变量（如开花时间、产量）显著相关，该因子可能捕获了与该性状相关的多组学协调变化。
- 验证稳定性：对数据做 bootstrap 重抽样，重新估算 MOFA2 因子，看因子结构和 top 特征的一致性。如果 top 特征在 > 80% 的 bootstrap 样本中保持在前 100，因子是稳定的。稳定性低的因子（< 50% 一致）不应该被深入解读。
- 交叉验证：如果数据量足够大，在 80% 样本上训练 MOFA2，在 20% 的 hold-out 样本上检查因子的预测性能（不是所有 MOFA2 实现都支持此功能；需要使用支持 missing-data prediction 的版本）。

## 数据预处理的植物学考量

### 数据归一化：按组学类型的差异化处理

组学整合的"调味"是各层数据在同一个模型中被同等对待。如果转录组的方差是代谢组的 100 倍，MOFA2/DIABLO 的第一个因子可能仅仅是转录组的方差。正确的归一化是整合分析的基础：

**归一化策略对照表**：

| 组学类型 | 推荐归一化 | 替代方案 | 植物特有注意事项 |
|---------|-----------|---------|-----------------|
| RNA-seq | VST/DESeq2 或 rlog | TMM + log2 | 叶绿体/线粒体基因可保留但注释 |
| 代谢组 | log2 + Pareto scaling | log2 + auto scaling | 未知代谢物保留（携带信息） |
| 蛋白质组 | log2 + median centering | Quantile normalization | 移除 Rubisco 主导信号 |
| 甲基化 (array) | M-values (logit) | Beta-mixture quantile | CG/CHG/CHH 三个背景分别处理 |
| ATAC-seq | DESeq2 + VST | CPM + log2 | 线粒体/叶绿体 reads 预去除 |
| 微生物组 (16S/ITS) | CLR (centered log-ratio) | RLE | 根际 vs 根内分类分析 |
| 宏基因组 | CLR 或 TSS | — | 植物 reads 预移除 |

**归一化的错误处理可能导致的问题**：
- 转录组做了 normalize.quantiles()（microarray 时代的做法）→ 在 RNA-seq 中破坏了生物学差异
- 代谢组没有 log 转换 → 高丰度代谢物主导模型，低丰度但关键的代谢物被忽视
- 甲基化用 Beta 值（0-1）而未用 M 值 → Beta 值的方差在接近 0 或 1 时被压缩，违反正态假设
- Pareto scaling vs auto scaling 的选择：auto scaling（每个特征除以标准差）使所有特征方差相等，但会放大噪声特征（方差小的特征通常是噪声）。Pareto scaling（除以标准差的平方根）是更稳健的选择，保留了一定程度的方差差异性

### 转录组

- 过滤低表达基因：至少在 20% 样本中 count >= 10
- 标准化：VST (DESeq2) 或 TMM (edgeR)，然后 log2 转换
- 批量效应：如果样本来自不同批次，用 ComBat 或 RUVseq 校正
- **植物特别注意**：叶绿体和线粒体基因可以保留，但在解释富集结果时加以说明

### 代谢组

- 通常已经过峰对齐和归一化（如 XCMS 或 MS-DIAL 处理）
- 进一步处理：log2 转换 + Pareto scaling（均值中心化 + 除以标准差平方根）
- 缺失值：如果 > 50% 样本缺失 → 删除该代谢物；如果 < 20% → 用最小值的 1/2 填充
- **植物特别注意**：次生代谢物在植物中高度多样化，许多代谢物无法鉴定。保留"未知"代谢物——它们可能携带有意义的信息

### 蛋白质组

- 常用 label-free quantification (LFQ) 强度值
- log2 转换（蛋白质组数据通常已经是 log2 的）
- 缺失值：如果 < 30% → 用最小值插补；如果 > 30% → 删除
- **植物特别注意**：Rubisco 等高丰度蛋白可能主导信号。考虑移除 Rubisco 相关蛋白后再标准化，或使用中位数中心化

### 表观组 (甲基化/DNase/ATAC-seq)

- 甲基化：β 值 (0-1)，用 M 值 (logit 转换) 进行统计分析
- ATAC-seq/DNase-seq：count-based，用 DESeq2 标准化
- **植物特别注意**：植物 CG/CHG/CHH 三种甲基化上下文需分别处理

### 微生物组

- 通常用相对丰度 (relative abundance) 数据
- CLR (centered log-ratio) 转换处理组分性
- 过滤：至少在 10% 样本中相对丰度 > 0.1%
- **植物特别注意**：根际 vs 根内微生物群差异极大，不要混合分析

### 跨平台的样本匹配

在多组学整合中，一个根本的挑战是样本匹配：转录组测序的样本可能是 Leaf3_Control_1，代谢组提取的同一份材料可能被标记为 LC1。如果在分析中把 Leaf3_Control_1 和 LC1 当作不同样本，整合分析的基础就坍塌了。

**样本匹配的层级**：

1. **完全匹配（Same-sample matching）**：同一份组织的同一份提取物，分成两部分分别做转录组和代谢组。这是最理想的情况（技术上最可靠），但很少能实现——RNA 提取需要不同缓冲液，代谢物提取需要不同溶剂。

2. **同组织相邻样本匹配（Adjacent-sample matching）**：同一植株上相邻的叶片/组织片，分别做 RNA 和代谢物提取。空间变异最小，是目前植物多组学最常用的做法。

3. **同处理同批次匹配（Batch-matched sampling）**：同一实验中同一处理的多个生物重复，分别分配给不同组学平台。样本不是同一份组织，但它们在同一实验中经受相同的处理。

4. **不完整匹配（Incomplete matching）**：并非所有样本都做了全组学。比如 200 个样本做了转录组，但只有 50 个做了代谢组。这是现实中极为常见的场景。

**处理不完整匹配的策略**：
- MOFA2 天然支持不完整匹配（每个组学允许不同的缺失模式），这是它相比 DIABLO 的一个重要优势。
- DIABLO 需要完整的组学-组学样本匹配（每个样本必须有所有组学的数据）。如果你的数据是不完整匹配，DIABLO 只能分析匹配的样本对/组。
- 不完整匹配引入了一个新的陷阱：做了代谢组的 50 个样本不一定能代表全部 200 个样本（选择偏差）。在报告结果时，需要明确讨论"代谢组数据的样本选择是否可能引入了系统性偏差"。

**样本名称统一的实战建议**：
1. 在实验开始之前，建立统一的样本 ID 系统（如：物种_品种_处理_生物重复_组织），并确保所有平台使用同一套 ID
2. 在分析代码中使用 ID 映射表，而非硬编码的名称转换
3. 对每一层组学数据，输出样本 ID 列表，用 `intersect()` 检查各层是否有共同样本
4. 在 MOFA2 分析中设置 `samples_in_common = TRUE` 确保只使用共同样本

### 植物特定的多组学实验设计

不同植物生物学问题对多组学设计有不同的要求：

**胁迫响应多组学**（干旱、盐、高温、病原菌）：
- 设计特点：对照 + 胁迫处理，通常加入时间维度（如 0h、6h、24h、48h 胁迫后）
- 每组学的最少样本：n >= 3 生物重复 × 2 处理 × 3-4 时间点 = 18-24 样本。如果少于这个数，时间点间的动态变化无法被可靠估计。
- 植物特有的问题：胁迫处理本身可能不是"均匀的"——干旱处理的土壤水分在盆中可能有梯度，高温处理的生长箱内温度可能有位置差异。这会在组学数据中引入"假"的变异。随机化盆/板的位置。
- 期望结果：MOFA2 通常会产生一个与"胁迫响应"相关的因子（包含跨组学的胁迫响应特征），以及一个与"发育/时间"相关的因子。

**发育多组学**（种子萌发、叶片衰老、果实成熟、花发育）：
- 设计特点：发育阶段作为"处理"，通常 3-6 个阶段
- 挑战：不同组学平台可能需要不同的组织量。RNA-seq 只需要 100 ng 总 RNA，但代谢组可能需要 50-100 mg 鲜重组织，蛋白质组需要更多。同一个发育阶段的组织可能不够分配给所有平台。计划好组织分配。
- 同一发育阶段的样本应该严格同步进行所有组学的采样——如果在不同批次/日子做转录组和代谢组采样，后续整合分析可能会被批次效应混淆。

**品种比较多组学**（多个品种在相同条件下比较）：
- 设计特点：n >= 5 个品种 × 3+ 生物重复
- 目的：理解品种间性状差异的分子基础
- MOFA2 的因子可能将不同品种分开，反映各品种的组学特征
- 在用 DIABLO 时，品种标签作为分类 outcome，结果可以帮助识别区分品种的关键组学特征
- 特别注意：品种间的基因组差异（SNPs）会影响转录组定量（mapping 偏差）和代谢组定量（代谢基因的活性差异）。在结果中讨论基因组变异对组学整合的潜在影响

### 非模式物种的多组学挑战

模式物种（水稻、拟南芥）有良好的基因组注释、代谢物数据库和文献资源。但绝大多数植物是多组学研究中的"非模式物种"。

**非模式物种多组学的额外挑战**：

1. **代谢物鉴定困难**：
   - 模式物种的代谢物有标准品数据库（如 AtMetExpress for Arabidopsis），非模式物种可能只有 20-40% 的代谢物能够被鉴定。
   - 未鉴定代谢物在 MOFA2 中仍然有用（作为特征参与因子估计），但在生物学解读阶段受限。
   - 建议：报告各平台上鉴定特征的比例。对于未知代谢物，至少报告其 m/z 和保留时间，以便后续技术进步后重新查询。

2. **转录本定量偏差**：
   - 非模式物种的参考转录组可能不完整（缺乏某些基因、或包含嵌合转录本）。
   - 低质量参考导致 mapping rate 偏低（< 70%），影响基因定量准确性。
   - 解决方案：使用基于 k-mer 的定量工具（如 Salmon、kallisto）而非基于比对的定量，减少参考转录组不完整性的影响。

3. **功能注释贫乏**：
   - 非模式物种中，50-70% 的基因可能没有 GO annotation。
   - MOFA 因子的 GO 富集分析会受限于此——因子的 top 基因可能富集到"unknown biological process"。
   - 缓解策略：(a) 通过跨物种同源性推断（如用 OrthoFinder 鉴定 orthogroups）；(b) 用 InterProScan 扫描蛋白结构域获得功能提示；(c) 在结果中明确讨论注释缺陷的限制。

4. **多倍体非模式物种**：
   - 很多观赏植物和林木是多倍体（且缺乏参考基因组）。组学整合中同源基因的分离几乎不可能。
   - 实用建议：在基因水平做定量（而非转录本水平），合并同源基因的信号。这降低了生物学分辨度但增加了技术稳健性。

## 常见陷阱

### 批次效应掩盖生物学信号

多组学数据常来自不同实验、不同时间、不同人员。如果 MOFA2 的第一个因子就完美区分了批次——你的数据存在严重的批次效应。

**怎么办**：
1. 在预处理阶段用 ComBat 等工具校正
2. 如果无法校正，在 MOFA2 模型中包含批次协变量
3. 在报告中诚实地讨论批次效应对结果的影响

### 跨平台批次效应的具体表现形式

批次效应在多组学中有几种特定的"伪装"：

**伪装 1：组学=生物学信号**：
- 如果转录组和代谢组分别由不同的核心实验室完成，且实验日期不同，MOFA2 的第一个因子可能恰好将转录组样本和代谢组样本完美分开。这看起来像是一个生物学因子（"转录 vs 代谢"），实际上是完美混淆的批次效应。
- 诊断：检查因子的 top 特征是否只来自一种组学且高度集中——如果因子 1 在转录组中解释了 50% 方差但在代谢组中解释了 2% 方差，这更可能是批次效应而非生物学信号。

**伪装 2：批次信号跨组学蔓延**：
- 如果你先用转录组数据做了批次校正（如 ComBat），但在整合分析中没有对代谢组做同样的校正，MOFA2 可能发现一个"跨组学"因子——但实际上转录组的校正信号与代谢组的未校正信号之间产生了虚假的"协变"。
- 教训：对所有组学在同一时间做批次校正（如果有跨组学的共同批次），或确保每组学至少独立地检查了批次效应。

**伪装 3：缺失数据模式**：
- 在不完整匹配的情况下，缺失数据本身可能携带信息。如果做了代谢组的样本恰好是高表型值样本，MOFA2 因子可能同时反映了代谢组的"存在/缺失"模式和表型差异。
- 在结果中，用不同标记区分有某组学和没有某组学的样本，检查因子得分是否与"是否有某组学数据"相关。

### 组学间的样本名称不一致

转录组的样本叫 "Leaf_Control_1"，代谢组的同一份样本叫 "LC1"——这会把所有样本都当唯一。**在进行任何分析之前，先确认样本名称统一。**

### 多组学整合不等于提高功效

错误认知："我的转录组 DEG 不显著，加上代谢组数据，用 MOFA2 就能找到信号了。"

组学整合不能弥补单个组学的统计功效不足。它能做的是：发现**跨组学协调变化**的信号，这些信号在单组学中可能被噪声掩盖但**并非不可能被发现**。

### 缺失数据模式的系统性偏差

缺失数据（不是一个样本没有某个组学数据，而是在某个组学内，特定特征在某些样本中缺失）是另一个容易被忽略的陷阱。

**缺失的来源**：
- 代谢组：某些代谢物在某些样本中低于检测限（left-censored missing）
- 蛋白质组：某些蛋白在某些样本中未被鉴定到
- 甲基化：某些 CpG 位点在某些样本中覆盖度不足

**MOFA2 对缺失的假设**：
- MOFA2 假设缺失是"随机缺失 (Missing at Random, MAR)"——即缺失模式与观察到的数据有关，而不是与缺失值本身有关。
- 在代谢组中，缺失通常不是 MAR。低浓度的代谢物倾向于缺失（低于检测限），而低浓度本身就携带有生物学信息（该代谢通路不活跃）。
- 在这种情况下，MOFA2 的缺失值处理可能消除重要的生物学信号。

**缓解策略**：
1. 在代谢组中，对于 left-censored missing（低于检测限），不要用随机插补。用半最小值插补（1/2 × 检测限）更接近生物学现实。
2. 在蛋白质组中，对于 missing not at random (MNAR)，用 left-censored 方法（如基于最小值的插补）或专门的 MNAR 插补工具（如 `imputeLCMD` R 包）。
3. 敏感性分析：用两种缺失值处理方法分别做 MOFA2，比较因子结构的稳定性。如果处理方式改变了因子结构，缺失值对结果的影响很大，需要在报告中讨论。
4. 对于缺失率 > 50% 的代谢物/蛋白，即使可能携带信息，它们对整合分析的贡献也有限。删除是合理的选择，但记录删除的特征数量和比例。

### 植物多倍体的同源基因问题

小麦 (6x)、棉花 (4x)、油菜 (4x) 等多倍体作物：
- 同源基因 (homeologs) 在转录组中可能各有不同表达模式
- MOFA2 无法自动区分同源基因——如果它们被当作独立特征，可能拉低单个因子的方差解释
- 建议：先按亚基因组分组做 MOFA2，再比较因子结构的异同

### 相关性不等于因果性

MOFA2 发现的共变因子不代表因果调控关系。比如一个因子同时驱动了某个转录因子和某个代谢物——这不意味着该转录因子直接调控该代谢物的合成。需要额外的实验验证（如酵母单杂/ChIP-qPCR 等）。

### 富集分析需注意背景基因集

对 MOFA2 因子的 top 基因做 GO/KEGG 富集分析时，背景基因集应该是"该组学中存在的所有基因"，而不是"全基因组的基因"。植物参考注释的质量因物种而异——拟南芥和水稻的注释远优于大多数作物。
