# De novo转录组分析结果解读

## 一句话解释

De novo转录组分析：在没有参考基因组的情况下，从头组装转录本并进行功能注释和差异分析。

## 能回答什么

- 该物种有哪些转录本？
- 转录本的功能是什么？
- 不同条件下哪些基因差异表达？
- 有哪些潜在的lncRNA？
- 编码区在哪里？

## 不能回答什么

- 转录本在基因组上的位置？（无参考基因组）
- 基因的调控元件？（需要基因组序列）
- 剪接变体的完整性？（组装可能有碎片）
- 基因家族的组织结构？（需要基因组信息）

## 典型输出

| 文件 | 含义 |
|------|------|
| `Trinity.fasta` | 组装的转录本序列 |
| `busco_output/` | 完整性评估结果 |
| `Trinity_nr.fasta` | 去冗余后转录本 |
| `trinotate_annotation.tsv` | 功能注释结果 |
| `longest_orfs.pep` | 预测的蛋白质序列 |
| `quant.sf` | 表达定量结果 |
| `deseq2_results.csv` | 差异表达结果 |

## 常见结果模式

### BUSCO完整度

- Complete > 80%: 良好的组装质量
- Complete < 60%: 需要改进组装或增加数据

### N50长度

- N50 > 1000bp: 中等质量
- N50 > 2000bp: 良好质量

### 转录本数量

- 10,000-50,000: 典型植物转录组
- > 100,000: 可能有冗余或复杂物种

### 差异表达基因

- 上调/下调分布均匀: 正常
- 某一方向偏多: 检查批次效应

## 结果报告建议

1. 报告组装统计信息（转录本数、N50、BUSCO完整度）
2. 报告功能注释比例（SwissProt、GO、KEGG）
3. 报告差异表达基因数量和分布
4. 报告关键通路的富集分析结果
5. 讨论非模式物种分析的限制
