# De novo转录组分析引导

## 这个分析解决什么问题

用户想在没有参考基因组的情况下进行转录组分析。典型表述：
- "我的物种没有参考基因组"
- "想做de novo转录组组装"
- "想从头组装转录本"

## 你需要问什么

1. **数据类型**: 只有Illumina还是有PacBio/Nanopore长读长数据？
2. **测序策略**: 有多少样本？测序深度？
3. **分析目标**: 转录本组装？功能注释？差异表达？
4. **近缘物种**: 有近缘物种的基因组或转录组吗？
5. **资源需求**: 可用的计算资源？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **高杂合度**: 植物物种杂合度可能很高，影响组装质量
- **多倍体**: 多倍体物种de novo组装更复杂
- **基因家族扩张**: 植物基因家族常有多个拷贝，需要仔细处理
- **非模式物种**: 非模式物种功能注释依赖近缘物种或通用数据库

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"RNA-seq数据(多组织，足够测序深度)、近缘物种参考序列可用"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有单一组织数据；或测序深度不足"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"测序数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
