# eQTL 分析引导

## 这个分析解决什么问题

用户想找影响基因表达的遗传位点。典型表述：
- "哪些 SNP 影响基因表达"
- "这个 GWAS 位点是不是通过影响某个基因表达起作用的"
- "我想做 GWAS 和 eQTL 的共定位分析"

## 你需要问什么

1. **样本量**: 多少样本有配对基因型+表达数据？
   - <50 → 仅能做 cis-eQTL；≥200 → 可以考虑 trans-eQTL
2. **物种和组织**: 什么组织？植物 eQTL 组织特异性极强
3. **多倍体**: 如果是小麦/棉花等，需分亚基因组分析
4. **是否有 GWAS**: 有的话可以做 coloc 共定位分析
5. **标记密度**: 是否能覆盖全基因组？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **多倍体**: 多倍体物种中，read 可能比对到多个同源位点。无法确定表达来自哪个亚基因组拷贝 → eQTL 解读复杂化。需要亚基因组特异性的表达定量
- **组织特异性**: 植物 eQTL 通常具有强烈的组织特异性（根 vs 叶 vs 花）。需要明确分析的组织背景
- **环境响应**: 很多植物 eQTL 仅在特定环境条件下才能检测到（响应 eQTL）。如果在单一环境下采样，会遗漏条件特异性的调控
- **群体大小**: 植物 eQTL 研究通常样本量较小（50-200），限制了 trans-eQTL 的检测能力。cis-eQTL 对样本量要求相对较低

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
