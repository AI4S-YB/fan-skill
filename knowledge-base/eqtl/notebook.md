# 植物 eQTL 分析：分析师笔记

这份笔记帮助你在做植物 eQTL 分析时像一个有经验的植物遗传学家一样思考和做决策。

## 在开始之前：理解你的数据

### 样本量意味着什么？

植物 eQTL 研究的样本量通常比人类小得多。诚实对待统计功效：

- **n < 50**：仅能做 cis-eQTL。即使是 cis，也只能检测到解释 > 20% 表达变异的大效应 eQTL。0 个显著 eGene 是正常结果。
- **n = 50-100**：cis-eQTL 功效可接受。trans-eQTL 基本不可能（功效 < 10%）。
- **n = 100-200**：cis-eQTL 功效良好。trans-eQTL 开始有有限功效，但假阳性率高。
- **n >= 200**：可以考虑 trans-eQTL。但需要非常严格的 FDR 校正。
- **n >= 500**：trans-eQTL 功效达到可用水平，可以检测 trans-eQTL 热点。

### 这是什么物种？

查看 `species_detected` 字段。植物 eQTL 分析中最关键的信息：

- **模式物种（水稻、玉米、拟南芥）**：有公开的 eQTL 数据库和组织表达图谱。你的结果可以与已发表数据比较。
- **多倍体物种（小麦、棉花、油菜）**：必须按亚基因组分别分析。同源基因的表达偏倚会使 trans-eQTL 混淆。亚基因组特异 SNP 注释是必需的。
- **非模式物种**：没有参考 eQTL 数据。重点验证 cis-eQTL（更可靠）。对候选基因做跨物种同源性功能推断。

### 表达数据来自什么组织？

植物 eQTL 的组织特异性极强：
- **叶片**：最常见的采样组织。光合作用相关基因的 eQTL 信号强。
- **根**：养分吸收和胁迫响应基因，与叶片 eQTL 重叠度低。
- **种子/籽粒**：发育调控基因，常有时序依赖性。
- **胁迫处理（干旱、盐、高温）**：诱导条件下会发现新的 eQTL，与正常条件很少重叠。
- **多组织**：如果有多组织数据，务必做组织特异性 vs 共享 eQTL 分类。单一组织的 eQTL 解释力有限。

### 你的标记密度是多少？

- **高密度（>100K SNP）**：适合全基因组 cis 和 trans 扫描。
- **中等密度（10K-100K SNP）**：cis-eQTL 足够，trans-eQTL 功效受限。
- **低密度（<10K SNP）**：仅能做候选基因区域的 cis-eQTL。不要声称是全基因组扫描。
- **RNA-seq 来源的 SNP**：注意 3' 偏倚和等位基因特异性表达的干扰。

## 选择你的 eQTL 方法

### MatrixEQTL — 标准选择

MatrixEQTL 是目前植物 eQTL 分析最常用的工具。原因：
- 速度极快（百万级检验秒级完成）
- 支持加性和 ANOVA 模型
- 直接输出 cis 和 trans 结果
- R 包易安装、易使用

**适合**：
- 几乎所有植物 eQTL 研究
- 样本量 >= 30（样本少则功效低，但不是方法的问题）
- 所有标记密度

**不适合**：
- 有复杂家系结构时（MatrixEQTL 假设独立样本）
- 需要建模等位基因特异表达时

### 什么时候考虑其他方法？

- **LIMIX**：样本间有亲缘关系时（如 MAGIC 群体、NAM 群体）
- **fastQTL**：需要排列检验获得基因水平显著性时
- **TensorQTL**：单细胞 eQTL 或极大数据集

## 隐藏协变量的处理

这是植物 eQTL 分析中最容易被忽视但最关键的一步。

### 为什么需要隐藏协变量？

基因表达受许多非遗传因素影响：环境、发育阶段、批次效应、细胞类型组成。不控制这些因素会导致假阳性 eQTL 和膨胀的检验统计量。

### 两种常用方法：

1. **PCA 方法**：对表达矩阵做 PCA，取前 5-20 个 PC 作为协变量。
   - 简单，直接
   - 可能同时移除真实的遗传信号

2. **PEER 因子**：专为 eQTL 设计的隐变量模型。
   - 推荐的因子数：n/4（如 100 样本 → 25 个因子）
   - 比 PCA 更好地保留遗传信号
   - 需要安装 peer 包（略复杂）

### 你需要加多少协变量？

- n < 50：3-5 个 PCA 或 PEER 因子
- n = 50-100：5-10 个
- n = 100-200：10-20 个
- n > 200：15-30 个

一个实用建议：同时跑两组分析（加和不加隐藏协变量），比较 eGene 数量和 λ。如果加协变量后 eGene 减少 > 50%，说明协变量加多了，可能移除了真实信号。

## eQTL 分析中的植物特有问题

### 多倍体物种的 eQTL 映射

这是植物 eQTL 最独特的挑战。以六倍体小麦（AABBDD）为例：

- 每个"基因"实际有三个同源拷贝（A、B、D 亚基因组各一个）
- RNA-seq reads 可能多映射到多个同源基因组
- 一个同源基因的 SNP 可能在另一个亚基因组上检测为 trans-eQTL（假 trans）

**解决方案**：
1. 仅使用亚基因组特异的 SNP（通过 reads 唯一比对率过滤）
2. 使用同源基因特异表达定量工具（如 HomeoRoq）
3. 分别对每个亚基因组做 cis-eQTL 分析
4. 谨慎解读 trans-eQTL：先检查 SNP 是否可能是另一个亚基因组的 cis 信号

### 组织特异性图谱

植物组织图谱资源：
- **水稻**：Rice eFP Browser、RiceXPro、OryzaExpress
- **玉米**：Maize Atlas (MaizeGDB)、qTeller
- **拟南芥**：AtGenExpress、TRAVA、eFP Browser
- **小麦**：expVIP、Wheat Expression Browser
- **大豆**：SoyBase、Soybean eFP
- **棉花**：CottonGen

在你的 eQTL 结果中，一个 eGene 如果在相关组织中高表达，其 eQTL 信号更可信。

### 环境效应

植物 eQTL 对环境极其敏感：
- 同一基因型在不同环境下可能有完全不同的 eQTL 景观
- "静态 eQTL"（所有环境都检测到）比"动态 eQTL"（仅特定环境）更可靠，也更有生物学意义
- 如果数据来自多环境，考虑 eQTL-by-environment 交互分析
- 胁迫响应 eQTL 通常比正常生长条件下的 eQTL 效应更大

## eQTL 与 GWAS 的共定位

如果同时有 GWAS 结果，共定位分析是将 eQTL 信息转化为功能洞察的关键步骤。

### coloc 分析的核心问题

对于每个 eQTL-GWAS 信号重叠的区域，coloc 回答：
- H0：两者都无关联
- H1：仅 GWAS 有关联
- H2：仅 eQTL 有关联
- H3：两者都有关联，但是独立的两个因果变异
- H4：两者都有关联，且共享同一个因果变异

**PP.H4 >= 0.75** 是常用的"共享因果变异"阈值。

### 植物 coloc 的注意事项

- 植物的 LD 衰减距离通常比人类长（尤其是自交物种），提前计算 LD 范围
- 如果 GWAS 和 eQTL 使用了不同的 SNP 集，先做 LD 匹配
- 多倍体物种需按亚基因组分别做 coloc
- 如果 coloc 结果很多区域都是 H3（独立信号），可能是 LD 差异导致的，而不是真正的独立因果变异

## 常见陷阱

### 零个 eGene 怎么办？

先别换方法。检查以下几点：
1. 样本量是否足够？（n < 30 → 零结果是正常的）
2. 表达数据是否有足够的变异？（检查表达矩阵的方差分布）
3. 协变量是否过多？（太多协变量会移除遗传信号）
4. 基因过滤是否过严？（低表达基因的 eQTL 可能被遗漏）
5. SNP 密度是否过低？（低密度意味着基因附近的 SNP 覆盖不足）

在报告中诚实说明功效局限。可以放宽到建议性 FDR < 0.1 作为探索性结果，但明确标注。

### 大量 trans-eQTL 热点

如果一个位点关联了数十或数百个远距离基因，需要怀疑：
1. 是不是多倍体物种的 cross-homeolog mapping？
2. 是不是着丝粒区域的异常 LD？
3. 是不是表达数据中有未校正的批次效应（这批样本同时在基因型和表达上不同）？
4. 是不是该位点附近有一个 master regulator TF？

如果一个热点同时是多个组织的 trans-eQTL 热点，它很可能是一个真实的调控枢纽。

### 基因型-表达批次效应

一个很隐蔽的陷阱：如果你的样本在基因型测序和 RNA-seq 之间经历了不同的批次，批次效应会完美混淆遗传效应。分析前用 PCA 检查表达数据是否按批次/日期聚类。

### cis vs trans 距离阈值

标准的 1 Mb 阈值对大多数植物是合适的。但注意：
- 自交物种 LD 长，某些真正的 trans-eQTL 可能在 1-5 Mb 范围内
- 着丝粒区域 LD 极长，1 Mb 阈值可能需要在着丝粒区域放宽
- 如果物种的基因组极大（如小麦 ~17 Gb），可以考虑适当调整阈值
