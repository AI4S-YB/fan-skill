# Iso-Seq全长转录组分析引导

## 这个分析解决什么问题

用户想使用PacBio Iso-Seq技术获得全长转录本。典型表述：
- "我有PacBio Iso-Seq数据"
- "想获得全长转录本"
- "想做转录本注释"
- "想分析可变剪接"

## 你需要问什么

1. **数据类型**: PacBio SeQUEL/Sequel II/Revio？是否有Illumina数据用于校正？
2. **分析目标**: 全长转录本鉴定？基因注释？可变剪接分析？
3. **参考基因组**: 有参考基因组吗？注释质量如何？
4. **样本信息**: 多少个样本/组织？是否需要样本间比较？
5. **分析深度**: 基础分析还是深度功能注释？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **大基因组**: 植物基因组通常较大，转录本数量多，需要更多计算资源
- **多倍体**: 多倍体物种的同源基因需要仔细区分
- **可变剪接**: 植物可变剪接研究相对较少，Iso-Seq可以发现新的剪接事件
- **非编码RNA**: Iso-Seq适合发现长链非编码RNA

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"PacBio Iso-Seq数据、Illumina短读长数据(用于校正)、参考基因组完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有Iso-Seq数据缺乏Illumina校正；或缺乏参考基因组"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"测序数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
