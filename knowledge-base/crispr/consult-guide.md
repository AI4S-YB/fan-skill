# CRISPR 基因编辑设计引导

## 这个分析解决什么问题

用户想设计 CRISPR/Cas 基因编辑实验。典型表述：
- "我想敲除这个基因"
- "我想做单碱基替换"
- "怎么设计 sgRNA"
- "有没有脱靶风险"

## 你需要问什么

1. **编辑目标**: Knockout / Knock-in / Base Editing / Prime Editing？
2. **物种和基因组**: 模式物种还是非模式？多倍体吗？
3. **Cas 变体**: SpCas9 / Cas12a / Cas9-NG / Base Editor？
4. **目标基因**: 基因ID？想敲所有同源拷贝还是特定拷贝？
5. **转化方法**: 农杆菌 / 基因枪 / 原生质体？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **多倍体**: 小麦（6x）、棉花（4x）、油菜（4x）等多倍体中，sgRNA 必须靶向所有同源等位基因才能产生完全敲除。需要检查靶位点在各亚基因组中的保守性
- **遗传转化**: 稳定的遗传转化是植物 CRISPR 编辑的限速步骤。不同物种/品种的转化效率差异极大（水稻容易，小麦困难）
- **sgRNA 设计工具**: CRISPOR 和 CRISPR-P 支持主要作物基因组。非模式植物需要上传自定义基因组
- **脱靶评估**: 植物基因组中旁系同源基因（paralogs）丰富，增加了脱靶编辑的风险
- **非编码 RNA 和调控区**: 靶向 lncRNA 或启动子区时，indel 的功能影响难以预测，需要结合表达量变化验证

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
