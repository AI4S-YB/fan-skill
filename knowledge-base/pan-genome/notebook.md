# 植物泛基因组分析：分析笔记

## 在开始之前

植物泛基因组分析的核心是理解"一个基因组不足以代表一个物种"。先问自己：

### 你有什么样的基因组？

查看 genome_list 和 data_profile.json：
- **基因组数量**：<20 → Minigraph 高效；≥20 → PGGB 全比对；100+ → 考虑分染色体
- **基因组大小**：水稻 ~400Mb, 玉米 ~2.3Gb, 小麦 ~17Gb → 决定计算资源需求
- **组装质量**：染色体水平（推荐）vs scaffold 水平 vs contig 水平 → 影响图结构质量
- **是否有单倍型**：二倍体+物种建议包含两个单倍型组装

### 你的科学问题是什么？

- **核心基因组有多大？** → core_variable 分析
- **哪些基因在不同材料间存在/缺失？** → PAV 检测
- **SV 如何影响表型变异？** → SV 基因分型 + GWAS
- **驯化和改良过程中基因组如何变化？** → 野生种+栽培种比较

### 你期望看到什么？

- **核心基因**：通常占基因家族的 40-60%，多为持家基因
- **可变基因**：通常占 20-40%，包括抗病基因、次生代谢基因
- **私有基因**：通常 <10%，可能是组装/注释假象
- **PAV 热点**：抗病基因簇、次生代谢基因簇通常 PAV 最高

## 图构建 vs 线性比较：选择正确的泛基因组策略

### 线性泛基因组方法

传统方法：将每个基因组与一个参考基因组做比对，提取 SVs。

**适合**：
- 已有高质量染色体水平参考基因组（如水稻 Nipponbare）
- 研究目标主要是 PAV 而非结构变异上下文
- 计算资源有限

**局限**：
- 严重受参考偏差（reference bias）影响 → 参考中不存在的序列完全不可见
- 无法捕获复杂结构重排（insertion + inversion + duplication 的组合事件）
- 多拷贝基因家族（如 NBS-LRR）在不同基因组间的真实关系被平面化

### 图基因组方法

构建序列图（sequence graph），每个节点代表一段序列，边代表邻接关系。变体（SNP/indel/SV）编码为图中的替代路径。

**核心优势**：
- 消除参考偏差：每个基因组都是图中的一条路径，没有"中心"参考
- 复杂 SV 的完整表示：嵌套结构变异（nested SV）可以在图中直接表示
- 读段比对质量提升：特别是对于高度多态性的基因组区域（如抗病基因簇）

**何时选择哪种方法**：

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| <5 个高质量基因组 | 线性比较 + Mummer/SyRI | 计算成本低，结果易解释 |
| 5-20 个基因组 | Minigraph 图构建 | 可以捕获主要 SVs，计算可控 |
| 20-100 个基因组 | PGGB 全比对图 | 完整变异信息（SNP+indel+SV） |
| >100 个基因组 | Minigraph-Cactus 渐进式 | 分染色体渐进添加，适合大规模群体 |
| 多倍体物种 | SibeliaZ + 亚基因组分拆 | 必须先分离亚基因组（见下方多倍体章节） |

## 选择你的泛基因组图构建工具

### Minigraph — 快速，适合小规模队列

使用序列到图的比对，逐步添加基因组。

**适合**：<20 个基因组，主要关注 SVs
**不适合**：需要全比对信息的大规模队列

Minigraph 输出的是一个序列骨架图，节点代表序列，边代表比对关系。对于植物，它能很好地捕获大片段 SVs，但不能展示 SNP/indel 级别的变异。

### PGGB — 全比对，适合中等规模

使用 all-vs-all 比对构建全比对泛基因组图。

**适合**：20-100 个基因组，需要完整变异信息
**不适合**：超大基因组（>3Gb）或超多基因组（>100）

PGGB 产生的图包含所有变异信息（SNP + indel + SV），但计算量大。对于植物，建议分染色体运行。

### Cactus — 渐进式，适合大规模复杂队列

使用渐进式比对策略。

**适合**：大基因组 + 多单倍型 + 进化距离远的材料
**不适合**：只需要快速概览的小项目

Cactus 在处理植物复杂基因组（如小麦六倍体）时表现最好，但配置复杂。

## 核心基因组分类

### 分类标准

植物泛基因组通常按以下频率分类：

- **核心基因 (Core)**：≥95% 个体中存在 → 物种必需的持家功能
- **软核心基因 (Soft-core)**：80-95% 个体中存在
- **可变基因 (Variable/Dispensable)**：15-80% 个体中存在
- **私有基因 (Private)**：<15% 个体中存在

### 核心基因组饱和分析

核心基因组是否已经达到平台期？这是判断样本量是否足够的关键指标。

**方法**：
1. 随机抽样：从 2 个基因组开始，逐步增加基因组数量，每次计算核心基因数
2. 拟合 Heap's Law 模型：核心基因数 = k * N^(alpha)，其中 N = 基因组数
3. 如果 alpha < 0.01 → 核心基因组已经饱和，继续加样本不会显著减小核心基因组

**植物实测数据参考**：
- 水稻 (3010 份)：核心基因组在 ~50 份材料时接近饱和
- 大豆 (26 份)：alpha ≈ 0.02 → 仍需更多基因组才能达到饱和
- 大麦 (20 份)：核心基因组仍有下降趋势 → 样本量不足

### 植物特殊考虑

- 抗病基因（NBS-LRR）几乎总是可变的 → 预期在可变基因组中富集
- 次生代谢基因（如萜类合酶、细胞色素 P450）→ 高度可变
- 转录因子 → 通常是核心或软核心
- 如果私有基因超过 15% → 检查是否因为不完整的注释导致

## PAV 检测策略

### PAV 在作物性状变异中的角色

Presence/Absence Variation 是作物驯化和改良的关键驱动力：

- **水稻**：Rc 基因（红色果皮）在栽培稻中的缺失 → 白色米粒
- **小麦**：VRN1 基因的拷贝数变异 → 春化需求差异
- **玉米**：tb1 基因上游的转座子插入/缺失 → 分蘖数变化
- **大豆**：GmFT 基因家族的 PAV → 光周期适应性

PAV-GWAS 是 GWAS 分析的升级版：不仅考虑 SNP，还考虑基因的完全存在/缺失对表型的影响。

### PanGene

基于基因序列比对的 PAV 检测。对每个参考基因，在其他基因组中寻找存在/缺失证据。

**适合**：有高质量参考基因组的物种
**Plant 注意**：对多拷贝基因（如抗病基因簇），可能高估 PAV

### PanGenie

基于 k-mer 的 PAV 检测。不需要完整组装，可以用短读长测序数据。

**适合**：重测序数据 → 不需组装即可检测 PAV
**Plant 注意**：高重复基因组（如玉米 85% 重复序列）k-mer 特异性降低

### 注释投影 (Annotation Projection) 的策略

将参考基因组的基因注释映射到其他基因组时的常见方法：

1. **Liftoff**：基于全基因组比对的基因结构投影
2. **GeMoMa**：基于同源性的基因模型映射，利用 RNA-seq 数据做调整
3. **PanGenome Annotation Pipeline**：专门的泛基因组注释流程

**常见陷阱**：
- 如果只用一条蛋白序列做 Liftoff → 可变剪接体信息丢失
- 不同基因组的基因模型不一致 → PAV 可能是注释差异而非真正的基因缺失
- **推荐**：对所有基因组进行统一注释（用同一套 pipeline 做 ab initio + homology + RNA-seq 证据组合）以减少注释偏差

## 多倍体泛基因组挑战

### 核心问题

六倍体小麦：A、B、D 三个亚基因组各有自己版本的每个基因。如果混在一起做泛基因组：
- 同源基因可能被错误地聚类为同一个基因家族
- 核心基因组大小会被低估（因为"同一个基因"在 3 个亚基因组中都有）
- 亚基因组间的结构变异（如染色体易位）会使图结构混乱

### 两种策略

**策略 A：亚基因组感知 (Subgenome-aware)**
1. 先用 SibeliaZ/k-mer 谱系分离亚基因组
2. 将每个亚基因组分别做泛基因组图
3. 最后比较亚基因组间的 PAV 差异

**适合**：已知亚基因组组成（小麦 A/B/D、棉花 At/Dt、油菜 A/C）
**优点**：生物学意义清晰，同源基因不会混淆
**缺点**：分亚基因组本身是一个困难的问题，容易出错

**策略 B：合并分析 (Merged)**
1. 所有亚基因组混在一起构建泛基因组图
2. 以全部基因（包括同源基因）做聚类分析
3. 通过同线性（synteny）信息追溯基因家族的亚基因组来源

**适合**：亚基因组组成不明确或亚基因组间分化较低
**优点**：不依赖准确的分亚基因组
**缺点**：核心基因组定义模糊，需要额外信息解释

### 推荐做法

如果物种是多倍体且有明确的亚基因组参考：
1. 先分离亚基因组
2. 在每个亚基因组内做泛基因组分析
3. 再整合亚基因组间比较

如果不确定亚基因组组成：
1. 用 merged 策略先做全局分析
2. 再用同线性分析追溯基因来源

## 作物特异性考虑

### 水稻 — 最大的泛基因组资源

- **3K RGP (3000 Rice Genomes Project)**：3010 个品种的全基因组重测序
- **Rice Pan-genome (RPAN)**：包含 > 20 个高质量 de novo 组装
- **特殊性状**：籼粳分化（indica/japonica divergence）导致两个亚群的核心基因组不同 → 做泛基因组时需要同时包含籼稻和粳稻材料
- 水稻基因组较小（~400Mb），可以对 50+ 个材料做全比对泛基因组图而无需分染色体

### 玉米 — 高结构变异负荷

- **NAM (Nested Association Mapping) founders**：25 个创始亲本的 de novo 组装
- 玉米基因组中 85% 是转座子 → 结构变异密度高
- 不同自交系间的全基因组序列差异可达 1-2%（远高于人类群体的 0.1%）
- **建议**：用 PGGB 的 `-L` 选项分染色体构建 + 需要用超过 128 GB 内存处理单个染色体

### 大豆 — 多倍体古演化

- 大豆经历了全基因组复制（~13 Mya），古多倍体化为二倍体
- 70% 的基因以同源基因对（homoeologous gene pairs）存在
- 泛基因组分析需要区分"同源基因对"和"等位基因变异"
- 使用 SoyBase 的基因家族分类信息辅助分析

### 小麦 — 资源需求最大的挑战

- ~17 Gb 六倍体基因组，80% 重复序列
- 单个品种的全基因组装需要 TB 级内存和数周计算时间
- **实用建议**：
  - 不一定要做全基因组水平的 PGGB
  - "泛基因（pan-gene）"方法更实用：对基因区域（外显子 ± 2kb）做聚类
  - 分别对 A、B、D 亚基因组做分析，然后整合

## SV 基因分型

### vg giraffe

将短读长/长读长比对到泛基因组图进行 SV 基因分型。

**适合**：已有泛基因组图，需要对大量样本进行 SV 分型
**特殊 Plant 需求**：对多倍体，建议先分亚基因组再分别分型

## 常见陷阱

### 多倍体的困扰

参见上方"多倍体泛基因组挑战"章节。核心要点：不分离亚基因组 = 结果难以解释。

### 组装质量偏差

低质量组装 → PAV 假阳性（基因看起来"缺失"，其实是组装不完整）
- 检查每个基因组的 BUSCO 完整性
- 如果某个基因组的 BUSCO < 80%，解释其 PAV 时需谨慎

### 参考偏差 (Reference Bias)

在泛基因组分析中残留的参考偏差：
- 如果以某个基因组作为"起始种子"（如 Minigraph 使用第一个基因组作为骨架）→ 该基因组中的序列在图中表示最好
- **缓解**：用不同基因组作为起始种子重复分析，比较结果差异
- **检查**：计算每个基因组在图中的路径长度与基因组大小的比值 → 比值异常小可能意味着该基因组在图中的代表性不足

### 注释投影错误

将一个基因组的基因注释投影到另一个基因组时：
- 基因融合和分裂：由于组装错误或真实的结构变异，一个基因可能在另一个基因组中被分成两部分
- **缓解**：使用 gene model refinement（如 GeMoMa 的 RNA-seq 调整模式）
- **验证**：对投影后基因 > 2 kb 差异的基因进行人工检查

### 核心基因组不饱和

如果样本量不足 → 核心基因组被高估 → 真正的核心基因被错分为可变基因
- **检查**：做核心基因组饱和曲线（见上方"核心基因组饱和分析"）
- **规则**：如果 alpha > 0.02，样本量可能不足，需谨慎解释结果

### 计算资源需求

| 基因组数量 | 基因组大小 | 推荐内存 | 推荐 CPU | 预计时间 |
|-----------|-----------|---------|---------|---------|
| 10 | ~400Mb | 64 GB | 16 | 2-4 小时 |
| 20 | ~1Gb | 128 GB | 32 | 12-24 小时 |
| 50 | ~400Mb | 256 GB | 64 | 2-5 天 |
| 100 | ~1Gb | 512 GB | 128 | 1-2 周 |

- **Rice Pan-genome (RPAN)**：水稻泛基因组资源
- **Sorghum Pan-genome (SorGSD)**：高粱泛基因组
- **Soybean Pan-genome (SoyBase)**：大豆泛基因组
- **Maize Pan-genome (MaizeGDB)**：玉米泛基因组
