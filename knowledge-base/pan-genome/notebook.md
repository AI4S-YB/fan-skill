# 植物泛基因组分析：分析笔记

## 在开始之前

植物泛基因组分析的核心是理解"一个基因组不足以代表一个物种"。先问自己：

### 你有什么样的基因组？

查看 genome_list 和 data_profile.json：
- **基因组数量**：<20 → Minigraph 高效；≥20 → PGGB 全比对；100+ → 考虑分染色体
- **基因组大小**：水稻 ~400Mb, 玉米 ~2.3Gb, 小麦 ~17Gb → 决定计算资源需求
- **组装质量**：染色体水平（推荐）vs scaffold 水平 vs contig 水平 → 影响图结构质量
- **是否有单倍型**：二倍体+物种建议包含两个单倍型组装

### 你的科学问题是什么？

- **核心基因组有多大？** → core_variable 分析
- **哪些基因在不同材料间存在/缺失？** → PAV 检测
- **SV 如何影响表型变异？** → SV 基因分型 + GWAS
- **驯化和改良过程中基因组如何变化？** → 野生种+栽培种比较

### 你期望看到什么？

- **核心基因**：通常占基因家族的 40-60%，多为持家基因
- **可变基因**：通常占 20-40%，包括抗病基因、次生代谢基因
- **私有基因**：通常 <10%，可能是组装/注释假象
- **PAV 热点**：抗病基因簇、次生代谢基因簇通常 PAV 最高

## 选择你的泛基因组图构建工具

### Minigraph — 快速，适合小规模队列

使用序列到图的比对，逐步添加基因组。

**适合**：<20 个基因组，主要关注 SVs
**不适合**：需要全比对信息的大规模队列

Minigraph 输出的是一个序列骨架图，节点代表序列，边代表比对关系。对于植物，它能很好地捕获大片段 SVs，但不能展示 SNP/indel 级别的变异。

### PGGB — 全比对，适合中等规模

使用 all-vs-all 比对构建全比对泛基因组图。

**适合**：20-100 个基因组，需要完整变异信息
**不适合**：超大基因组（>3Gb）或超多基因组（>100）

PGGB 产生的图包含所有变异信息（SNP + indel + SV），但计算量大。对于植物，建议分染色体运行。

### Cactus — 渐进式，适合大规模复杂队列

使用渐进式比对策略。

**适合**：大基因组 + 多单倍型 + 进化距离远的材料
**不适合**：只需要快速概览的小项目

Cactus 在处理植物复杂基因组（如小麦六倍体）时表现最好，但配置复杂。

## 核心基因组分类

### 分类标准

植物泛基因组通常按以下频率分类：

- **核心基因 (Core)**：≥95% 个体中存在 → 物种必需的持家功能
- **软核心基因 (Soft-core)**：80-95% 个体中存在
- **可变基因 (Variable/Dispensable)**：15-80% 个体中存在
- **私有基因 (Private)**：<15% 个体中存在

### 植物特殊考虑

- 抗病基因（NBS-LRR）几乎总是可变的 → 预期在可变基因组中富集
- 次生代谢基因（如萜类合酶、细胞色素 P450）→ 高度可变
- 转录因子 → 通常是核心或软核心
- 如果私有基因超过 15% → 检查是否因为不完整的注释导致

## PAV 检测策略

### PanGene

基于基因序列比对的 PAV 检测。对每个参考基因，在其他基因组中寻找存在/缺失证据。

**适合**：有高质量参考基因组的物种
**Plant 注意**：对多拷贝基因（如抗病基因簇），可能高估 PAV

### PanGenie

基于 k-mer 的 PAV 检测。不需要完整组装，可以用短读长测序数据。

**适合**：重测序数据 → 不需组装即可检测 PAV
**Plant 注意**：高重复基因组（如玉米 85% 重复序列）k-mer 特异性降低

## SV 基因分型

### vg giraffe

将短读长/长读长比对到泛基因组图进行 SV 基因分型。

**适合**：已有泛基因组图，需要对大量样本进行 SV 分型
**特殊 Plant 需求**：对多倍体，建议先分亚基因组再分别分型

## 常见陷阱

### 多倍体的困扰

六倍体小麦：A、B、D 三个亚基因组各有自己版本的每个基因。如果混在一起做泛基因组：
- 同源基因可能被错误地聚类为同一个基因家族
- 核心基因组大小会被低估（因为"同一个基因"在 3 个亚基因组中都有）
- **建议**：先用 SibeliaZ 等方法分离亚基因组

### 组装质量偏差

低质量组装 → PAV 假阳性（基因看起来"缺失"，其实是组装不完整）
- 检查每个基因组的 BUSCO 完整性
- 如果某个基因组的 BUSCO < 80%，解释其 PAV 时需谨慎

### 计算资源需求

| 基因组数量 | 基因组大小 | 推荐内存 | 推荐 CPU | 预计时间 |
|-----------|-----------|---------|---------|---------|
| 10 | ~400Mb | 64 GB | 16 | 2-4 小时 |
| 20 | ~1Gb | 128 GB | 32 | 12-24 小时 |
| 50 | ~400Mb | 256 GB | 64 | 2-5 天 |
| 100 | ~1Gb | 512 GB | 128 | 1-2 周 |

### 物种特异性数据库

- **Rice Pan-genome (RPAN)**：水稻泛基因组资源
- **Sorghum Pan-genome (SorGSD)**：高粱泛基因组
- **Soybean Pan-genome (SoyBase)**：大豆泛基因组
- **Maize Pan-genome (MaizeGDB)**：玉米泛基因组
