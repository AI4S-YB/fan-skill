# 泛基因组分析引导

## 这个分析解决什么问题

用户想分析一个物种多个个体/品种的基因组中哪些是共有的、哪些是可变的。典型表述：
- "这个物种的核心基因组有多大"
- "不同材料间哪些基因存在/缺失 (PAV)"
- "结构变异 (SV) 和表型有什么关系"

## 你需要问什么

1. **基因组数量和质量**: 几个基因组？染色体水平还是 scaffold？
2. **科学问题**: 核心基因组大小 / PAV 分析 / SV-GWAS？
3. **基因组大小**: 决定着计算资源需求。>3Gb 需分染色体
4. **多倍体**: 需分亚基因组后再构建泛基因组图
5. **工具选择**: <20 基因组用 Minigraph；>20 用 PGGB

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **结构变异 (SV)**: 植物泛基因组中 PAV（存在/缺失变异）是最重要的变异类型。很多农艺性状相关基因（抗病、耐逆、品质）是 PAV——在部分种质中存在而在另一些中缺失
- **种质选择**: 构建泛基因组时种质的选择至关重要——应覆盖物种的核心多样性（地方品种 + 野生近缘种 + 优良品种）
- **组装质量**: 泛基因组分析对每个种质的基因组组装质量要求高。graph-based 泛基因组（minigraph、PGGB）比 iterative mapping 方法对组装质量更敏感
- **非参考基因组**: 基于单一参考基因组的分析会遗漏非参考种质特有的基因。泛基因组分析的核心价值就是捕获这些"暗物质"基因

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"多个基因组组装(染色体级别)、基因注释、表型数据完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"部分基因组仅为scaffold级别；或缺乏基因注释"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"基因组组装尚未完成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
