# 蛋白质组学分析引导

## 这个分析解决什么问题

用户想分析蛋白质表达谱及其实验条件下的变化。典型表述：
- "处理组和对照组的蛋白质谱有什么不同"
- "我想验证这个候选蛋白的丰度变化"
- "想做蛋白互作网络分析"

## 你需要问什么

1. **采集模式**: DDA / DIA / TMT？影响 missing values 程度
2. **物种和蛋白数据库**: 有 UniProt 参考蛋白质组吗？
3. **实验设计**: 处理/对照？每组几个重复？
4. **分析目标**: 差异蛋白 / 通路富集 / PPI网络 / 磷酸化组？
5. **非模式物种**: 可能需要用转录组构建样本特异性蛋白库

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **蛋白提取**: 植物组织含大量细胞壁多糖、多酚、色素、脂质。标准动物组织提取方法直接应用会失败。TCA/丙酮沉淀或酚抽提法适合大多数植物组织
- **蛋白数据库**: 非模式植物优先用同物种的 RNA-seq 翻译蛋白库（而非近缘物种参考蛋白质组）。物种特异性蛋白会被近缘物种数据库遗漏
- **RuBisCo 干扰**: 叶片组织中 RuBisCo（核酮糖-1,5-二磷酸羧化酶/加氧酶）可占总蛋白的 30-50%。高丰度 RuBisCo 会掩盖低丰度蛋白的鉴定。可以考虑 depletion 策略或使用高分辨质谱
- **翻译后修饰**: 植物蛋白的翻译后修饰（磷酸化、泛素化、乙酰化）在信号转导和胁迫响应中至关重要。常规蛋白质组学只能看丰度，不能看修饰状态

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
