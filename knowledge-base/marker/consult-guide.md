# 分子标记开发引导

## 这个分析解决什么问题

用户想把 GWAS/QTL 发现的位点转化为实验室可用的检测标记。典型表述：
- "怎么把 GWAS 结果做成 KASP 标记"
- "我想设计引物区分这个 SNP 的两个等位基因"
- "需要做标记辅助选择，用什么标记类型"

## 你需要问什么

1. **标记类型**: KASP / InDel / SSR / dCAPS？
2. **目标位点**: 有具体 SNP 位置和侧翼序列吗？因果变异还是 LD 标记？
3. **检测场景**: MAS前景选择 / MABC背景选择 / 指纹图谱？
4. **通量需求**: 多少个标记 × 多少个样本？
5. **物种和倍性**: 多倍体需确保引物特异性

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **育种系统**: 自交物种（水稻、大豆）开发 KASP/dCAPS 标记——单个 SNP 即可区分纯合等位基因；异交物种（玉米）开发标记时需考虑杂合度
- **标记通量**: 少量标记（1-10 个）适合 KASP/TaqMan 验证；中等（10-100）适合育种芯片；大规模（1000+）用于 GS 或 GWAS
- **多态性验证**: 开发的标记在目标育种群体中必须有多态性。一个在 GWAS 群体中显著的 SNP 可能在育种亲本中无多态性（固定了）
- **侧翼序列**: 植物基因组高重复序列 → 设计引物时需要 BLAST 验证特异性，避免扩增旁系同源序列

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
