# 杂交预测分析引导

## 这个分析解决什么问题

用户想在种植杂交种之前预测哪些亲本组合会产生最佳表现。典型表述：
- "哪些亲本配组最好"
- "怎么划分杂优群"
- "我想预测未测试杂交组合的表现"

## 你需要问什么

1. **实验设计**: NCII / 双列杂交 / 因子设计？
2. **亲本数据**: 有基因型吗？还是只有系谱？
3. **育种目标**: 筛选最佳组合 / 划分杂优群 / 预测未测组合？
4. **作物**: 水稻(加性为主)还是玉米(非加性也重要)？
5. **Baker's ratio**: GCA 占比高吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **杂种优势**: 植物杂种优势（heterosis）的遗传基础复杂（显性假说、超显性假说、上位性假说）。杂种预测模型需要能捕捉这些非加性效应
- **育种系统**: 自交作物（水稻、小麦）的杂交种生产依赖于雄性不育系（CMS/EGMS）；异交作物（玉米）的杂种生产相对容易
- **杂种组合数量**: 可能的杂种组合数 = 母本数 × 父本数。即使中等规模的亲本池（如 100 × 100），可能的组合也达 10,000 个——实际测试所有组合不可能，因此需要 GS 预测
- **多性状**: 杂种品种需要同时在多个性状上表现优良（产量、品质、抗逆）。预测模型应评估多性状的综合表现

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
