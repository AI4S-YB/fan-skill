# 基因流分析引导

## 这个分析解决什么问题

用户想研究物种间的基因流动和渐渗。典型表述：
- "我想分析物种间的基因流"
- "想做渐渗分析"
- "想研究杂交事件"
- "想分析祖先基因流"

## 你需要问什么

1. **物种信息**: 有多少个物种？它们的系统发育关系是什么？
2. **群体样本**: 每个物种有多少个样本？采样覆盖范围？
3. **基因组数据**: 有全基因组数据还是特定区域？
4. **分析目标**: 检测基因流？估计基因流方向？量化渐渗程度？
5. **外群**: 有合适的外群物种吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **杂交普遍性**: 植物杂交非常普遍，种间基因流常见
- **多倍化**: 多倍化常伴随杂交，需要区分渐渗和多倍化
- **自交/异交**: 繁殖方式影响群体结构和基因流检测
- **生境连续性**: 植物群体分布影响基因流模式

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"多物种VCF文件、群体样本信息、系统发育树完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"有VCF但缺乏系统发育树"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"群体遗传数据尚未采集"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
