# 植物表型数据分析：分析笔记

## 在开始之前

表型分析是所有后续分析的基础。GWAS 的功效、GS 的准确性——都受表型数据质量的约束。

### 先看数据质量

- **变异系数 (CV)**：田间试验 CV < 15% 是好的，CV > 30% 需要检查试验设计或数据录入
- **缺失值模式**：某些基因型在多个环境中系统缺失 → 可能是这些基因型在这些环境中未存活或不适应
- **分布形态**：严重偏态 → 考虑 Box-Cox 变换

---

## 试验设计类型：选择正确的设计

试验设计是表型数据质量的根基。设计选错了，后续分析无法补救。

### 随机完全区组设计 (RCBD) — 最常用

将试验田划分为若干个区组（block），每个区组内包含所有参试基因型，随机排列。

**适合**：
- 参试基因型 ≤ 50 个
- 田间土壤变异呈块状分布（大多数田块如此）
- 单一地点、单一处理

**植物的实际约束**：
- 区组不能太大——如果区组内土壤变异超过了区组间变异，RCBD 的收益减小。一般区组内小区数 ≤ 30-50 为宜。
- 如果参试基因型 > 100 → RCBD 的区组过大，区组内还是存在显著的土壤变异 → 需要 Alpha-Lattice。

**RCBD 的统计模型**：
```
Y_ij = μ + Genotype_i + Block_j + ε_ij
```
区组作为随机效应。当区组间变异显著时，RCBD 相对于完全随机设计可以将误差方差降低 20-50%。

### Alpha-Lattice (不完全区组) — 大规模试验的标准

将区组进一步划分为不完全的"小区组 (incomplete block / lattice)"，每个小区组内只包含部分基因型。

**适合**：
- 参试基因型 > 50（特别是 > 100 时强烈推荐）
- 育种项目的产量试验（通常 100-1000 个基因型）
- 田间变异复杂的田块

**为什么 Alpha-Lattice 对植物育种很重要**：
- 育种项目的产量试验从 F5/F6 代开始，基因型数量通常 200-1000 个。RCBD 不可能容纳这么多基因型在一个区组内。
- Alpha-Lattice 通过不完全区组在"空间校正"和"可操作性"之间取得平衡。
- 相对于 RCBD，Alpha-Lattice 通常能将误差方差再降低 10-30%。

**分析工具**：
- ASReml-R（最佳，商业软件）
- lme4 + sommer（R，免费替代）
- SpATS（R，空间分析 + 平滑）

### Augmented Design（增广设计）— 育种早期代

早期世代（F2-F4）每个基因型的种子量极少，只能种一个小区（无重复）。

**策略**：
- 每隔 5-10 个测试基因型插入一个"对照品种"（有重复）。
- 用对照品种的表现估计空间变异，然后用空间模型校正测试基因型的表型值。
- 这不能替代真正的重复，但在早期代筛选中有实用价值。

**现实**：Augmented Design 下的遗传力估计非常粗糙。不要基于 Augmented Design 的 BLUP 值做出"保留 or 淘汰"的决定——应该用更宽松的筛选阈值（保留前 50% 而非前 10%）。

### p-Rep Design（部分重复设计）— 中间选择

每个基因型重复 1.1-1.5 次（部分基因型有 2 次重复，其余只有 1 次）。

**适合**：
- 种子量不足以对所有基因型做 2 次重复
- 但对遗传力的要求比 Augmented 高
- 通常用于育种早-中期的筛选

### 试验设计选择决策指南

| 场景 | 推荐设计 | 重复数 | 典型遗传力 |
|------|---------|--------|----------|
| 育种后期产量试验 (n=50-500) | Alpha-Lattice | 2-4 | H²=0.4-0.8 |
| 育种后期产量试验 (n=30-50) | RCBD | 3-4 | H²=0.5-0.8 |
| 早期代筛选 (n=500-2000) | Augmented | 0 + 对照 | H² 不可靠 |
| 早中期筛选 (n=200-500) | p-Rep | 1.2-1.5 | H²=0.2-0.5 |
| GWAS 多样性面板 | RCBD/Alpha | 2-3 | H²=0.5-0.9 |
| 精密表型（生理性状） | RCBD | 3-5 | H²=0.6-0.9 |

### 遗传力估计

遗传力是决定 GWAS 和 GS 能否成功的关键参数：
- H² < 0.1 → GS 预测准确性上限仅 ~0.32。考虑是否可以增加重复或环境数。
- H² = 0.3-0.5 → 中等遗传力，大多数农艺性状在这个范围。
- H² > 0.7 → 高遗传力，标记辅助选择可能已足够有效。

## 遗传力深度解读：H² vs h²

这是一个植物育种中最常被混淆的概念。正确理解这两种遗传力非常关键。

### 广义遗传力 (H², broad-sense heritability)

**定义**：H² = V_G / V_P = V_G / (V_G + V_E)

即遗传方差占总表型方差的比率。这里的 V_G 包括了加性、显性和上位性方差。

**什么时候使用 H²**：
- 评估单环境试验的数据质量——一个单环境试验的 H² 应该 > 0.5，否则这个环境的数据可能有严重的误差问题。
- 比较不同性状在同一试验中的"可遗传性"——H²=0.8 的性状比 H²=0.3 的性状更容易通过表型选择改良。
- 计算遗传进展的预期值。

**如何计算 H²**：
```
H² = 1 - (MSE / (r * σ²_g + MSE))
# 基于方差分析表
# MSE = 误差均方
# r = 重复数
# σ²_g = 遗传方差
```

### 狭义遗传力 (h², narrow-sense heritability)

**定义**：h² = V_A / V_P

即加性遗传方差占总表型方差的比率。V_A 是育种值（GEBV）的方差来源。

**什么时候使用 h²**：
- GS 准确性上限——GS 预测的理论最大准确性 = √h²。如果你的性状 h²=0.3，你无法期望 GS 准确性 (r) 超过 ~0.55（即 √0.3）。
- 预测对表型选择的响应——ΔG = i × h² × σ_P（育种家公式）。
- 评估标记辅助回交的效率。

**植物中 H² 和 h² 的数量关系**：
- 在自交物种的纯合群体中：非加性方差很小（因为大多数个体纯合），H² ≈ h²。
- 在杂交种/异交群体中：非加性方差可以占到总遗传方差的 20-50%，H² > h²。
- 在 F2 群体中：有大量显性效应，H² >> h²。

**实际建议**：
- 如果用户混淆了 H² 和 h²，先确认他们在分析自交系（H² ≈ h²）还是杂交种/分离群体（需要区分）。
- 对于大多数育种应用（GS、选择），需要的是 h²，不是 H²。

### 什么遗传力值意味着什么——育种决策视角

| H² 范围 | 解释 | 育种意义 |
|---------|------|---------|
| < 0.2 | 极低 | 当前试验设计和环境数量不足以对该性状做出任何可靠判断。需要增加重复、增加环境、改进表型测量方法。 |
| 0.2-0.4 | 低 | 只能筛选"极端表现"的基因型。不能区分中等的差异。GS 准确率上限 ~0.45-0.63。 |
| 0.4-0.6 | 中等 | 大多数农艺性状（产量、品质）的范围。GS 可行但准确率中等。可以淘汰最差的 30%。 |
| 0.6-0.8 | 高 | 可靠选择。标记辅助选择可行。GS 准确率上限 ~0.77-0.89。 |
| > 0.8 | 极高 | 表型选择本身就很有效。某些高度遗传的物候性状（抽穗期、株高）和品质性状（蛋白质含量）。 |

### 提升低遗传力性状的策略

1. **增加环境数**：这是最有效的方法。H² 的提升约为 n_env 的递增函数。从 2 个环境增加到 4 个环境，H² 可能从 0.3 提升至 0.5。
2. **增加重复数**：在一个环境内增加重复数。效果低于增加环境数，但成本低。从 2 重复到 4 重复，H² 可能从 0.4 提升至 0.55。
3. **空间校正**：用空间模型校正田间变异。H² 通常提升 0.05-0.15。
4. **改进表型测量方法**：用手持光谱仪测 NDVI 代替目视评分冠层衰老。用图像分析代替目视评分。自动化和标准化减少人为误差。
5. **选择更均匀的试验田**：这是试验规划阶段的考虑。如果某个田块的 CV 持续 > 20%，换田。

## 空间变异：田间试验的隐藏敌人

### 什么时候需要空间校正？

- 大田试验（>100 个小区）→ 几乎总有益
- 观察到明显的块状或条带状空间模式 → 必须做
- 小试验（<50 个小区）→ 是否需要取决于布局

空间校正后 H² 通常会提高——这意味着你之前把空间变异错误地归入了误差方差。

### 如何检测空间变异

**目视检查**（第一道防线）：
- 将每个小区的表型值绘制在田间布局图上（行 × 列 grid）。
- 用颜色编码表型值——如果看到清晰的块状或条带状模式，说明存在空间变异。
- 这是植物育种家的标准操作。不画图就做分析是危险的习惯。

**统计检测**：
- **Moran's I**：空间自相关指数。如果 Moran's I 显著 > 0，说明相邻小区的残差更相似（空间自相关存在）。
- **Variogram**：检查半方差是否随距离增加——如果是，空间相关存在。
- **SpATS** (Spatial Analysis of field Trials with Splines)：使用 P-spline 平滑表面建模空间趋势。这是目前最先进的植物田间试验空间分析方法。

### 空间校正的模型选择

| 方法 | 适合场景 | R 包 |
|------|---------|------|
| 行列效应模型 | 规则网格，条带状变异 | lme4 / sommer |
| AR1×AR1 自回归 | 相邻小区相关性强 | ASReml / sommer |
| P-spline (SpATS) | 不规则/平滑变异 | SpATS |
| 移动平均 (moving average) | 简单/快速筛查 | 自编 / mvoutlier |

**实际建议**：如果 ASReml 可用，用 AR1×AR1。如果只有 R 免费工具，用 SpATS（对于 >100 个基因型的试验）或行列效应模型（小型试验）。

### BLUP vs BLUE

- **BLUP（随机效应）**：基因型作为随机效应 → 估计值向均值收缩 → 更适合排序和选择
- **BLUE（固定效应）**：基因型作为固定效应 → 不收缩 → 更适合比较特定基因型的绝对表现

对于育种：BLUP 通常更合适。对于品种审定或特定基因型评估：BLUE。

#### BLUP vs BLUE 深入：不同场景的实际选择

**植物育种中的 BLUP 最佳实践**：

**场景 1：育种选择（从 500 个品系中选前 50 个）→ BLUP**
- BLUP 的收缩效应（shrinkage）实际上是一个"安全机制"——极端高值在 BLUP 中会被拉向均值，减少了因为随机误差而误选差基因型的概率。
- 特别是在早期代（仅有 1-2 个环境的数据），BLUP 的收益最大。

**场景 2：品种审定（比较 3 个待审品种 vs 2 个对照品种）→ BLUE**
- 你需要对特定品种的绝对表现做出判断。
- BLUE 不收缩，给出的是该品种在该试验条件下的"真实表现"估计。
- 如果试验设计良好（多个环境、有重复），BLUE 和 BLUP 的结果应该高度相关。

**场景 3：GWAS 表型 → BLUP（推荐）或 per-environment 表型**
- 跨环境 BLUP 作为 GWAS 表型可以提高遗传力，从而提高 GWAS 功效。
- 但是：如果不同环境的处理不同（如灌溉 vs 干旱），用 per-environment 表型做 per-environment GWAS 更有意义。

**场景 4：GS 训练 → BLUP 或 deregressed BLUP**
- 简单使用 BLUP 值作为训练表型是可以的。
- Deregressed BLUP：去除收缩效应，将 BLUP 值"反收缩"回原始尺度。这在高遗传力性状中影响不大，但在低遗传力性状中可能有帮助。

## 多环境试验 (MET) 分析策略

### 第一步：检查环境间的遗传相关

在跑任何模型之前，先看每对环境间的遗传相关系数矩阵：
- r_G > 0.9 → 两个环境几乎等价——可以合并分析
- r_G = 0.6-0.8 → 环境间存在交叉互作（cross-over interaction）但大部分基因型在两个环境中排名相似
- r_G < 0.5 → 有严重的 G×E——某些基因型在两个环境中的排名完全颠倒。需要环境特异选择

### 第二步：选择 MET 分析模型

| 目标 | 推荐方法 | 输出 |
|------|---------|------|
| 估计跨环境综合育种值 | 多环境 BLUP (MET-BLUP) | 每个基因型的一个 BLUP 值 |
| 识别最优环境-基因型组合 | AMMI / GGE Biplot | 双标图，which-won-where |
| 预测未测试环境的表型 | 因子分析模型 (FA) | 更准确的环境间外推 |
| 评估基因型稳定性 | Finlay-Wilkinson 回归 / Eberhart-Russell | 稳定性参数 |
| 同时考虑遗传相关和空间变异 | 多环境 + 空间模型 | ASReml 多变量 |

### AMMI 和 GGE 的双标图（biplot）是植物育种家的标配工具：
- "Which-won-where"：告诉你每个环境中哪个品种最好
- "Mean vs stability"：告诉你哪些品种既高产又稳定
- 如果你的目标环境明确（特定区域的特定管理）→ 选环境特异性的品种
- 如果你的目标环境不明确 → 选广泛适应的品种

### 跨年份数据：年效应 vs 遗传效应

多年的表型数据是无价的——但年效应本身通常不是你的研究对象。

**年效应通常被视为随机效应**：你想知道基因型的"一般表现"，而不是"2024 年比 2023 年好"。所以年份作为随机效应，你的兴趣在于跨年的 BLUP。

**除非**：你在研究气候适应性。如果你想回答"哪些基因型在好年份和差年份都表现一样好"——年份变成了一个需要建模的环境协变量（降雨量、温度）。这时适合用因子分析模型（FA model）或反应规范（reaction norm）模型。

**重复力 (repeatability) vs 遗传力**：
- 单年内的 H² 是一回事。跨年的"重复力"是另一回事。
- 重复力 < 单年 H² 是正常的，因为年×基因型互作是真实存在的。
- 如果单年 H² = 0.8 但跨年重复力 = 0.3 → 你在研究一个 G×E 非常严重的性状。GS 在这种性状上的跨年预测准确性会很差。

### 单株 vs 小区表型：数据粒度决策

**小区表型（plot-level phenotype）**：
- 一个小区（数到数十株）的平均值。
- 优点：取样误差小，代表基因型的"一般表现"。
- 缺点：忽略了株间的遗传变异（如果你是 F2 群体，每株基因型不同！）。
- 大多数育种产量试验用的是小区表型。

**单株表型（single-plant phenotype）**：
- 优点：适合分离群体（F2, F3）、精密遗传分析（如单个 QTL 的效应估计）。
- 缺点：单株的测量误差极大——单株产量 30±10g，小区平均值 32±2g。单株的 H² 比小区的 H² 低得多。
- 对于 GS 训练：单株表型通常不可用——要么测量误差太大，要么单株不能代表育种中的"品种"（品种是纯合或均一的群体）。

**实际建议**：
- QTL mapping → 单株表型（每株的基因型不同，需要对应）
- GWAS（纯合自交系）→ 小区表型
- GS 训练 → 小区表型
- 高通量表型平台 → 单株表型 + 空间校正 + 重复（技术重复而非生物学重复）

### 多环境分析 (AMMI/GGE)

AMMI 和 GGE 的双标图（biplot）是植物育种家的标配工具：
- "Which-won-where"：告诉你每个环境中哪个品种最好
- "Mean vs stability"：告诉你哪些品种既高产又稳定
- 如果你的目标环境明确（特定区域的特定管理）→ 选环境特异性的品种
- 如果你的目标环境不明确 → 选广泛适应的品种
