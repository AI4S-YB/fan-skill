# 表型数据分析引导

## 这个分析解决什么问题

用户有表型试验数据，需要做统计分析：估计遗传力、BLUP/BLUE、基本统计。典型表述：
- "这些表型数据怎么分析"
- "算一下遗传力和 BLUP 值"
- "不同基因型的差异显著吗"

## 你需要问什么

1. **试验设计**: RCBD / Alpha-Lattice / Augmented？什么设计决定什么模型
2. **性状类型**: 产量/品质/抗性？数值型还是分类？
3. **环境**: 单环境还是多环境？几个重复？
4. **数据质量**: 缺失值多吗？CV 多高？分布正常吗？
5. **分析目标**: 只要 BLUP？还要遗传力？还要多重比较？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **环境效应**: 植物表型受环境影响极大（同一基因型在不同环境下产量可差 2-5 倍）。需要记录气象、土壤、管理措施等环境协变量
- **性状测量标准化**: 生育期（播种到开花天数）、株高（cm）、千粒重（g）等标准性状的测量协议应遵循该作物的标准操作规程（SOP）
- **多环境数据**: 单环境表型无法估计 G×E 互作，育种价值有限。推荐 ≥ 2 个环境，≥ 3 个更好
- **纵向表型**: 高通量表型平台（无人机、轨道车）产生时间序列表型数据（如生育期内多次 NDVI 测量）。分析方法不同于传统的单时间点表型
- **数据分布**: 抗病性评级、倒伏等级等有序分类表型不满足正态假设。需要用 GLM 或进行数据转换

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"表型数据(多环境、多重复)、试验设计信息、环境协变量完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有单环境数据；或缺乏环境协变量"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"表型试验尚未开展"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
