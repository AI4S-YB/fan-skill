# GWAS 分析引导

## 这个分析解决什么问题

用户想找到控制某个性状的基因组区域/基因位点。典型表述：
- "找到控制粒重的基因/QTL"
- "这个性状和哪些 SNP 关联"
- "我做了基因分型，想看看有什么显著位点"

## 你需要问什么

每次只问一个问题。根据用户已说的跳过已知信息。

1. **基因型数据**: 有什么平台？芯片 / GBS / 重测序？大概多少个标记？
   - <500 SNP → 做不了 GWAS，建议候选基因关联分析
2. **群体信息**: 多少样本？什么物种？自交还是异交？
3. **表型数据**: 测了什么性状？数值还是分类？几个环境？
4. **分析目标**: 找新位点？验证已知 QTL？做 GS 预测？

## 什么时候推荐这个分析

- 有基因型+表型数据，≥100 样本，≥500 标记
- 用户想找"控制性状的基因组区域"
- 不是双亲群体（双亲群体 → QTL mapping）
- 不是主要做预测（预测 → genomic selection）

## 植物特定考量

- **群体结构**: 植物群体通常有强的群体结构（indica/japonica、冬/春性等），不校正会导致大量假阳性。推荐用 PCA + kinship 的混合线性模型
- **多倍体**: 异源多倍体（小麦、棉花、油菜）需分亚基因组分析；同源多倍体需剂量感知的 GWAS 方法
- **LD 衰减**: 自交物种（水稻、大豆）的 LD 衰减慢（可达数百 Kb），候选区间很大；异交物种（玉米）LD 衰减快，定位精度高

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"基因型数据(VCF格式，>500 SNP)、表型数据(多环境)、群体结构信息完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有基因型数据，缺乏表型；或标记密度过低(<500 SNP)"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"群体尚未构建"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
