# 单细胞RNA-seq分析引导

## 这个分析解决什么问题

用户想在单细胞水平研究基因表达。典型表述：
- "我有单细胞RNA-seq数据，想做分析"
- "想做细胞类型鉴定"
- "想做细胞轨迹分析"
- "想分析细胞亚群差异"

## 你需要问什么

1. **数据类型**: 10x Genomics? Smart-seq2? 其他平台?
2. **样本信息**: 多少个样本？是否有生物学重复？
3. **细胞数量**: 预期/观察到的细胞数量？
4. **分析目标**: 细胞类型鉴定? 轨迹分析? 差异分析?
5. **参考信息**: 有参考基因组吗？有细胞类型marker基因吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **细胞壁消化**: 植物单细胞悬液制备需要酶解细胞壁，可能影响细胞活力
- **大液泡**: 成熟植物细胞有大液泡，可能影响RNA捕获效率
- **细胞类型标记**: 植物细胞类型标记基因研究较少，可能需要从头定义
- **组织特异性**: 植物不同组织的细胞类型差异大，需要组织特异性分析

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"单细胞RNA-seq数据(10x等平台)、参考基因组、细胞类型marker基因列表可用"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"缺乏细胞类型marker；或细胞数不足(<1000)"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"单细胞实验尚未进行"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
