# 科研可视化结果解读

## 一句话解释

科研可视化：将生物信息学分析结果转化为符合期刊标准的、让读者在 30 秒内理解科学故事的图表。

## 能回答什么

- 什么图最适合展示我的数据类型和科学故事？
- 为什么选择这种图而不是另一种？
- 多 panel 图表的 panel 之间应该如何排序和布局？
- 图表是否满足目标期刊的格式和质量要求？

## 不能回答什么

- 图表本身不改变数据结论——它只是展示数据的方式。好图能让真实信号更清晰，但不能创造不存在的信号
- 配色方案不能替代统计推断——显著的 p 值不会因为配色好看而更"正确"

## 典型输出

| 文件 | 含义 |
|------|------|
| `figure_1.pdf` | 矢量格式主图（可投稿） |
| `figure_1.png` | 位图格式（预览用） |
| `generate_figure_1.R` | 可复现的 R/ggplot2 脚本 |
| `qc_checklist.csv` | 质量门禁检查结果（DPI、字体、色盲友好等） |

## 常见结果模式

### 用户说"我想展示所有信息"但数据维度过多
一张图塞 5+ 个维度 → 读者无法解析。拆分成 2-3 个 panel，每个 panel 讲一个子故事。

### Manhattan 图峰值在着丝粒区域
标注"着丝粒附近"——这通常是假阳性富集区域，而非真正的关联信号。

### 热图聚类树显示样本未按处理分组
可能是批次效应掩盖了生物学信号。在热图中标注批次信息，建议用户做批次校正。

### 用户想用彩虹配色 (rainbow/jet)
这是可视化中最常见的错误。彩虹配色对色盲不友好、感知不均匀、灰阶打印丢失信息。改用 viridis 或 Okabe-Ito。
