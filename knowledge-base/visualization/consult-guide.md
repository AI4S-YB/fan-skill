# 科研可视化分析引导

## 这个分析解决什么问题

用户想把植物生物信息学分析结果转化为发表级图表。典型表述：
- "帮我画一个曼哈顿图"
- "我想展示差异表达基因"
- "怎么把 GWAS 结果可视化"
- "帮我做一组图讲清楚这个基因的功能"

## 你需要问什么

1. **数据故事**: "如果读者只看这张图 10 秒，你最想让他们记住什么？"——这决定了图的形式
2. **受众**: 分子生物学家（要看基因名和 p 值）/ 育种家（要看品种和产量）/ 期刊审稿人（要完整精确）？
3. **数据类型**: 位置数据？数值分布？矩阵关系？不同数据对应不同图类型
4. **目标期刊**: Plant Journal / Nature Plants / TAG / New Phytologist？不同期刊有不同的格式和宽度要求
5. **是单图还是多 panel 组合**？多 panel 需要规划叙事顺序（如 A 表型 → B 遗传 → C 分子 → D 验证）

## 三条对话路径

1. **用户明确要某类图** → 确认数据和细节 → 选模板 → 生成 → 质量门禁 → 交付
2. **用户描述多张图需求** → 确认故事线 → 拆解 panel → 排序布局 → 逐个确认 → 执行
3. **用户不确定用什么图** → 读数据 → 理解故事 → 推荐 1-3 种形式 → 解释理由 → 用户选择 → 执行

## 植物特定考量

- 多倍体物种的 Manhattan 图需要标注亚基因组
- 多环境 GGE biplot 是育种家的"母语"
- 时间序列热图需要标注 Zeitgeber time (ZT) 和光周期
- 基因组浏览器截图需要展示基因结构和 RNA-seq 覆盖轨道
