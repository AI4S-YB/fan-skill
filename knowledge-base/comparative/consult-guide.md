# 比较基因组学分析引导

## 这个分析解决什么问题

用户想通过比较多个基因组来回答进化问题。典型表述：
- "我的物种经历过几次全基因组加倍"
- "某个基因家族为什么这么大"
- "哪些基因受到正选择"
- "我跟水稻/拟南芥的共线性怎么样"

## 你需要问什么

1. **有哪些基因组**: 几个物种？几个品种？每个都有基因注释吗？
2. **比较尺度**: 种内还是种间？近缘种还是远缘种？
3. **基因组质量**: 有 CDS 序列吗？注释是染色体水平吗？
4. **分析目标**: 共线性分析？Ks 分析？基因家族鉴定？dN/dS？
5. **物种倍性**: 二倍体还是多倍体？多倍体需分亚基因组分析

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **全基因组加倍 (WGD)**: 植物进化史上有多次 WGD 事件（如双子叶植物共有的 gamma 加倍、禾本科的 rho 加倍）。不识别这些事件 → 误将 paralog 判为 ortholog
- **多倍体**: 异源多倍体（小麦、棉花、油菜）的亚基因组经历了不同的进化轨迹。需要分亚基因组分析
- **基因家族扩张**: 植物抗病基因 (NBS-LRR)、转录因子家族在植物中高度扩张。家族鉴定时需要适当调整聚类阈值
- **GFF3 格式**: 许多植物基因组的 GFF3 格式不标准（mRNA/transcript 层级缺失），分析前需要用 AGAT 或 gffread 验证和修复

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"多个基因组组装和注释、外群基因组、系统发育背景信息完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"有基因组组装但缺乏基因注释；或缺乏外群"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"目标基因组尚未测序"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
