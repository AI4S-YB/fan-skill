# 基因组组装分析引导

## 这个分析解决什么问题

用户想从测序数据组装一个完整的基因组。典型表述：
- "我有 PacBio HiFi 数据，想组装基因组"
- "我有 ONT 数据，怎么组装"
- "组装的基因组质量怎么评估"

## 你需要问什么

1. **测序数据**: PacBio HiFi / ONT / 混合？覆盖度多少？
2. **基因组特征**: 估计大小？杂合度？倍性？(先做 GenomeScope)
3. **辅助数据**: 有 Hi-C 做染色体挂载吗？有 Illumina 短读长吗？
4. **多倍体**: 同源/异源多倍体？需要分亚基因组组装吗？
5. **组装目标**: contig 水平 / scaffold 水平 / 染色体水平？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **基因组大小**: 植物基因组大小跨越三个数量级（Genlisea ~60Mb 到 Paris japonica ~150Gb）。大基因组（>10Gb）的组装计算资源需求呈非线性增长
- **多倍体**: 多倍体植物的基因组组装需要考虑 phasing。hifiasm 支持 --n-hap 参数。同源多倍体（autopolyploid）的 haplotype phasing 极其困难
- **杂合度**: 高杂合度物种（如果树、林木）组装时会产生 inflated genome size（两个单倍型被分别组装）。需要 purging（purge_dups）或专用的 trio-binning 方法
- **细胞器基因组**: 组装前应识别和分离叶绿体/线粒体基因组。它们的高拷贝数会影响主基因组的覆盖度估计

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"PacBio HiFi数据(30x)、Hi-C数据(50x)、Illumina短读长数据完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有PacBio CCS数据，缺乏Hi-C用于染色体挂载"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"测序数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
