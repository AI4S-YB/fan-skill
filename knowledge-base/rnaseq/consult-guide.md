# RNA-seq 差异表达分析引导

## 这个分析解决什么问题

用户想找处理条件下哪些基因的表达发生了显著变化。典型表述：
- "干旱处理后哪些基因变了"
- "处理和对照的差异表达基因"
- "我想看这个突变体在转录组层面有什么变化"

## 你需要问什么

1. **实验设计**: 处理 vs 对照？几组？每组几个生物学重复？
   - 无对照 → 不能做差异表达
   - 1 重复 → 不能做统计推断
   - 2 重复 → 可用但功效低
2. **物种和参考基因组**: 有高质量参考吗？非模式物种？
3. **分析目标**: 只找 DEGs？还要做富集？WGCNA？
4. **后续**: 想做图表发表吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **无参考基因组**: 很多植物物种没有参考基因组。可以做 de novo 转录组组装 (Trinity) + 功能注释，但分析的完整性和可靠性低于有参考的分析
- **叶绿体和线粒体 reads**: 植物总 RNA 中包含大量细胞器 rRNA（占比可达 20-50%）。rRNA depletion 比 polyA selection 更适合植物（polyA 会丢失非编码 RNA 和部分 mRNA）
- **多倍体**: 异源多倍体（小麦、棉花、油菜）的 reads 可能比对到多个亚基因组。需要合并参考或分亚基因组比对策略
- **组织采样时间**: 植物基因表达有昼夜节律。如果对照和处理在不同 Zeitgeber time 取样，差异可能是节律效应而非处理效应

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"RNA-seq数据(处理组和对照组各3重复)、参考基因组、基因注释完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有处理组数据，缺乏对照组；或无生物学重复"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"测序数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
