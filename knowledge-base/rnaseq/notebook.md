# 植物 RNA-seq 差异表达分析：分析笔记

## 在开始之前

植物 RNA-seq 的核心挑战不是分析本身，而是实验设计。先问自己：

### 你有什么样的数据？

查看 data_profile.json 和 count matrix：
- **基因数**：植物基因组跨度大，草莓 ~25K, 水稻 ~35K, 小麦 ~100K（含同源基因）
- **样本数**：≥3 重复 → 正规分析；2 重复 → 可用但功效低；1 个 or 无重复 → 不能统计推断
- **同源基因**：小麦/棉花/油菜中同一基因在多个亚基因组各有拷贝，需要决定是合并还是分开分析

### 这个实验设计是什么？

- **简单两组比较**（处理 vs 对照）：DESeq2 标准流程
- **多组比较**（多个处理梯度/时间点）：DESeq2 + 所有 pairwise 对比
- **连续协变量**（如温度梯度、发育阶段）：limma-voom 更灵活
- **多因子设计**（基因型 × 处理 × 时间）：DESeq2 的 ~ genotype + treatment + genotype:treatment

### 你期望看到什么？

- **处理起始效应**：对初级胁迫响应 → 预期 5-15% 的 DEGs
- **全局转录重编程**（如某个处理 vs 对照的 >30% DEG）→ 处理强度很高
- **无 DEGs** → 实验可能确实无响应（正常结果），或功效太低（重复不够）

## 实验设计：植物 RNA-seq 最重要的决策

没有任何分析工具能弥补糟糕的实验设计。以下是植物特有的设计陷阱和决策。

### 生物学重复 vs 技术重复

**生物学重复 = 独立种植的、独立处理的不同植株（或独立田块中采集的不同植株）。**

- 技术重复（同一份 RNA 跑两次测序）：告诉你测序平台的噪音水平。仅在评估实验室 protocol 稳定性时有用。
- 生物学重复：告诉你你关心的生物效应相对于植株间自然变异的显著性。这才是你需要的。

植物实验中，"生物学重复"的粒度是个关键决策：
- 同一温室中同一托盘的不同植株 = 生物学重复（但共享微环境）
- 同一田块不同位置的植株 = 更好的生物学重复（更少微环境共享）
- 不同温室/不同田块 = 最强的生物学重复（但变异更大，功效更低）

**实践经验**：如果预算有限，2 个田块各 3 个生物学重复 > 1 个田块 6 个生物学重复。前者可以估计田块间变异。

### 组织/器官特异性与取样策略

植物的同一基因在不同组织中的表达可以差异数千倍。取样策略决定了你能检测到什么。

**取样粒度决策**：
- **整株取样**：仅适用于幼苗或极小植物（拟南芥幼苗）。任何稍大的植物，整株取样会把根、茎、叶的表达信号全混在一起——对于大多数生物学问题来说，这种混合没有意义。
- **器官级取样**（根 vs 茎 vs 叶 vs 花）：最常用的策略。但"叶"不是均匀的。
- **精细取样**（根尖 vs 根成熟区，幼叶 vs 老叶，雌蕊 vs 雄蕊）：当你有明确的生物学假设时采用。

**叶片取样的隐藏陷阱**：
- 同一植株上第 3 片叶和第 8 片叶的转录组可以非常不同（叶片年龄效应）
- 同一片叶的叶尖、叶中、叶基表达谱不同（发育梯度）
- 朝阳面和背阴面的叶片表达谱不同（光响应）
- **建议**：如果不做叶龄/位置的专门分析，至少固定取样位置——"所有样本采集第 4 片完全展开叶的中部 2 cm"。

**根取样的隐藏陷阱**：
- 根尖（分生区）、伸长区、成熟区、侧根原基的转录组完全不一样
- 水培 vs 土培的根系转录组差异巨大——生长介质本身就是一种处理
- 洗根过程（从土壤中取出根并洗净）是一个胁迫处理——转录组在几分钟内就会变化
- **建议**：使用水培或琼脂培养进行根取样，减少"取样本身是胁迫"的问题。如果必须从土壤取样，整个取样过程控制在 2 分钟内。

**发育阶段的重要性**：
植物的发育阶段是转录组最大的变异来源之一。水稻的三叶期 vs 分蘖期 vs 抽穗期——差异基因的数量可能超过任何胁迫处理。
- **务必记录发育阶段**：不仅仅是"第 4 周"，而是"Zadoks 25 / BBCH 30 / 分蘖盛期"等标准化阶段。
- 如果不同基因型的发育速度不同（如早熟 vs 晚熟品种），同一日历时间 ≠ 同一发育阶段。这种情况最好用发育阶段而非日历时间进行分组。

### 植物特有的批次效应

植物实验中的批次效应比医学研究更难控制，因为环境因素难以标准化。

**温室 vs 大田**：
- 温室中同一时间、同一温室采集的样本更相似（共享温度/湿度/光照日变化）——这是一个批次。
- 如果你有 24 个样本在温室中分 3 批种植、3 批取样，你必须记录批次并在模型中（~ batch）校正。
- 大田试验：田块位置、土壤类型（即使在同一块田里也有空间变异）、光照（边缘行 vs 内部行）都是隐藏的批次效应。

**取样时间（时间生物学效应）**：
- 植物基因表达有强烈的昼夜节律（circadian rhythm）。大约 30% 的拟南芥基因有昼夜振荡表达模式。
- 如果你上午 9 点取对照，下午 2 点取处理——你检测到的 DEGs 中有相当一部分可能只是昼夜差异。
- **强制要求**：对照和处理的所有重复应在同一时间段（≤2 小时窗口内）取样。或者在实验设计中交叉平衡——上午取一半对照+一半处理，下午取另一半。
- 如果你在做时间序列实验且无法在固定时间取样：在模型中加取样时间作为协变量。

**土壤水分与微环境**：
- 同一温室内的不同位置，光照强度可以差 2-3 倍（靠窗 vs 靠墙）。
- 同一温室内不同托盘的灌溉可能有细微差异。
- **最简单的保护措施**：随机化摆放。不要把所有的对照放在左边、所有处理放在右边。
- **更好的保护措施**：记录每个植株位置的微环境数据（PAR 传感器、土壤湿度计），如果有就作为协变量。

**RNA 降解**：
- 植物组织（特别是富含多糖多酚的组织——如成熟的叶片、果实、木质化茎）RNA 降解很快。
- 液氮速冻是金标准。RNAlater 也可以但可能影响某些下游应用。
- 样品从植株上切下来到进入液氮的时间应该 < 30 秒。

## 选择你的 DE 工具

### DESeq2 — 最稳健，默认选择

使用负二项分布 + 经验贝叶斯收缩。

**适合**：所有有重复的实验设计
**不适合**：仅在没有重复时才需要替代方案

植物 RNA-seq 中 DESeq2 几乎总是第一选择。它处理离散度估计好，对 outlier 稳健，shrinkage 降低了低表达基因的假阳性。

### edgeR — 低重复/无重复时考虑

**适合**：2 个重复 → edgeR 的 quasi-likelihood 帧框略有优势
**不适合**：≥3 重复时没有相对 DESeq2 的显著优势

植物实验中如果因为资源限制只有 2 重复，edgeR 值得考虑。但报告时诚实说明功效限制。

#### 什么时候 2 个重复可以接受——什么时候绝对不行

这是植物 RNA-seq 中最常见的预算与统计功效之间的博弈。

**2 个重复可以接受的场景**：
- 验证性实验。你已经从独立实验（如 qRT-PCR 或之前的 RNA-seq）中知道了预期效应，只想确认这批样本中也存在同样的模式。
- 极强的处理效应。如果预期处理导致的 DEG 数量 > 30%（如热激、强病原体感染），2 个重复足够检测到多数大效应基因。
- 时间序列的初筛。你想从 10 个时间点中筛选出 2-3 个"最有趣"的时间点进行深入分析（深入分析时补充重复）。
- 当 n=2 时使用 edgeR + quasi-likelihood F-test，并设置更严格的阈值（如 FDR < 0.01 而非 < 0.05，或同时要求 |log2FC| > 2）。

**2 个重复绝对不能接受的场景**：
- 探索性/发现性实验。你完全不知道会看到什么，想从无假设的数据中找模式。2 个重复的假阳性和假阴性率都不可接受。
- 微妙的处理效应。预期 DEG < 5%（如低浓度农药、温和干旱）。2 个重复完全没有功效。
- 高生物学变异性的性状。多倍体物种（小麦、棉花）、杂合群体（F2 群体、自然群体）。2 个重复的离散度估计完全不可靠。
- 准备发表的结果。大多数植物期刊审稿人会直接拒掉 n=2 的 RNA-seq 实验。最低可接受标准是 n≥3。

**诚实的最低要求**：
- n=3 是"可发表"的最低标准。
- n=4-6 是"良好实践"标准。
- n>6 很少需要——生物学重复再多，增益递减。

### limma-voom — 大样本/复杂设计

**适合**：30+ 样本，或包含连续协变量的模型
**不适合**：简单两组比较（DESeq2 更好）

对植物超大规模转录组数据（>50 样本来自多个组织/处理/发育阶段），limma-voom 效率远高于 DESeq2。

## 多倍体中的同源表达偏向

这是植物转录组学的独特领域——人类转录组学中没有对应的概念。

### 什么是同源表达偏向

在异源多倍体（小麦、棉花、油菜）中，同一个基因在每个亚基因组中都有拷贝（同源基因, homeolog）。这三个拷贝通常具有 >95% 的编码区序列相似性，但它们的表达水平可能完全不同。

例如，小麦中一个转录因子可能在 A 亚基因组上高表达，在 B 亚基因组上中等表达，在 D 亚基因组上几乎沉默。这种现象就是**同源表达偏向 (homeolog expression bias)**。

### 为什么重要

- **功能分化**：不同亚基因组的拷贝可能承担了不同的功能（亚功能化, subfunctionalization）。合并所有亚基因组拷贝的表达量会掩盖这种生物学上重要的分化。
- **育种利用**：如果你想通过 knock-out 或 RNAi 来调控这个基因，选择沉默哪个拷贝会产生不同的表型效果。
- **多倍体优势**：部分同源表达偏向被认为是多倍体适应性的分子基础——不同的亚基因组拷贝在不同环境条件下被调动。

### 分析策略

**分开分析（推荐）**：
- 使用亚基因组特异性参考转录组（如小麦 IWGSC 的各亚基因组注释）
- 将 reads 分别比对到各亚基因组，或用竞争性比对策略（如 HomeoRoq, PolyCat）
- 每个同源基因 triplet 作为三个独立的"基因"进入 DESeq2
- 优点：可以看到亚基因组特异性的差异表达模式
- 缺点：需要解决多比对问题

**合并分析**：
- 将三个同源拷贝的 counts 合并为一个"基因"的总表达量
- 优点：简单，不需要解决多比对问题
- 缺点：丢失了所有亚基因组特异信息

**如果不确定，默认分开**，但在报告中说明多比对的处理方法和不确定性。

### 多比对问题的工具选择

- **Salmon/Alevin** + 亚基因组特异性转录组索引 → 可以处理多比对
- **STAR** + `--outFilterMultimapNmax` → 限制每个 read 的最大比对位置数
- **Subread/featureCounts** + `-M` → 对多比对 read 进行计数分配
- **HomeoRoq** (小麦专用) → 直接估计同源基因表达量的比率

## 非模式物种：转录组策略

如果用户的研究物种不是拟南芥/水稻/玉米/大豆（没有高质量的参考基因组和注释）：

### 有参考基因组但没有注释

→ 做 genome-guided transcriptome assembly (StringTie + Ballgown)
→ 然后用 TransDecoder 预测蛋白质编码区
→ 跨物种功能注释 (BLASTP 到 UniProt/Swiss-Prot 植物蛋白)

### 没有参考基因组

→ **de novo transcriptome assembly** (Trinity, rnaSPAdes)
→ 然后用 TransDecoder + BLASTP 做功能注释
→ **关键约束**：de novo 组装无法区分同源基因和不同的剪接异构体——它们会被组装成不同的 transcript cluster。在没有参考基因组的多倍体中，这个问题被放大。

### 有近缘物种的参考基因组

→ 可以尝试 cross-species mapping（将 reads 比对到近缘物种的基因组）。但有一个距离限制：
- **属内**（如野生稻 → 栽培稻）：通常可行，mapping rate > 70%
- **族内**（如小麦 → 大麦）：勉强可行，mapping rate 50-70%，许多基因区域有序列分歧
- **科内**（如拟南芥 → 油菜）：不建议，mapping rate < 50%，差异表达分析不可靠
→ 如果在科一级——做 de novo 组装

### 非模式物种 DEG 的功能解释

没有 GO 注释怎么办：
1. 用 Mercator4 (https://www.plabipd.de/portal/mercator4) 做在线功能注释。Mercator 支持几乎所有有参考转录组的植物物种。
2. 用 eggNOG-mapper 做跨物种同源注释。
3. 用 MapMan 分类系统（通过 Mercator 得到的 MapMan bin）。

## 功能富集分析

### 植物的最优选择

1. **Plant Reactome**（水稻/玉米/拟南芥）→ 质量最高的通路注释
2. **AgriGO v2** → 覆盖更多农业物种
3. **MapMan/Mercator** → 非模式植物，在线注释后导出
4. **KEGG Plant** → 备选

### 常见误区

- 不要只看 GO 的 P 值——GO terms 之间有很强的层级依赖（parent-child 关系）
- DEGs 太少（<30）→ 任何富集分析都没有足够的统计功效
- 背景基因集应该是"所有表达的基因"，不是"基因组所有基因"

## WGCNA 共表达分析

### 什么时候值得做？

- 样本 ≥ 15 → WGCNA 开始有意义
- 有连续的性状数据 → 可以做模块-性状关联
- 有多个组织/发育阶段/处理 → 共表达模式往往比 DEG 更有生物学意义

### 植物 WGCNA 的特殊考虑

- 植物发育梯度（叶片年龄、果实成熟）→ 共表达模式往往非常清晰
- 不同组织（根/茎/叶/花）→ 组织特异性模块占主导
- 如果 90% 的基因都聚到一个"超级模块"→ minModuleSize 可能太小或 power 选择有问题

## 常见陷阱

### 同源基因的困扰

小麦的 TraesCS1A01G0001、TraesCS1B01G0001、TraesCS1D01G0001 → 三个亚基因组的同源拷贝。分开分析还是合并？

- **分开**：可以看到亚基因组特异性表达
- **合并（sum counts）**：只有一个"基因"进入 DEG 列表
- 如果不确定，默认分开

### 植物特有的"零膨胀"

某些组织的 RNA-seq 中 >50% 的基因 count 为 0 → 完全正常（基因不在此组织中表达）。DESeq2 的独立过滤会自动处理。

### 表型数据缺失

如果你的 DEG 列表很丰富但无法解释 → 检查是否有表型/metadata 可以帮助解释（组织类型、发育阶段、胁迫处理、基因型差异）。

### 昼夜节律对表达模式的影响

植物中有约 30% 的基因表达受昼夜节律调控。如果你的实验设计没有控制取样时间，昼夜节律可能是你 DEG 列表中的"隐藏处理效应"。

**识别昼夜效应**：
- 检查 DEG 列表中是否有已知的昼夜节律基因（如 CCA1, LHY, TOC1, GI, PRR 家族成员）。如果这些基因在 DEG 列表中——你的取样时间可能不一致。
- 检查连续的时间序列数据——如果基因表达呈现 ~24 小时周期的正弦波模式，那是昼夜节律，不是你的处理效应。

**如何报告**：
- 如果存在昼夜混淆：在论文中诚实说明"由于对照和处理在不同时间取样，部分 DEGs 可能反映昼夜节律差异而非处理效应。我们排除了已知的核心节律基因（列出基因）以降低混淆。"
