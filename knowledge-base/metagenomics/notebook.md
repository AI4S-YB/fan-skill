# 植物宏基因组分析：分析笔记

## 在开始之前

植物宏基因组学的核心挑战不是测序深度，而是样品本身的复杂性。先问自己：

### 你取样的部位是什么？

- **根际 (Rhizosphere)**：根周围土壤 → 微生物最丰富，每克土壤 10^9-10^10 个细胞
- **根内 (Endosphere/Root)**：植物根内部 → 微生物量少，植物DNA污染严重（>90% 可能是植物reads）
- **叶际 (Phyllosphere)**：叶片表面 → 微生物量极少，外源污染影响大
- **土壤 (Bulk soil)**：非根际土壤作为对照 → 微生物丰富度最高
- **种子内生 (Seed endophyte)**：种子内部 → 垂直传播微生物，生物量极低

### 样品复杂性如何？

- **根际/土壤** → 高复杂度（数千种微生物），需大量测序数据（>10Gbp/sample）
- **根内/叶际** → 低复杂度但植物DNA干扰严重
- **关键参数**：微生物与植物的DNA比例 → 决定有效数据量

### 你的科学问题是什么？

- **群落组成**：谁在那里？→ 16S扩增子可能已足够（另见 bio-plant-amplicon）
- **MAGs 恢复**：能否重建微生物基因组？→ 需要宏基因组深度测序
- **功能潜力**：微生物能做什么？→ 宏基因组功能注释
- **植物-微生物互作**：哪些微生物基因与植物表型相关？

## 分析策略选择：组装优先 vs 基于读段的分类

植物宏基因组的第一个关键决策：需要通过组装恢复基因组，还是仅需分类信息？

### 组装优先路线 (Assembly-first)

工具链：MEGAHIT/metaSPAdes → MetaBAT2 → CheckM → GTDB-Tk → 功能注释

**选择条件**：
- 需要重建微生物基因组（MAGs）用于功能分析
- 研究目标是未知功能基因的发现（novel gene discovery）
- 测序深度足够（通常 > 10 Gb per sample for soil）
- 主机 DNA 污染可控（根际/土壤样本）

**优点**：获得基因组级别的信息，可以发现新基因、新代谢通路、新物种
**缺点**：计算量大，低丰度微生物无法组装，操作链较长

### 基于读段的分类路线 (Read-based)

工具链：Kraken2/Bracken → MetaPhlAn4 → HUMAnN → 功能通路丰度

**选择条件**：
- 只需知道"谁在那里"和"能做什么"的粗略信息
- 计算资源有限或需要快速结果
- 样本中植物 DNA 比例极高且无法有效去除
- 作为组装路线的补充/验证

**优点**：快速（几小时 vs 几天），低丰度微生物也能分类
**缺点**：依赖参考数据库（大量植物微生物未被培养/未被测序），无法发现新基因

### 决策指南

| 场景 | 推荐策略 | 理由 |
|------|---------|------|
| 根际土壤，追求高质量 MAGs | 组装优先（MEGAHIT + MetaBAT2 + DASTool） | 复杂度高但有足够测序量 |
| 叶际样本，植物 DNA >95% | 读段分类（Kraken2 + Bracken）优先，有余力再尝试组装 | 有效微生物数据量太少 |
| 研究根-微生物互作机制 | 组装优先 + 读段分类双重验证 | 需要基因组级别功能信息 |
| 快速群落筛查 | 读段分类（MetaPhlAn4 快速模式） | 速度优先于深度 |
| 有长读长 (ONT/PacBio) | 混合组装（metaSPAdes hybrid + Flye meta） | 显著提升 MAG 连续性 |

## 选择你的组装工具

### MEGAHIT — 标准选择

基于 succinct de Bruijn graph，内存效率高。

**适合**：中等到大数据量（>5Gbp/样本），复杂群落
**不适合**：极小的数据集（<1Gbp）→ metaSPAdes 可能更敏感

MEGAHIT 是植物宏基因组最常用的组装器。对根际和土壤样本表现优异。

### metaSPAdes — 小数据更敏感

**适合**：小数据量（<5Gbp），追求高灵敏度
**不适合**：大数据量 → 内存消耗大，速度慢

对低丰度微生物更敏感，但对植物DNA污染也敏感。

### 混合组装 (Hybrid)

如果同时有短读长（Illumina）和长读长（ONT/PacBio）数据，混合组装可以显著提升 MAG 的连续性。

**适合**：需要闭环基因组的高质量研究
**不适合**：仅有短读长数据

## 宿主DNA去除

### 植物宏基因组的核心痛点

根内和根际样本中，植物 DNA 可以占 80-99% 的 reads。不先去宿主：
- 组装被植物基因组片段淹没
- 有用的微生物 contigs 极低丰度
- MAG 恢复率接近零

### 为什么去除细胞器序列如此关键

植物基因组中的叶绿体（~120-160kb，每个细胞数百个拷贝）和线粒体（~200-600kb）在宏基因组数据中丰度极高。一个典型的根际样本中：
- 叶绿体 reads 占植物 reads 的 30-60%
- 线粒体 reads 占植物 reads 的 5-15%
- 核基因组 reads 只占剩余的

这意味着：即使你"成功去除了宿主"，如果只用了核基因组参考，叶绿体和线粒体 reads 仍会被组装器误认为细菌基因组（叶绿体与蓝细菌基因组高度相似！）。

### 双重清除策略

**策略 A：比对去宿主**
1. bowtie2/bwa 比对到植物参考基因组（**必须包含核基因组 + 叶绿体 + 线粒体**）
2. 去除匹配的 reads（严格模式推荐 --very-sensitive）
3. 确认去除效率：抽样比对，计算剩余 reads 的植物:微生物比例

**策略 B：分类后过滤**
1. Kraken2 使用包含植物基因组的完整数据库分类 reads
2. 滤除分类为 Viridiplantae（绿色植物）的 reads
3. 确认去除效率：同样抽样检测

**双重策略（强烈推荐用于根内和叶际样本）**：
1. 先做比对去宿主（移除大部分植物 reads）
2. 再用 Kraken2 对剩余 reads 分类
3. 如果仍有 > 30% 的 reads 分类为植物 → 增加比对严格度或使用植物 + 微生物联合数据库

### 常见误区

只用一个不完整的植物参考基因组 → 大部分植物 reads 没被过滤掉。特别是：
- 如果参考基因组不包含线粒体和叶绿体基因组 → 这些 reads 会被保留
- 不同品种/生态型的基因组有差异，某些区域不可比对 → 参考基因组越近缘越好

## 分箱策略

### MetaBAT2 — 默认首选

基于四核苷酸频率和丰度协方差，自动分箱。

**适合**：中等复杂度群落（100-500 个基因组）
**参数建议**：`--minContig 1500` 用于植物相关样本（排除植物 contig 碎片）

### 多工具共识分箱

荟萃分析表明：单个分箱工具恢复的 MAGs 有限。推荐：

```
MetaBAT2 + MaxBin2 + CONCOCT → DASTool → 共识 MAGs
```

DASTool 整合三个工具的结果，选择最优分箱方案。通常比单工具多恢复 20-30% 的高质量 MAGs。

### 植物特异性分箱建议

- 根际样本：多种分箱工具都用，多样性高
- 根内样本：先严格去宿主（比对+分类），再分箱
- 如果 MAG 数量异常少（<5）：检查宿主去除效率

### 复杂植物群落的特殊分箱挑战

植物相关微生物群落的复杂度带来独特的分箱难题：

1. **菌株水平的异质性**：同一种的多个菌株共存 → 四核苷酸频率非常接近 → 分箱工具难以区分 → 导致嵌合 MAGs 或碎片化 MAGs
   - **缓解**：使用 DESMAN 进行菌株级分析（strain-level deconvolution）

2. **微真核生物 (Microeukaryotes) 干扰**：真菌、卵菌（oomycetes）等真核生物 contigs 与细菌 contigs 在 k-mer 特征上差异大 → 可能污染细菌 MAGs
   - **缓解**：分箱前用 EukRep 去除真核生物 contigs

3. **根内低生物量挑战**：根内样本可能每个样品只有 5-15 个 MAGs → 标准分箱工具的聚类算法在低样本量下表现不稳定
   - **缓解**：将多个样本的 contigs 合并后分箱（co-assembly binning）

## MAG 质量评估

### CheckM 标准

| 质量等级 | 完整性 | 污染 | 备注 |
|---------|--------|------|------|
| 高质量 | >90% | <5% | 含 rRNA 和 tRNA 基因 |
| 中等质量 | >50% | <10% | 部分基因组，仍可用于分析 |
| 低质量 | <50% | - | 只能用于存在/缺失分析 |

### 植物 MAG 的 CheckM 解读注意事项

- **CheckM 完整性可能被高估**：植物微生物组中有大量未培养的谱系 → 其单拷贝标记基因可能不遵循标准分布 → CheckM 基于谱系的完整性估计可能不准确
- **污染可能被低估**：如果同一个 bin 中混入了近缘菌株的 contigs，单拷贝标记基因可能不显示额外的拷贝
- **推荐补充检查**：用 GUNC (Genome UNClutter) 检查分类学一致性

### GTDB-Tk 分类

基于 120 个细菌和 122 个古菌标记基因的系统发育分类。

**注意**：植物微生物组中有大量未培养的微生物 → 可能只能分类到属或科水平

## 功能注释

### 选择注释策略

- **Prokka**：细菌基因组功能注释，需要完整或近完整的 MAGs
- **DRAM**：Distilled and Refined Annotation of Metabolism → 专门为 MAGs 设计，代谢通路分析强大
- **EggNOG-mapper**：蛋白质功能注释，基于直系同源群
- **KEGG-Decoder**：KEGG 通路热图可视化，适合植物相关通路分析

### 植物相关功能

关注以下功能类别：
- 植物激素合成（IAA、细胞分裂素）
- 铁载体合成与转运（植物铁营养）
- 氮固定（nif 基因）
- 磷溶解
- ACC 脱氨酶（降低植物乙烯水平）
- 抗生素和抗真菌化合物合成

## 常见陷阱

### 植物DNA污染

最常见的失败原因。如果 reads 中 >90% 是植物来源，即使 100M reads 也只有 10M 微生物 reads。
- **检查**：用 Kraken2 快速分类 10K reads，评估植物:微生物比例
- **解决**：不满意的话，用 PMA 处理（活菌富集）或增加测序深度

### 细胞器DNA的伪装

叶绿体 DNA 和线粒体 DNA 不是"微生物"，但它们：
- 在宏基因组数据中占比可能高达 30%
- 叶绿体序列与蓝细菌高度同源 → 可能被分箱工具误判为蓝细菌 → 产生假阳性 MAGs
- **检查**：用 BLAST 将所有 MAGs 与植物细胞器基因组比对
- **解决方案**：在分箱前，将已组装的 contigs 与植物细胞器基因组比对并移除匹配的 contigs

### 叶际极低微生物量

叶际（phyllosphere）是植物宏基因组中最具挑战性的样本类型：
- 每克鲜叶仅 10^5-10^7 个微生物细胞（vs 根际的 10^9-10^10）
- >99% 的 DNA 来自植物和叶绿体
- 即使 50 Gbp 测序量，有效微生物数据可能仅 0.5 Gbp

**应对策略**：
- 测序前富集：用差速离心或 PMA 处理富集微生物组分
- 放弃 MAG 恢复期望：以读段分类为主要分析手段
- 尽量使用 16S rRNA 扩增子互补验证（宏基因组可能检测不到稀有种）

### 批量效应

不同样本之间的序列批次、DNA 提取试剂盒、PCR 循环数都会引入批量效应。
- **解决**：随机化样本顺序，在 metadata 中记录提取批次

### 假阳性 MAGs

组装错误 + 分箱错误 = 嵌合 MAGs（两个不同微生物 genome 碎片被合并）
- **检查**：MAG 中单拷贝标记基因的异质性（CheckM 可以检测）
- **解决**：用 GUNC 检查基因组嵌合

### 覆盖度不足

低丰度微生物 → 测序深度不足以组装 → 无法恢复 MAG。
- **经验规则**：>10x 覆盖度的微生物才能被组装
- **检查**：用 coverM 或 bedtools 计算 contig 覆盖度
