# 植物宏基因组分析：分析笔记

## 在开始之前

植物宏基因组学的核心挑战不是测序深度，而是样品本身的复杂性。先问自己：

### 你取样的部位是什么？

- **根际 (Rhizosphere)**：根周围土壤 → 微生物最丰富，每克土壤 10^9-10^10 个细胞
- **根内 (Endosphere/Root)**：植物根内部 → 微生物量少，植物DNA污染严重（>90% 可能是植物reads）
- **叶际 (Phyllosphere)**：叶片表面 → 微生物量极少，外源污染影响大
- **土壤 (Bulk soil)**：非根际土壤作为对照 → 微生物丰富度最高
- **种子内生 (Seed endophyte)**：种子内部 → 垂直传播微生物，生物量极低

### 样品复杂性如何？

- **根际/土壤** → 高复杂度（数千种微生物），需大量测序数据（>10Gbp/sample）
- **根内/叶际** → 低复杂度但植物DNA干扰严重
- **关键参数**：微生物与植物的DNA比例 → 决定有效数据量

### 你的科学问题是什么？

- **群落组成**：谁在那里？→ 16S扩增子可能已足够（另见 bio-plant-amplicon）
- **MAGs 恢复**：能否重建微生物基因组？→ 需要宏基因组深度测序
- **功能潜力**：微生物能做什么？→ 宏基因组功能注释
- **植物-微生物互作**：哪些微生物基因与植物表型相关？

## 选择你的组装工具

### MEGAHIT — 标准选择

基于 succinct de Bruijn graph，内存效率高。

**适合**：中等到大数据量（>5Gbp/样本），复杂群落
**不适合**：极小的数据集（<1Gbp）→ metaSPAdes 可能更敏感

MEGAHIT 是植物宏基因组最常用的组装器。对根际和土壤样本表现优异。

### metaSPAdes — 小数据更敏感

**适合**：小数据量（<5Gbp），追求高灵敏度
**不适合**：大数据量 → 内存消耗大，速度慢

对低丰度微生物更敏感，但对植物DNA污染也敏感。

### 混合组装 (Hybrid)

如果同时有短读长（Illumina）和长读长（ONT/PacBio）数据，混合组装可以显著提升 MAG 的连续性。

**适合**：需要闭环基因组的高质量研究
**不适合**：仅有短读长数据

## 宿主DNA去除

### 植物宏基因组的核心痛点

根内和根际样本中，植物 DNA 可以占 80-99% 的 reads。不先去宿主：
- 组装被植物基因组片段淹没
- 有用的微生物 contigs 极低丰度
- MAG 恢复率接近零

### 策略

1. **比对去宿主**：bowtie2/bwa 比对到植物参考基因组，去除匹配的 reads
2. **K-mer 去宿主**：如果植物参考基因组不完整，用 Kraken2/Bracken 分类后过滤
3. **双重过滤**：根内样本建议先比对去宿主，再用 Kraken2 分类确认

**常见误区**：只用一个不完整的植物参考基因组 → 大部分植物 reads 没被过滤掉

## 分箱策略

### MetaBAT2 — 默认首选

基于四核苷酸频率和丰度协方差，自动分箱。

**适合**：中等复杂度群落（100-500 个基因组）
**参数建议**：`--minContig 1500` 用于植物相关样本（排除植物 contig 碎片）

### 多工具共识分箱

荟萃分析表明：单个分箱工具恢复的 MAGs 有限。推荐：

```
MetaBAT2 + MaxBin2 + CONCOCT → DASTool → 共识 MAGs
```

DASTool 整合三个工具的结果，选择最优分箱方案。通常比单工具多恢复 20-30% 的高质量 MAGs。

### 植物特异性分箱建议

- 根际样本：多种分箱工具都用，多样性高
- 根内样本：先严格去宿主（比对+分类），再分箱
- 如果 MAG 数量异常少（<5）：检查宿主去除效率

## MAG 质量评估

### CheckM 标准

| 质量等级 | 完整性 | 污染 | 备注 |
|---------|--------|------|------|
| 高质量 | >90% | <5% | 含 rRNA 和 tRNA 基因 |
| 中等质量 | >50% | <10% | 部分基因组，仍可用于分析 |
| 低质量 | <50% | - | 只能用于存在/缺失分析 |

### GTDB-Tk 分类

基于 120 个细菌和 122 个古菌标记基因的系统发育分类。

**注意**：植物微生物组中有大量未培养的微生物 → 可能只能分类到属或科水平

## 功能注释

### 选择注释策略

- **Prokka**：细菌基因组功能注释，需要完整或近完整的 MAGs
- **DRAM**：Distilled and Refined Annotation of Metabolism → 专门为 MAGs 设计，代谢通路分析强大
- **EggNOG-mapper**：蛋白质功能注释，基于直系同源群
- **KEGG-Decoder**：KEGG 通路热图可视化，适合植物相关通路分析

### 植物相关功能

关注以下功能类别：
- 植物激素合成（IAA、细胞分裂素）
- 铁载体合成与转运（植物铁营养）
- 氮固定（nif 基因）
- 磷溶解
- ACC 脱氨酶（降低植物乙烯水平）
- 抗生素和抗真菌化合物合成

## 常见陷阱

### 植物DNA污染

最常见的失败原因。如果 reads 中 >90% 是植物来源，即使 100M reads 也只有 10M 微生物 reads。
- **检查**：用 Kraken2 快速分类 10K reads，评估植物:微生物比例
- **解决**：不满意的话，用 PMA 处理（活菌富集）或增加测序深度

### 批量效应

不同样本之间的序列批次、DNA 提取试剂盒、PCR 循环数都会引入批量效应。
- **解决**：随机化样本顺序，在 metadata 中记录提取批次

### 假阳性 MAGs

组装错误 + 分箱错误 = 嵌合 MAGs（两个不同微生物 genome 碎片被合并）
- **检查**：MAG 中单拷贝标记基因的异质性（CheckM 可以检测）
- **解决**：用 GUNC 检查基因组嵌合

### 覆盖度不足

低丰度微生物 → 测序深度不足以组装 → 无法恢复 MAG。
- **经验规则**：>10x 覆盖度的微生物才能被组装
- **检查**：用 coverM 或 bedtools 计算 contig 覆盖度
