# 宏基因组分析引导

## 这个分析解决什么问题

用户想研究植物相关微生物群落的全部遗传信息（不仅是"谁在那里"还要"能做什么"）。典型表述：
- "根际微生物有什么功能基因"
- "想从宏基因组中重建微生物基因组 (MAGs)"
- "植物-微生物互作机制是什么"

## 你需要问什么

1. **样本类型**: 根际土壤 / 根内圈 / 叶圈？植物DNA污染程度？
2. **分析目标**: 群落组成 / MAGs恢复 / 功能注释？
3. **测序数据**: 测序量多少？≥10Gb/sample 的土壤才够组装
4. **宿主DNA**: 根内圈植物DNA可达 >90%，需先去宿主
5. **长读长**: 有ONT/PacBio吗？混合组装更好

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **宿主 DNA 去除**: 植物根际/叶圈 shotgun 宏基因组中宿主 DNA 占比 50-99%。必须通过比对去除宿主 reads，否则大量测序数据浪费在植物 DNA 上
- **参考基因组不足**: 许多植物相关微生物（特别是真菌和原生生物）缺乏参考基因组。MAGs（Metagenome-Assembled Genomes）的组装和分箱是重要的补充策略
- **微生物群落复杂性**: 根际土壤微生物组包含细菌、古菌、真菌、原生生物、病毒。shotgun 宏基因组能同时捕获所有类群，但需要不同的分析流程
- **植物基因型效应**: 同一作物不同品种的根际微生物组可能显著不同。实验设计时应记录品种信息

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"shotgun宏基因组数据(≥10Gb/sample)、宿主参考基因组、样本metadata完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"测序量不足(<5Gb/sample)；或缺乏阴性对照"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"样本尚未采集"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
