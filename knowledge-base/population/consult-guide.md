# 群体遗传结构分析引导

## 这个分析解决什么问题

用户想了解样本间的遗传关系：有没有亚群、有几个亚群、哪些样本属于哪个亚群。典型表述：
- "我的样本可以分几个亚群"
- "哪些样本遗传上更接近"
- "做 PCA 看看群体结构"
- "这个群体适合做 GWAS 吗"

## 你需要问什么

1. **群体背景**: 野生种质/地方品种/现代育成品种？混合群体？
2. **物种**: 自交还是异交？不同物种"正常"的群体分化水平不同
3. **标记密度**: 多少 SNP？≥1K 独立标记才足够
4. **分析目标**: 只做 PCA？还要 ADMIXTURE？系统发育树？
5. **需要了解**: 下游分析(GWAS/GS)需要群体结构作为协变量

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **繁殖方式**: 自交物种（水稻、大豆、小麦）→ 高纯合度，LD 衰减慢，群体结构通常与地理来源/育种计划相关；异交物种（玉米、向日葵）→ 高杂合度，LD 衰减快
- **驯化瓶颈**: 作物经历了驯化瓶颈，多样性远低于野生近缘种。仅分析栽培品种会严重低估物种的遗传多样性
- **多倍体**: 异源多倍体（小麦 AABBDD、棉花 AADD）的亚基因组可能有不同的进化历史。群体遗传分析应按亚基因组分别进行
- **种质重复**: 育种材料中同一品种可能以不同名称保存在多个种质库中。需要做 identity-by-state (IBS) 分析识别重复种质

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
