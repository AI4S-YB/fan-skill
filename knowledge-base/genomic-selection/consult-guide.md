# 基因组选择分析引导

## 这个分析解决什么问题

用户想用全基因组标记预测育种值，筛选最优亲本。典型表述：
- "怎么用标记预测育种值"
- "哪些材料该选做下一代的亲本"
- "我想用 GS 加速育种周期"

## 你需要问什么

1. **训练群体**: 多少样本有表型+基因型？≥100 才实用
2. **性状遗传力**: 遗传力是多少？GS 准确性上限 ≈ √h²
3. **标记密度**: 什么平台？芯片/GBS/WGS？多少标记？
4. **育种类型**: 自交系选择还是杂交种预测？
5. **环境**: 单环境还是多环境？需要 G×E 建模吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **育种系统**: 自交物种（水稻、大豆、小麦）的 GS 模型比较简单（加性效应为主）；异交物种（玉米、黑麦）需要考虑显性效应
- **群体结构**: 训练群体和育种群体之间的遗传关系决定预测准确性。跨家系/跨群体的预测准确性通常远低于家系内预测
- **多倍体**: 多倍体植物的 GS 需要特殊的基因型编码（考虑等位基因剂量）或用 GBLUP 直接利用亲缘关系矩阵
- **多性状选择**: 育种中通常需要同时改良多个性状（产量、品质、抗性）。多性状 GS 模型可以利用性状间的遗传相关提高低遗传力性状的预测准确性

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
