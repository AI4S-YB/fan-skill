# 植物基因组选择：分析笔记

## 在开始之前

基因组选择（GS）和 GWAS 虽然使用相同类型的输入数据，但目标完全不同：
- **GWAS** 问：哪些位点与性状相关？
- **GS** 问：这个材料的育种值是多少？该选哪些材料做下一代的亲本？

### 你有多少训练数据？

训练群体大小是 GS 准确性的第一决定因素：
- **n < 100**：预测准确性通常很低 (r < 0.3)。GS 可能不实用。
- **n = 100-500**：中等准确性 (r 0.3-0.6)。对遗传力高的性状可行。
- **n > 500**：良好准确性 (r 0.5-0.8)。大多数育种计划的目标区间。
- **n > 2000**：高准确性 (r > 0.7)。可以考虑深度学习。

### 性状的遗传力

GS 准确性的理论上限是这个性状的 √h²（遗传力的平方根）。
- h² < 0.1 → 几乎无法预测
- h² = 0.3 → 最大 r ≈ 0.55
- h² = 0.7 → 最大 r ≈ 0.84

在分析前先估计这个性状的遗传力——如果 h² 很低，GS 的预测准确性必然会低。

### 标记密度

- **< 1K SNP**：几乎只能做 rrBLUP，效果靠连锁不平衡（育种群体中可能还好）
- **1K-10K**：GBLUP。大多数育种芯片在这个范围。
- **10K-50K**：GBLUP 或 BayesB。高密度芯片。
- **> 50K**：BayesB/C 的变量选择可能优于 GBLUP。考虑深度学习。

---

## 训练群体设计：GS 成功的第一驱动力

训练群体 (training population, TP) 的大小和质量比算法选择更重要。一个设计良好的 TP 用 rrBLUP 可以得到比一个设计糟糕的 TP 用深度学习更好的预测准确性。

### 训练群体大小

训练群体大小和准确性之间呈对数关系——n 从 100 翻到 200 的收益远大过从 1000 翻到 2000。

| 训练群体大小 | 预期准确性 (h²=0.5) | 建议 |
|------------|-------------------|------|
| 50 | r ≈ 0.2-0.3 | GS 不太可能比表型选择更好 |
| 100 | r ≈ 0.3-0.4 | GS 有限的可行性 |
| 200 | r ≈ 0.4-0.5 | GS 开始实用 |
| 500 | r ≈ 0.5-0.6 | GS 明显优于表型选择（节省时间） |
| 1000 | r ≈ 0.55-0.65 | 接近理论上限 |
| 2000+ | r ≈ 0.6-0.7 | 增量收益递减 |

**关键约束**：如果训练群体和选择候选群 (selection candidates) 之间的遗传相关度低，即使训练群体再大，准确性也不会高。

### 训练群体与选择候选群的关系

这是 GS 在植物育种中实践时被低估的问题：

- **理想情况**：TP 和 selection candidates 来自同一个育种群体（如上一轮的育种品系 → 预测这一轮的育种品系）。这种情况下遗传相关度高，GBLUP 很有效。
- **危险情况**：TP 来自品种审定试验（elite 品种），而 selection candidates 来自早期代育种品系。这两个群体的遗传方差结构可能完全不同——早期代有更广泛的遗传背景，但 GS 模型是基于"elite"的。预测准确性会显著下降。
- **最坏情况**：TP 是多样性面板（包含野生种质和地方品种），selection candidates 是现代育种材料。全基因组标记可能不携带跨群体的相关性——因果变异和标记之间的 LD 相态在不同群体中完全不同。

**实用建议**：
- 每隔 2-3 个育种周期，用当前代的表型数据更新一次训练群体。
- 如果你的育种目标发生改变（如从高产转向优质），需要重新构建与当前目标相关的训练群体。
- 不要从文献中拿一个"已发布"的模型直接用于你的群体——除非你的群体和已发布群体高度相关。

## 选择你的 GS 模型

### GBLUP — 默认选择

用基因组关系矩阵 (G matrix) 替代传统的系谱关系矩阵。

**适合**：大多数育种场景，特别是中等密度标记
**优点**：计算快、收敛好、不需要调太多超参数
**缺点**：假设所有标记贡献相等的方差（无穷小模型）

### rrBLUP — 最简单的实现

**适合**：小群体 (<200)，标记中等密度
**优点**：极其稳健，不会不收敛
**缺点**：与 GBLUP 数学上等价但计算略慢

### BayesB/BayesC — 变量选择

**适合**：高密度标记 (>50K)，假设只有部分标记有效应
**优点**：可以做标记效应选择，辨别"重要"标记
**缺点**：计算慢、需要调超参数（nIter, burnIn, pi）

植物中：如果标记密度 >50K，BayesB 通常比 GBLUP 准确 2-5%。差异不大但可能在各轮选择中累积。

### 深度学习 — 超大群体的选择

**适合**：>2000 样本 + >100K 标记
**优点**：可以捕获非加性和上位效应
**缺点**：需要大量调参、过拟合风险高、可解释性低

植物育种中：目前深度学习的优势还不明确——在大多数情况下，GBLUP 或 BayesB 已经接近理论准确性上限。

### 模型选择的实际差异：rrBLUP vs Bayes vs GBLUP vs DL

植物 GS 文献中有大量模型比较研究。以下是实际结论：

| 模型 | 计算时间 (n=1000, p=50K) | 典型准确性 | 什么时候选它 |
|------|----------------------|----------|------------|
| GBLUP | < 1 min | Baseline | 默认。几乎所有情况下都值得先试 |
| rrBLUP | 1-5 min | ≈ GBLUP | 等效，在教科书中更直观 |
| BayesA | 1-4 hours | +0-3% vs GBLUP | 性状已知为寡基因 |
| BayesB | 1-6 hours | +0-5% vs GBLUP | 高密度标记 + 已知少数位点有较大效应 |
| BayesCπ | 2-8 hours | +0-3% vs GBLUP | 不想手动调 pi 时的 BayesB 替代 |
| GBLUP + 加性+显性 | < 5 min | ±0-5% | 杂交种预测 |
| RKHS (核方法) | 5-30 min | +0-5% vs GBLUP | 强非加性效应性状 |
| DL (MLP) | hours-day | -5% to +3% | >2000 样本，但通常不值得 |
| DL (CNN) | hours-day | 不稳定 | 更多炒作，实际优势有限 |

**植物 GS 的共识 (2024)**：
- GBLUP 是基线。如果 BayesB/DL 没有比 GBLUP 好 >3% → 不值得增加复杂度。
- 模型间的差异通常小于训练群体设计、CV 策略、环境数目的影响。
- 对自交作物 (inbred)：模型差异尤其小（加性效应主导）。
- 对杂交种 (hybrid)：需要考虑加性和显性效应的模型可能带来 5-15% 的提升。

## 自交系 vs 杂交种 GS：模型差异和准确性预期

植物 GS 有两个主要的应用场景，算法选择完全不同。

### 自交系 GS（水稻、小麦、大豆、大麦）

**目标**：预测一个纯合自交系的育种值。

**流程**：
1. 训练自交系群体 → 表型 + 基因型
2. 为候选自交系预测 GEBV（还没有表型的新品系）
3. 选择 GEBV 最高的自交系作为下一轮的亲本

**模型**：GBLUP (加性 G 矩阵) 几乎总是够用了。加性效应占主导，G×E 是主要的不确定性。

### 杂交种 GS（玉米、水稻杂交稻、向日葵、蔬菜）

**目标**：预测一个未测试杂交组合的表型。

**流程**：
1. 训练部分杂交种（factorial 或 diallel 设计）→ 表型 + 亲本基因型
2. 为所有可能的未测试杂交组合预测杂交种表型
3. 只把预测前 5-10% 的杂交组合拿到田间验证

**为什么杂交种 GS 挑战更大**：
- 你预测的不是"一个个体"而是"两个亲本的组合"。可能的亲本组合数量巨大——如果有 1000 个候选母本和 100 个测试父本，就有 100,000 个可能的杂交组合。你只能测试其中极少一部分。
- 需要建模显性效应：杂交种的表型 = 母本 GCA + 父本 GCA + 母×父 SCA。GBLUP 可以扩展为 GCA 模型（只估 GCA）或 GCA+SCA 模型。
- 杂种优势的遗传基础还没有被完全理解——非加性效应在杂交种中比在自交系中重要得多。

**杂交种 GS 的模型选择**：
- **GCA 模型**（只预测一般配合力）→ 最简单，多数育种场景下接近最优。
- **GCA + SCA 模型** → 当你有足够多的测试杂交组合且 SCA 方差显著时考虑。
- **GBLUP + 加性 + 显性 G 矩阵** → 需要构建显性关系矩阵 (D matrix)。前提是显性方差确实重要。
- **BayesB/DL** → 当特定基因的加性+显性效应都很重要时。

**准确性预期**：
- 自交系 GS (跨多代) 典型准确性：r = 0.4-0.7
- 杂交种 GS (从未测试过的组合) 典型准确性：r = 0.3-0.6
- 杂交种 GS 的准确性比自交系低 ~0.05-0.15，因为只能使用亲本基因型预测，丢失了分离/重组带来的信息

## 交叉验证

### 植物的关键考虑

**不要随机分折。** 如果群体有结构（品种、家系、育种世代），随机 CV 会高估准确性。

- **有群体结构** → 按亚群/家系做 stratified CV
- **多个环境** → 做 leave-one-environment-out CV（预测新环境中的表现）
- **多个世代** → 前代训练、后代验证（forward prediction）

### CV 参数

- **> 300 样本** → 5-fold × 5 repeats
- **100-300 样本** → 10-fold × 10 repeats
- **< 100 样本** → 留一法 (LOO)

### 随机 k-fold 为什么高估：结构化群体中的 CV 策略

这是植物 GS 实践中最常见的错误。随机 k-fold CV 假设每个样本独立同分布 (i.i.d.)——植物育种群体几乎从不满足这个假设。

**问题根源**：
- 在随机 k-fold 中，同一个半同胞家系的个体可能一部分在训练集中、一部分在测试集中。GS 模型在预测"同家系个体"时的准确性远高于预测"无关个体"——因为同家系个体不仅共享组成基因组的大片段（IBD），还共享了多基因背景。
- 一个育种项目真正面临的是：用上一代的品系预测下一代品系。这些下一代品系和训练群体最多是"表亲"关系，不是"兄弟"关系。
- 随机 CV 报告的准确性 0.6 在实际育种中可能只有 0.3-0.4。

**正确的 CV 策略**：

| 育种场景 | CV 策略 | 为什么 |
|---------|--------|--------|
| 世代间 GS (forward prediction) | Leave-one-family-out | 模拟真实的"预测未来世代" |
| 多环境 GS | Leave-one-environment-out | 测试 G×E 下的预测能力 |
| 杂交种预测 | 保持特定的 tester 组合在测试集 | 测试对"新母本 × 新 tester"的预测 |
| 无强结构的小群体 | 随机 k-fold (必须报告"这可能是高估") | 没有更好的选项时勉强使用 |

**CV 参数优化**：
- 如果 n > 300：5-fold × 5 repeats（平衡了计算量和稳定性）
- 如果 n = 100-300：建议 10-fold（每次测试集更小，训练集更大）
- 如果 n < 100：LOO（留一法），但此时置信区间会很宽
- **永远报告重复间的标准差**——它告诉你 CV 估计的稳定性

## 遗传力与 GS 准确性的关系：理论天花板

GS 准确性的理论上限是 √h²。这是不可逾越的——如果 h² = 0.3，永远不可能达到 r > 0.55（=√0.3）。

**这个上限的推导**：
- GS 试图预测育种值 (breeding value, g)
- 但我们只能用表型值 (phenotype, y = g + e) 来验证预测
- y 和 g 之间的最大可能相关系数 = √(V_A / V_P) = √h²
- 所以预测准确性的理论最大值就是 √h²

**实践中能达到的上限**：
- 标记密度极高 + n 极大 → r ≈ 0.9 × √h²（非常接近理论上限）
- 中等标记密度 + 中等 n → r ≈ 0.6-0.8 × √h²
- 低密度 + 小 n → r ≈ 0.3-0.5 × √h²

**如果得到的准确性 > √h² → 你的 CV 有问题（几乎肯定是高估了）**

## G×E 在基因组预测中

### 什么时候多环境模型有帮助

**反应规范 (Reaction Norm) 模型**：
- 将环境作为协变量纳入 GS 模型
- 分子标记 × 环境互作 (M×E) 可以建模
- 如果环境间遗传相关 < 0.6 → 多环境 GS 通常比单环境 GS 更好

**什么让你的 GS 在不同环境中失效**：
- 不同的 QTL 在不同环境中起作用（QTL×E 互作）
- 因果变异和标记之间的 LD 在不同环境中相同，但环境改变了因果等位基因的效应方向和大小
- 预测结果在目标环境 (target environment) 和训练环境差异很大时最差

**多环境 GS 策略**：
1. 为每个目标环境单独训练一个模型（如果每个环境有足够数据）
2. 多环境 GBLUP（使用非结构化方差-协方差矩阵建模环境间遗传相关）
3. 环境特征 (enviromics) + GS：将环境的特征（天气、土壤数据）作为协变量，预测基因型在"任何给定环境"中的表现

## 长期遗传增益：维持多样性同时最大化响应

GS 的一个被低估的优势是：它可以同时考虑短期增益和长期遗传多样性。

### 短期 vs 长期遗传增益的权衡

- **Truncation selection**（选 GEBV 前 10%）：短期增益最大，但多样性损失最快。5 个育种周期后，群体内几乎只剩 1-2 个"超级家系"。
- **Optimal Contribution Selection (OCS)**：在选择时约束亲本间的平均共祖系数 (coancestry)，在短期增益和长期多样性之间取得平衡。Woolliams 和 Meuwissen 的理论工作表明，OCS 在长期（>10 轮）中的累积增益显著高于 truncation。
- **Genomic OCS (GOCS)**：用 G 矩阵代替系谱计算共祖系数——比系谱 OCS 更精确。

**实际育种建议**：
- 大多数育种项目的前 3-5 轮可以用 truncation，但应同时维护一个"遗传储备群体"。
- 对于长期育种计划（>10 轮），应引入 GOCS。
- 考虑使用"亲本选择指数"：Index = GEBV + λ × (1 - coancestry)

## 育种值排名与选择

### 单性状

根据 GEBV 从高到低排序，选择前 10-20%。

### 多性状选择指数

当多个性状需要同时改良时：
- 每个性状的经济权重 × 每个性状的 GEBV → 选择指数
- 或者设约束条件（如：产量提高 + 品质不降低）

## 常见陷阱

### GS 对近交物种的特殊性

自交作物的 GS 中，加性效应占主导（非加性效应在纯合状态下消失）。所以：
- GBLUP/rrBLUP 已经足够好
- 深度学习通常不会带来显著提升
- G×E 效应是主要的不确定性来源

### 不要过高估计准确性

CV 准确性 0.6 ≠ 实际育种中的遗传进展是 0.6。因为：
- CV 中的"环境"是随机的，但育种的目标环境是具体的
- G×E 可能使实际表现低于预测
- 长期选择后标记-性状关联会衰减

### 标记-QTL LD 衰减

GS 假设标记和 QTL 之间的 LD 在训练群体和选择群体之间一致。经过几轮重组和选择后，这个假设会逐渐失效。需要定期重新训练模型——不要指望一个 10 年前的模型在今天还有相同准确性。
