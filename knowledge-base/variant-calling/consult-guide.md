# 变异检测分析引导

## 这个分析解决什么问题

用户想从重测序数据中鉴定 SNP、InDel 等遗传变异。典型表述：
- "帮我做 variant calling"
- "从 FASTQ 到 VCF 的完整流程"
- "怎么过滤假阳性的 SNP"

## 你需要问什么

1. **测序平台和数据**: Illumina 短读长 / PacBio HiFi / ONT？决定比对工具选择
2. **物种和倍性**: 二倍体 / 异源多倍体（小麦、棉花）/ 同源多倍体（马铃薯、苜蓿）？多倍体需要特殊策略
3. **测序深度**: <5x 只能做群体遗传；5-10x 可做基因型鉴定但需填充；>=10x 标准深度
4. **样本数量**: <10 个用 bcftools mpileup；>=10 个二倍体推荐 GATK GVCF joint-calling
5. **育种系统**: 自交物种（水稻、大豆、小麦）预期杂合度 <5%；异交物种（玉米、向日葵）预期 10-30%
6. **分析目标**: 个体基因型鉴定 / 群体 SNP 集 / 突变体定位？

## 与 GWAS/QTL 的区别
- variant calling → 从 reads 到 VCF（上游：鉴定变异）
- GWAS/QTL mapping → 从 VCF + 表型到关联位点（下游：关联表型）
- 用户说"我有 FASTQ，想要变异"→ variant calling

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **参考基因组质量**: 许多植物参考基因组仍然是 scaffold 水平（N50 < 10 Mb），变异检测的灵敏度在 gap 区域会下降
- **多倍体**: 这是植物变异检测中最大的挑战。异源多倍体（小麦、棉花）需分亚基因组比对和 calling；同源多倍体（马铃薯、苜蓿）的个体基因型鉴定极困难
- **自交物种**: 水稻、大豆、小麦等自交物种预期杂合度 < 5%。杂合度 > 10% 通常意味着样本污染或分析问题
- **细胞器基因组**: 叶绿体和线粒体 DNA 占比高（5-20%），覆盖度极高。应单独分析细胞器变异，不要混入核基因组 SNP 集
- **BQSR 和 VQSR**: 非模式植物通常缺乏已知高置信度变异集（用于 BQSR 和 VQSR 训练）。硬过滤是更实用的选择
