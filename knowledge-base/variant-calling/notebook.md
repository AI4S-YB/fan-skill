# 植物变异检测：分析笔记

这份笔记帮助你像一个有经验的植物基因组学家一样思考和做决策。

## 在开始之前：理解你的实验设计

### 测序深度是关键决策因素

植物重测序的深度范围很广，从 5x 到 50x 都有。深度影响你能检测到什么：

- **< 5x**: 极低深度。仅适合群体遗传学（如估计 π、Fst），不适合个体基因型鉴定。可用 ANGSD 这种基于基因型似然的方法。诚实地说明：这个深度不具备个体 SNP 鉴定的能力。
- **5-10x**: 低深度重测序。常见于大群体筛选。基因型鉴定有一定不确定性。推荐 bcftools mpileup + imputation（如 Beagle）。不要使用 GATK 硬过滤的默认阈值——放宽 QD 到 1.5，MQ 到 30。
- **10-20x**: 标准深度。可以可靠地鉴定 SNPs 和大部分 InDels。GATK 或 bcftools 均可。此深度下 BQSR 的收益有限。
- **20-30x**: 较深。可以进行可靠的 InDel 鉴定和稀有变异发现。强烈推荐 BQSR。
- **> 30x**: 深度测序。可能出现了很多重复序列导致的 mapping 问题。如果覆盖度极高（>100x），考虑下采样到 ~30x 以减少计算负担，除非你在找体细胞突变或低频率亚群变异。

### 比对工具选择：BWA-MEM vs minimap2

植物基因组比对中的选择比人类研究更复杂——基因组大小、重复序列含量和多倍体都会影响比对质量。

| 维度 | BWA-MEM | minimap2 |
|------|---------|----------|
| 数据类型 | Illumina 短读长 (50-300bp) | PacBio HiFi / ONT 长读长 |
| 最佳基因组大小 | 不限（已验证在 17Gb 小麦上可行） | 不限 |
| 大基因组速度 | 中等（需要足够内存） | 快（专为大基因组优化） |
| 重复序列处理 | 通过 split-read 和 soft-clip | 通过链算法 (chaining) |
| 多倍体适用性 | 直接支持（比对到合并参考） | 直接支持 |
| 植物社区采用 | 绝对主流（>90% 的植物短读长研究） | 长读长研究首选 |

**BWA-MEM 在植物中的特定考量**：
- 对于 > 1Gb 的基因组：使用 `bwa-mem2`（BWA 的改进版本）比原版 BWA-MEM 快约 1.5-2 倍，且内存使用更低。
- `-M` 参数（标记较短的 split hits 为 supplementary）：在植物中尤其重要，因为重复序列产生的 split-read 比对更频繁。使用 `-M` 确保 Picard/GATK 正确识别这些 supplementary alignments。
- `-R` 参数（read group）：标准的 `@RG\\tID:sample\\tSM:sample\\tPL:ILLUMINA`。植物研究中容易忽略 read group 的设置，但如果后续要做群体 SNP calling 会出问题。

**minimap2 在植物中的特定考量**：
- 对于长读长植物数据（PacBio HiFi 或 ONT），minimap2 是首选。对 > 5Gb 的植物基因组（大麦 5Gb，小麦 17Gb），minimap2 的速度优势非常显著。
- 参数选择：
  - PacBio HiFi: `-x map-hifi`（预设参数）
  - ONT: `-x map-ont`（预设参数）
  - 如果你的参考基因组是分染色体版本且每个染色体 < 500Mb，可以加上 `--split-prefix` 以获得更高的并行效率
- **特别注意**：对于多倍体物种，minimap2 默认输出多个 mapping score 相同的比对（因为同源染色体的序列相似性）。使用 `--secondary=no` 只保留 primary alignment，但这样会丢失多比对的信号（对后续的多倍体变异检测有重要影响）。

**何时选择哪个**：
- Illumina 短读长 → BWA-MEM（或 bwa-mem2）。没有争议。
- PacBio HiFi → minimap2。也没有争议。
- 混合数据（短读长用于 SNP，长读长用于 SV）→ 短读长用 BWA-MEM，长读长用 minimap2。两种比对分别处理，后续整合。
- 高度重复的基因组（如玉米 80% 重复序列）→ 考虑增加 BWA-MEM 的 `-k` 参数（种子长度从默认 19 增加到 25-30）来减少虚假比对。

### GATK vs bcftools：植物变异检测的决策指南

GATK 和 bcftools 不是"高级 vs 低级"的关系，而是"不同设计哲学适用于不同场景"。

| 维度 | GATK (HaplotypeCaller) | bcftools mpileup |
|------|------------------------|-------------------|
| 设计哲学 | 单样本单倍型组装 + 联合调用 | 群体层面的 pileup 直接调用 |
| 多倍体支持 | 通过 `-ploidy N` 参数（有限） | 通过 ploidy 参数（灵活） |
| 大样本（30+） | 推荐（GVCF + joint-calling 效率高） | 可以但不优化 |
| 小样本（< 10） | GVCF 开销不值得 | 直接有效 |
| 计算资源需求 | 高（每个样本一个 GVCF） | 低（直接输出 VCF） |
| 后续 imputation | 优秀（GVCF 格式天然兼容 Beagle） | 可以但需要额外处理 |
| 非模式物种 | 适用但参考注释依赖度高 | 宽容度高 |
| BSA（混池分析） | 不适用 | 适用（`bcftools mpileup` + `--ploidy` + 混池频率） |
| 易用性/学习曲线 | 较陡（理解 GVCF 工作流） | 较浅（一行命令） |

**GATK 工作流的植物考量**：
- BQSR (Base Quality Score Recalibration) 需要一个已知的变异集合。对于非模式物种，这个集合不存在。解决方案：(a) 做一轮初步 SNP calling（bcftools 快速扫描），用高置信度 SNP 作为 BQSR 的 known sites；(b) 对非模式物种，BQSR 的收益递减更快，因为协变量（如 GC bias）在良好测序质量的植物数据中通常不严重。
- HaplotypeCaller 的 `--max-reads-per-alignment-start` 参数在植物中需要特别注意：着丝粒和重复区域的极高覆盖度可能导致 HaplotypeCaller 在该位点"失速"。如果某个位置有 >10000 reads 比对到同一起始位置，设置 `--max-reads-per-alignment-start 50` 下采样。
- GVCF 模式在样本量大时的优势：GVCF 存储每个样本在所有基因组位置的基因型信息（包括参考纯合子位置），而非仅变异位置。在后续 joint-calling 时，GATK 可以区分"这个样本在此位置是参考纯合子"（GVCF 中有记录）和"这个样本在此位置没有数据"（没有记录）。这在填充时是至关重要的信息。

**bcftools mpileup 的植物考量**：
- `--max-depth` 参数：植物线粒体和叶绿体基因组通常有极高的覆盖度（细胞器 DNA 多拷贝，在总 DNA 中占据 5-20%）。如果不设置 max-depth，这些区域的 pileup 会极其缓慢并消耗大量内存。设置 `--max-depth 10000`（或更高）限制 organelle contig 上的深度。
- `--min-MQ` 和 `--min-BQ`：对于非模式物种（参考基因组质量未知），这些阈值可能需要放宽。`--min-MQ 20` 和 `--min-BQ 20` 是一个较平衡的起点。
- `-C 50`（调整 mapping quality 在含 InDel 区域的降权）：推荐在植物中使用，减少 InDel 周围的假 SNP。

### 多倍体的复杂性

植物变异检测最困难的部分不是方法选择，而是**多倍体带来的根本性挑战**：

#### 异源多倍体（如小麦 6x AABBDD，棉花 4x AADD）
- 亚基因组之间有足够的序列分化，read 能较大比例正确地分配到对应亚基因组
- 但亚基因组间的同源交换和部分同源性依然会导致一些 reads 的 mapping quality 低
- **策略**：
  1. 如果亚基因组参考可用（如小麦的 A、B、D 分别有参考），**分别比对**到各亚基因组参考
  2. 比对到全基因组参考也可以，但要预期亚基因组间的 mapping quality 会低于亚基因组内的区域
  3. 比对完成后，按 subgenome 分别鉴定变异

#### 同源多倍体（如马铃薯 4x，苜蓿 4x，甘蔗 8-12x）
- 同源染色体之间的序列高度相似，read 无法区分属于哪条同源染色体
- 无法做单倍型 phasing
- **策略**：
  1. GATK `-ploidy N` 可以处理，但对于 4x 以上的多倍体，freebayes 的多倍体模式通常更准确
  2. 对于高倍性物种（>4x），考虑用专门的多倍体工具如 polyRAD、GWASpoly、updog
  3. 不要期望个体 SNP 水平的高准确度——在群体层面统计等位基因频率更可靠

### 多倍体变异检测策略：深入

**全基因组 vs 分亚基因组比对**：
- 分亚基因组比对的优势：减少了 cross-homeolog mapping，reads 只与一个 subgenome 比对时 mapping quality 更高。变异检测的精度显著提升。
- 分亚基因组比对的劣势：如果某 read 实际上来自 A 基因组但由于序列相似性被比对到 B 基因组参考上，会产生一个假的 B 基因组"变异"（实际上是 A 基因组的真实变异 cross-mapped）。需要比对后过滤步骤来识别这些情况。
- 推荐策略（对于有亚基因组参考的物种）：
  1. 将 reads 比对到合并的全基因组参考
  2. 同时将 reads 比对到各个亚基因组参考
  3. 合并两种比对结果：对于 mapping quality 高的区域（MQ > 40），两个方法的一致性高，结果可信
  4. 对于 mapping quality 低的区域（MQ < 20），检查是否是多比对导致的

**同源多倍体的剂量问题**：
- 在同源四倍体中，基因型不仅仅是 0/1/2（像二倍体），而是 0/1/2/3/4（一个杂合子可能有多种剂量状态：A/B/B/B、A/A/B/B、A/A/A/B 对应的表型剂量不同）
- GATK 的 `-ploidy 4` 假设所有基因型类别等概率出现（priors），这在真实生物学中不是真的。对于一个特定的四倍体物种和特定的群体结构，某些剂量类别的概率可能极低
- freebayes 可以通过设置 `--genotype-qualities` 和调整 `--ploidy` 得到更准确的剂量估计
- 对于更高倍性的物种（>4x，如甘蔗 8-12x），不建议做个体基因型鉴定。改为在群体层面做等位基因频率估计或等位基因剂量估计。此时 ANGSD 是个更好的选择（不输出个体基因型，直接输出群体摘要统计）

### 自交 vs 异交：杂合度期望完全不同

查看 `species_detected` 和 `breeding_system` 字段：

- **自交物种**（水稻、大豆、小麦、大麦）：预期杂合度 < 5%，通常 < 1%。如果某个样本的杂合度 > 5%，可能是：
  - 样本污染或混样
  - 这个"品种"其实是群体或混合材料
  - 参考基因组错误导致的假杂合信号
  
- **异交物种**（玉米、向日葵、黑麦、苜蓿）：预期杂合度 10-30%。如果 < 5%，可能是：
  - 该材料经历了多代自交（如重组自交系）
  - 使用了近交系
  - 参考基因组偏差

### 过度杂合度在自交系中的诊断

在植物变异检测中，"过度杂合度"（excessive heterozygosity）是最常见的质控红旗之一。对自交物种尤其严重：

**杂合度过高的原因排查清单**：
1. **DNA 样本混合**：如果两个或多个个体的 DNA 被混合，杂合度会异常高。检测：杂合 SNP 在基因组中均匀分布是混合的标志。
2. **重复序列区域的假杂合**：参考基因组中的重复序列导致不同拷贝的 reads 堆叠在同一位置，产生假的杂合信号。检测：杂合度升高的区域是否与 RepeatMasker 注释的重复区域重叠。
3. **比对偏差（mapping bias）**：在多倍体中，如果等位基因在亚基因组间存在差异，比对算法可能从中性（随机 alignment score 相同）偏向其中某个等位基因。这导致杂合度的系统性偏差：杂合位点偏向纯合（如果 reads 只能比对到其中一个亚基因组）。
4. **PCR 重复的干扰**：如果在比对后没有用 Picard 去 PCR 重复，同一个起始分子的多份 PCR 拷贝会被计为独立的 reads，在杂合位点可能导致等位基因频率偏离 0.5。
5. **杂合位点的覆盖度异常低**：真正的杂合位点应该有两个等位基因的支持 reads，且覆盖度不应该系统性地低于纯合位点。如果高杂合度的位点集体性低覆盖度，说明可能是测序/比对错误。

**解决过度杂合度的步骤**：
1. 标记重复 reads → Picard MarkDuplicates
2. 重比对 InDel 周围区域 → GATK IndelRealigner（GATK3）或 HaplotypeCaller 的组装步骤（GATK4）
3. 硬过滤：FS > 60 且 QD < 2.0 → 可能的假杂合
4. 如果过度杂合度集中在特定 contig（如 ChrC/Mt），分离出来单独处理
5. 在报告中报告：过滤前后的杂合度变化，哪些因素被识别为杂合度膨胀的原因

## 选择你的变异检测方法

### GATK HaplotypeCaller — 最完整的解决方案

**适合**：
- 二倍体物种
- ≥ 10 个样本（可以用 GVCF + joint-calling）
- 有良好的参考基因组和注释
- 需要最高的 SNP 和 InDel 准确度

**不适合**：
- 高倍性多倍体（8x 以上）
- 样本太少（< 5 个）且没有已知位点做 BQSR
- 计算资源有限

### bcftools mpileup — 最简单直接

**适合**：
- < 10 个样本
- 非模式物种
- 快速探索性分析
- 作为 GATK 的 pre-screen
- 混池测序（BSA）

**不适合**：
- 大群体（≥ 30 个样本）——用 GATK 效果更好
- 需要 gVCF 做后续 imputation

### DeepVariant — 长读长最佳

**适合**：
- PacBio HiFi 数据
- 需要最高准确度的单样本分析
- ONT 数据

**不适合**：
- 多倍体（不支持）
- 没有 GPU（速度太慢）
- 大样本群体（DeepVariant per-sample + GLnexus 比 GATK 复杂）

### Joint-Calling vs Single-Sample Calling

植物研究中的一个关键设计决策：对所有样本联合 calling 还是每个样本单独 calling 再合并。

**Joint-calling（GATK GVCF + GenotypeGVCFs）**：
- 优势：利用所有样本的信息来改善变异位置的鉴定和基因型分配。低频等位基因如果只在一个样本中出现，单独 calling 可能被当作测序错误过滤掉，但 joint-calling 可以通过交叉验证保留。
- 劣势：所有样本需要同时处理，如果后续新增样本，需要重新运行整个 pipeline（除非使用 incremental joint-calling）。
- 植物适用：如果你有一个固定群体的样本（如 QTL 作图群体的 200 个 RIL），joint-calling 是最佳选择。这个群体的遗传背景一致（来自两个亲本），joint-calling 可以大幅提升罕见重组事件的检测。

**Single-sample calling（per-sample 输出 VCF）**：
- 优势：样本独立，后续可以灵活添加新样本。每个样本的 VCF 可以独立过滤和分析。
- 劣势：在低深度时，单个样本的变异 caller 对低覆盖位点的鉴定能力很弱。新突变或罕见变异可能在单独 calling 中被忽略。
- 植物适用：对于不断累积的种质资源收集，新样本持续加入，single-sample calling 是唯一可管理的策略。后期可以用 bcftools merge 合并多样本 VCF。

**建议**：
- 如果你的样本是"一次性"的群体设计（如 GWAS 面板、QTL 群体），用 joint-calling
- 如果你的样本是"持续性"的种质资源库，考虑 single-sample calling 或 incremental joint-calling
- 如果你不确定，joint-calling 的统计功效总是更高

### 硬过滤 vs VQSR：植物的过滤阈值

GATK 提供了两种变异过滤策略：硬过滤和 VQSR。植物场景下，VQSR 的适用性受限。

**硬过滤（Hard Filtering）**：
- 基于预定义的阈值直接在变异质量注释上过滤。
- 优势：透明、可解释、可重现、不需要训练数据。
- 劣势：阈值的选择是阈值选择有一定任意性，可能过滤掉真正的变异或保留假阳性。

**VQSR（Variant Quality Score Recalibration）**：
- 基于已知的"true sites"（通常是高置信变异集合）训练一个高斯混合模型，然后用该模型评估每个变异的"真实性概率"。
- 优势：比硬过滤更准确（当有训练数据时），过滤阈值的生物学/统计基础更牢固。
- 劣势：需要一个可靠的训练集（已知高置信度变异）。对于非模式植物物种，这个训练集通常不存在。

**植物 VQSR 的困境**：
- VQSR 在人类基因组研究中是标准做法，因为在人类中有大量已知高置信度的变异资源（dbSNP, 1000 Genomes, HapMap）。
- 在植物中，除了水稻、玉米、拟南芥等极少数模式物种外，没有足够大的已知变异集用作 VQSR 训练。
- 对于非模式植物物种：硬过滤是唯一实用的选择。使用硬过滤不会显著降低 SNP calling 的质量——关键是选择正确的阈值。

**推荐的植物硬过滤阈值**（基于植物全基因组重测序的经验）：

| 变异类型 | QD | FS | MQ | MQRankSum | ReadPosRankSum | SOR |
|---------|-----|-----|-----|-----------|----------------|-----|
| SNP（二倍体） | < 2.0 | > 60.0 | < 40.0 | < -12.5 | < -8.0 | > 3.0 |
| SNP（多倍体） | < 1.5 | > 80.0 | < 30.0 | < -12.5 | < -8.0 | > 4.0 |
| InDel（二倍体） | < 2.0 | > 200.0 | — | — | < -20.0 | > 10.0 |
| InDel（多倍体） | < 1.5 | > 250.0 | — | — | < -20.0 | > 12.0 |

**植物特定的调整**：
- 自交物种（水稻、大豆）：GATK 的默认 MQRankSum 和 ReadPosRankSum 阈值不适合自交物种——自交物种的等位基因频率分布与 HWE 期望不同。ReadPosRankSum 可能会系统性地偏离 0，因为 reads 在自交物种中不会因等位基因不同而出现位置偏倚。建议对自交物种放宽 ReadPosRankSum 到 < -5.0 和 > 5.0.
- 高重复基因组（玉米、小麦）：着丝粒和重复区域的 MQ 会系统性偏低，但要过滤掉这些区域会导致全基因组的覆盖度不均。区分"MQ 低但区域是着丝粒"（预期，可接受）和"MQ 低但区域是单拷贝"（异常，需过滤）。可以通过与 RepeatMasker 注释交叉引用来区分。

## 硬过滤参数如何选择

VCF 过滤不是一刀切的。根据数据特征调整阈值：

### QD (Quality by Depth)
- 标准：< 2.0 过滤
- 低深度（5-10x）：放宽到 < 1.5
- 高深度（> 50x）：可收紧到 < 5.0（深度太高往往意味着重复区域）

### FS (Fisher Strand Bias)
- 多倍体必然有更高的 FS（同源区域的 reads 会错配产生链偏向）
- 二倍体：FS > 60 过滤
- 多倍体（4x）：放宽到 FS > 80
- 多倍体（6x）：放宽到 FS > 100

### MQ (Mapping Quality)
- 二倍体：< 40 过滤
- 多倍体/高重复基因组：可放宽到 < 30
- 如果看到大量 MQ=0 的位点，检查是否是叶绿体/线粒体基因组的高覆盖引起的

### SOR (Strand Odds Ratio)
- 标准：> 3.0 过滤
- 如果数据有明显的 GC bias，可能需要放宽到 > 4.0
- 植物基因组有时会看到一段连续区域的 SOR 偏高，这通常是参考基因组的组装错误

---

### 细胞器基因组的变异处理

植物的线粒体和叶绿体基因组在总 DNA 中占比高（5-20%），且具有与核基因组不同的遗传特性。它们不能被简单地当作"额外的染色体"处理。

**叶绿体基因组 (cpDNA)**：
- 遗传方式：在大多数被子植物中是母系遗传。这意味着所有后代（无论杂交方向）共享母本的叶绿体基因型。叶绿体 SNP 不应该被用于推断个体间的核基因组遗传关系。
- 覆盖度：叶绿体 DNA 的拷贝数高（每个细胞 100-1000 个叶绿体，每个叶绿体 100 个基因组拷贝），导致测序覆盖度极高（通常是核基因组的 50-200x）。这会产生两个问题：
  1. 计算负担：叶绿体 contig 上的 pileup 极深，消耗大量计算资源
  2. 杂合度伪像：由于拷贝数极高，叶绿体变异很少是杂合的（虽然有叶绿体异质性 heteroplasmy 存在）
- 策略：
  1. 首先识别叶绿体 contig（名称通常包含 "ChrC"、"Pt"、"chloroplast"、"cpDNA"）
  2. 单独提取叶绿体 reads（或用 `--ignore` 在 nuclear calling 时排除叶绿体 contig）
  3. 叶绿体变异用专门的分析流程：降低 min-depth（如 10 而不是 50，因为叶绿体 reads 多），但不进行基因型 calling（直接用等位基因频率）
  4. 在后续分析中（如 GWAS/群体结构），不要将叶绿体 SNP 纳入核基因组 SNP 的合并分析

**线粒体基因组 (mtDNA)**：
- 植物线粒体基因组非常特殊：大小高度可变（从 200 Kb 到 > 10 Mb），结构复杂（多染色体、大量重复序列），突变率极低（比叶绿体和核基因组都低得多）
- 由于序列复杂度高（重复序列多），线粒体 contig 上的 mapping quality 偏低，产生的变异大部分是重复序列导致的假阳性
- 策略：通常不建议对植物线粒体进行常规变异检测，除非你的研究问题明确涉及线粒体功能或细胞质雄性不育 (CMS)

**细胞器变异的后续使用**：
- 细胞器变异（尤其是叶绿体）在系统发育和群体遗传结构分析中非常有用
- 但它们是独立于核基因组的遗传标记，不应与核基因组 SNP 混合使用

## 常见陷阱

### Ti/Tv 比率异常

- **Ti/Tv < 1.5**: 假阳性太多。最可能的原因是：测序错误过多（检查 FastQC）、比对参数不对（ref 和 reads 非同类物种）、或者捕获了重复区域
- **Ti/Tv > 3.0**: 罕见的，可能是假阴性太多（过滤过严），或者只在编码区鉴定变异。检查是否误用了外显子组 Capture 模型
- **植物的预期范围**：全基因组 Ti/Tv 约 1.8-2.2，略低于人类（2.0-2.5）。外显子区域 2.0-2.5

### 叶绿体/线粒体变异干扰

植物总 DNA 提取中，叶绿体 DNA 通常占 5-20%。如果不处理：
- 叶绿体变异会使 Ti/Tv 和杂合度指标误导
- 假阳性可能集中在 organelle contig 上

**建议**: 先鉴定 organelle contig（名称通常包含 "ChrC"、"Mt"、"Pt"、"chloroplast"、"mitochondria"），分别过滤/分析。

### 着丝粒区域的变异密度骤降

植物着丝粒区域通常长达数 Mb，含有大量重复序列。在这些区域：
- 变异密度极低（reads 难以唯一 mapping）
- 出现的任何"变异"大概率是假阳性
- 在做全基因组变异密度图时，着丝粒区域的低密度是正常的，不是数据质量问题

### 大基因组的分染色体分析

对于 > 1Gb 的植物基因组（玉米 2.3Gb，大麦 5Gb，小麦 17Gb）：
- **必须分染色体分析**,不要尝试一次性把整个基因组装入 GATK
- 使用 `--intervals` 或 `-L` 参数按染色体分批
- 每个染色体独立进行 HaplotypeCaller 和 GenotypeGVCFs
- 最后用 `bcftools concat` 合并各染色体的 VCF

### 多倍体中的比对偏差 (Mapping Bias)

这是植物变异检测中一个特别隐蔽的陷阱。在异源多倍体中：

**比对偏差的来源**：
- 如果一个 read 包含一个 SNP，而该 SNP 的等位基因与亚基因组 A 的参考序列匹配但与亚基因组 B 的不匹配，比对算法会偏向于将该 read 比对到亚基因组 A（因为 alignment score 更高）
- 结果：该位点在亚基因组 B 上看起来像是"没有 reads 覆盖"的区域，而不是"有 SNPs 但被 cross-mapped 到 A"的区域
- 这种现象会导致：(a) 亚基因组特异的 SNPs 被系统性地错配到其他亚基因组；(b) 真实杂合位点被伪装为纯合位点；(c) 某些基因组区域的覆盖度在不同亚基因组间不均衡

**检测比对偏差**：
- 模拟 reads：从亚基因组 A 的参考序列生成含有已知 SNPs 的 reads，将其比对到亚基因组 A 和 B 的合并参考上，统计正确比对的比例。
- 如果正确比对率 < 80%，该区域的变异检测结果不可靠。
- 在真实数据中：检查两个同源基因的覆盖度是否相差 > 2 倍。如果是，存在比对偏差。

**缓解比对偏差**（在分析层面）：
1. 使用 WASP (WAvelet-based Snp Polisher) 等去偏倚工具
2. 分亚基因组比对（如上文所述）
3. 对 suspect 区域（覆盖度差异 > 2x 的同源区域），用 ANGSD 基于基因型似然的方法代替硬基因型 calling
4. 在分析报告中明确讨论比对偏差的可能影响及其限制

### 没有一个显著位点——这是失败吗？

不一定，取决于你的目标：
- 如果你在做突变体定位（mapping-by-sequencing），应该有显著位点
- 如果你在做群体重测序，零个位点可能代表：
  - 该性状由多个微效位点控制
  - 样本量不够
  - 标记密度不够
  
诚实报告上述情况，不要为了有结果而降低阈值至无意义的水平。
