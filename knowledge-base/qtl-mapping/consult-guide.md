# QTL 作图分析引导

## 这个分析解决什么问题

用户想用双亲群体定位控制性状的数量性状位点。典型表述：
- "我有 F2/RIL 群体，想做 QTL 定位"
- "怎么找控制粒重的 QTL"
- "这个性状有几个 QTL"

## 你需要问什么

1. **群体类型**: F2 / RIL / DH / BC / MAGIC？决定遗传模型
2. **群体大小**: 多少个体？<100 只能检测大效应 QTL
3. **标记密度和遗传图谱**: 有遗传图谱吗？标记数多少？
4. **表型数据**: 几个环境？QTL×E 效应重要吗？
5. **分析目标**: 找 QTL 位置 / 效应估计 / 候选基因？

## 与 GWAS 的区别
- 双亲群体 → QTL mapping（已知亲本来源的等位基因分离）
- 自然群体/多样性面板 → GWAS
- 不确定时：如果用户说"我杂交了两个品种"→ QTL mapping

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **多年生作物**: 果树和林木的 F1 群体（全同胞家系）是主要的 QTL 作图群体——不同于一年生作物的 F2/RIL 设计
- **多环境 QTL**: 植物数量性状普遍存在 QTL×E 互作。单环境 QTL 结果需在其他环境中验证
- **BSA (Bulked Segregant Analysis)**: 对于质量性状或主效 QTL，BSA（混池分析）是一种低成本快速定位策略，在植物中广泛应用

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"双亲群体基因型数据、多环境表型数据、遗传图谱完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"群体大小不足(<100)；或仅有单环境表型"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"作图群体尚未构建"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
