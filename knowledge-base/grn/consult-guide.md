# 基因调控网络分析引导

## 这个分析解决什么问题

用户想从表达数据推断基因之间的调控关系。典型表述：
- "哪些转录因子调控这个通路"
- "我的数据中有哪些共表达模块"
- "想找关键的 hub 基因"

## 你需要问什么

1. **科学问题**: 找上游 TF / 找共表达模块 / 比较不同条件的网络？
2. **样本量**: n<15 只能做候选分析；n≥30 才够建网
3. **物种**: PlantTFDB 中有覆盖吗？没覆盖需要同源推断 TF
4. **数据**: 只有表达矩阵还是有其他证据（ChIP/DAP-seq）？
5. **多倍体**: TF 在多个亚基因组有拷贝，需分亚基因组分析

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **TF 家族**: 植物 TF 家族（MYB、NAC、WRKY、bHLH、AP2/ERF 等）种类多且功能分化。需要植物特异的 TF 注释数据库（PlantTFDB、PlantRegMap）
- **非模式物种**: 大多数植物没有现成的 TF 注释。需要用拟南芥/水稻同源基因推断，或使用 domain-based 方法（InterProScan 识别 DNA-binding domain）
- **多倍体**: 多倍体中同源基因（homeologs）可能具有不同的调控网络。需要亚基因组特异的 GRN 分析
- **条件特异性**: 很多植物调控关系只在特定环境条件（胁迫、发育阶段）下活跃。单一条件下的 GRN 推断会遗漏大量条件特异性调控

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"表达矩阵(≥30样本)、TF注释信息、参考基因组完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"样本量不足(<15)；或缺乏TF注释"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"表达数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
