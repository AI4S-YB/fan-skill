# 植物基因型填充：分析笔记

这份笔记帮助你像一个有经验的植物遗传学家一样思考基因型填充问题。

## 在开始之前：理解你的目标

`inspect_data.sh` 产出了 data_profile.json。先不着急选工具，先理解你面对的是什么。

### 为什么要做填充？

基因型填充在植物育种中有几个核心目的：

1. **提升标记密度用于 GWAS**：将低密度芯片（10K-50K SNP）填充到测序级别（>1M SNP），显著提升精细定位能力
2. **合并不同平台的基因型数据**：不同批次用了不同的芯片，填充到公共参考面板后可以合并分析
3. **为基因组选择准备数据**：GS 模型通常要求所有个体有相同的标记集，填充是一种解决方案
4. **填补缺失基因型**：QC 后某些标记被过滤，填充可以恢复部分信息

明确你的目的会影响很多后续决策。

### 你的数据长什么样？

看 `data_profile.json` 的关键字段：

- **sample_count**：样本量。影响工具选择和执行时间。
- **snp_count**：当前 SNP 数量。< 1000 几乎无法填充，需要先收集更多标记。
- **species_detected / breeding_system**：物种与繁育系统。影响参考面板选择。
- **ploidy**：倍性。二倍体物种所有工具都可用，多倍体需特别处理。

### 你有参考面板吗？

植物基因型填充最核心的问题之一。

**有参考面板**（如 1000 Genomes、HapMap 等公共资源）：
- 可以选择 IMPUTE2 或 Minimac4，这些工具专门为利用参考面板设计
- 填充精度通常更高，尤其是在低频变异上
- 但需要参考面板与你的群体有相似的 LD 结构

**没有参考面板**：
- Beagle 5 是首选，它使用 internal 参考（样本自身的 LD 模式）进行填充
- 精度取决于样本量和标记密度——样本越多、标记越多、填充越好
- 适合大多数植物育种场景（尤其是非模式物种）

### 参考面板选择：按作物指南

不同作物的参考面板资源差异极大。在启动填充前确认你的物种有哪些可用资源：

**主要作物参考面板资源**：

| 水稻 | 3000 Rice Genomes (华大) | 3,000 | >29M | GigaDB / NCBI SRA |
| 玉米 | HapMap3 | 1,218 | >83M | MaizeGDB / Panzea |
| 玉米 | 507 Ames Panel | 507 | >25M | MaizeGDB |
| 小麦 | 10+ Wheat Genomes (IWGSC) | 16 | >60M | IWGSC / Ensembl Plants |
| 小麦 | WEWseq (野生二粒小麦) | 800+ | >60M | GrainGenes |
| 大豆 | SoySNP50K reference | >20K | 60K (填充后 >5M) | SoyBase |
| 大豆 | 302 Wild + Cultivated | 302 | >10M | NCBI SRA |

| 大麦 | Barley 1K Exome | 1,063 | >1M (exome) | e!DAL / IPK |

**非模式作物（没有参考面板）的现实路径**：
1. 如果没有公共参考面板，你的选择范围缩小到 Beagle 和使用内部参考。
2. 但并非绝望：如果你有全基因组测序（WGS）数据的一个子集（如 10-30 个代表性个体），可以构建一个"内部参考面板"——对这 30 个个体进行深度测序（>10x），鉴定高置信度变异，用作其余芯片样本的填充参考。
3. 这种方法在作物育种项目中非常实用：对核心亲本/骨干材料测深度，对大量后代/品系测芯片或低深度，然后用核心亲本做参考面板填充后代数据。
4. 内部参考面板的精度上限取决于：(a) 参考个体数量（越多越好，建议 ≥ 20）；(b) 参考个体与目标个体的遗传关系（越近越好，遗传距离 Fst < 0.05 理想）；(c) 参考个体的测序深度（建议 ≥ 10x）。

### 你的物种特殊吗？

- **主要作物**（水稻、玉米、小麦、大豆）：可能有公共参考面板。查 `bio-plant-infra/references/plant-databases.md`。
- **非模式物种**（多数园艺作物、林木）：大概率没有参考面板。Beagle 几乎是唯一合理选择。
- **多倍体**：IMPUTE2 和 Minimac4 原生不支持多倍体。Beagle 对多倍体的支持最好。异源多倍体可以按亚基因组拆分后各亚基因组独立填充。

## 选择你的填充工具

每种工具的设计理念不同，适用于不同场景。

### Beagle 5 -- 最通用的选择

不需要参考面板，使用 Li-Stephens 模型的变体。内置定相功能。

**适合**：
- 没有参考面板（大多数植物场景）
- 小到中等数据集（n<500，SNP<50K）
- 多倍体物种（Beagle 支持多等位基因模型）
- 快速原型和初步分析

**不适合**：
- 有高质量参考面板+大样本量（此时 Minimac4 更高效）
- 需要整合外部参考面板信息

**关键参数**：
  - `ne=` 有效群体大小（默认 1,000,000，植物育种群体通常 100-1000）
  - `window=` 滑动窗口大小（cM，默认 40，LD 范围长的自交物种可增大到 60）
  - `nthreads=` 并行线程数
- 对于自交物种，适量调低 `ne=`（200-500），反映其较小的有效群体

### IMPUTE2 -- 参考面板驱动

需要分两步：SHAPEIT 定相 → IMPUTE2 填充。严格依赖参考面板。

**适合**：
- 有高质量单倍型参考面板（如 1000 Genomes 水稻/玉米数据）
- 参考面板与目标群体有相似的遗传背景
- 对填充精度要求极高（如精细定位）

**不适合**：
- 没有参考面板（无法使用）
- 多倍体（不原生支持）
- 极小的样本量（<50，定相不稳定）

**关键参数**：
  - `-Ne` 有效群体大小（通常 100-1000，与物种匹配）
  - `-k_hap` 参考单倍型数量（越多越好，但计算时间线性增加）
  - `-buffer` 区间缓冲（kb，默认 250）
  - `-burnin` 迭代次数
- 执行时间比 Beagle 长 5-10 倍

### Minimac4 -- 大样本最快

基于状态空间缩减的高效填充算法。需要 M3VCF 格式参考面板。

**适合**：
- 大样本量（n>=500）
- 已有 M3VCF 格式参考面板
- 需要快速完成填充
- 与 TOPMed 等公共参考面板兼容

**不适合**：
- 没有 M3VCF 参考面板（需要先用 Minimac3 转换）
- 小样本量（优势不明显）
- 多倍体

**关键参数**：
  - `--cpus` 并行线程（默认 4）
  - `--minRatio` 最小等位频率比（默认 0.01）
  - `--meta` 输出嵌入等信息

### Beagle vs IMPUTE2 vs Minimac4：完整决策指南

三种工具不是"哪个更好"的问题，而是"哪个更适合你的场景"。

| 维度 | Beagle 5 | IMPUTE2 | Minimac4 |
|------|---------|---------|----------|
| 参考面板需求 | 不需要（内部参考） | 必须有 | 必须有（M3VCF 格式） |
| 最佳样本量 | 20-500 | 50-1000 | 500+ |
| 计算速度 | 快（分钟到小时） | 慢（小时到天） | 很快（分钟到大小时） |
| 多倍体支持 | 部分（多等位基因模型） | 不支持 | 不支持 |
| 定相功能 | 内置 | 需要 SHAPEIT 预步骤 | 不需要（M3VCF 已定相） |
| 低频变异 (MAF<1%) | 中等精度 | 高精度（有参考面板时） | 高精度 |
| 植物领域使用 | 最广泛 | 主要作物（水稻/玉米） | 较大规模的作物项目 |
| 典型执行命令复杂度 | 低（一行） | 中（两步） | 中（需要 M3VCF 转换） |

**Beagle 在植物中的主导地位**：
Beagle 在植物遗传学社区占据主导地位（>80% 的植物填充研究使用 Beagle），不是因为它在技术上最优，而是因为：
1. 大多数植物物种没有公共参考面板——IMPUTE2 和 Minimac4 直接不可用
2. 植物育种群体的有效群体小，Beagle 的 Li-Stephens 模型在这种情况下表现良好
3. Beagle 的易用性（单命令行）降低了入门门槛
4. 非模式物种的基因组质量参差不齐，Beagle 对此容忍度更高

但这条经验法则也有例外：
- 水稻：有 3K 基因组参考面板，IMPUTE2 + SHAPEIT 在低频变异上优于 Beagle
- 玉米：HapMap 参考面板大且多样，Minimac4 是合理选择
- 小麦：没有如此大的多样性参考面板，Beagle 仍然是主流

### 预填充 QC 策略

在运行填充之前，数据质量决定了填充效果的上限。GIGO 原则（Garbage In, Garbage Out）在填充中尤其明显——填充算法会"自信地"把错误基因型传播到邻近标记。

**必须做的 QC 步骤**：

1. **标记缺失率**：
   - 去掉缺失率 > 20% 的标记。缺失率高的标记在 LD 估计中不可靠。
   - 如果标记密度本身很低（< 5000 SNP），可以放宽到 30%，但要记录这是一种有风险的妥协。
   - 如果某个标记的缺失集中在特定样本群中（如某个亚群/批次），这可能暗示芯片杂交问题，需要专门调查这些样本。

2. **样本缺失率**：
   - 去掉缺失率 > 20% 的样本。高缺失率的样本可能源于 DNA 质量差。
   - 在去除样本前，检查缺失率分布。如果缺失率与环境/地点相关（如某个地点的样本集体缺失率高），可能有技术原因（DNA 提取方法、运输条件），而不是样本本身的问题。

3. **最小等位基因频率 (MAF)**：
   - 标准：过滤 MAF < 0.01 的标记（二倍体）或 MAF < 0.05（多倍体）。
   - 在填充前过滤 MAF 的争议：过滤太严格会丢失稀有变异，但 MAF < 0.005 的 SNP 在填充中几乎不可能达到可靠精度。
   - 折衷方案：填充前用宽松的 MAF 过滤（>0.005），填充后再用更严格的 R²/INFO 阈值筛选低频变异。

4. **哈代-温伯格平衡 (HWE)**：
   - 在填充前检查 HWE。显著偏离 HWE (P < 10⁻⁶) 的标记可能有基因分型错误。
   - 重要例外：如果你的群体是 F2/RIL/MAGIC 等实验群体，HWE 期望本身就不满足（群体的等位基因频率不等于基因型频率）。在这种情况下，跳过 HWE 过滤，或将 HWE 检验替换为：检查基因型频率是否符合该群体类型的期望。

5. **等位基因编码一致性**：
   - 确认你的数据中 A/B 等位基因的定义与参考面板一致。如果是 A/T 多态性但你的数据编码为 C/G，这会导致 fill 出错误等位基因。
   - 在填充前，用 `bcftools +fixref` 或类似的工具检查并修正等位基因的 strand 和 reference/alternative 定义。

6. **连锁不平衡修剪 (LD pruning)（可选但有争议）**：
   - 不建议在填充前进行严格的 LD 修剪。填充算法依赖于标记间的 LD 模式，修剪 LD 强相关的标记会丢失填充所需的信息。
   - 唯一例外：如果输入标记密度极高（> 500K SNP）且计算资源有限，可以进行温和的 LD 修剪（--indep-pairwise 50 5 0.9），主要目的是减少完全共分离的冗余标记。

### 后填充过滤：R² 和 INFO 阈值

填充后，需要过滤掉低质量的填充基因型。不同的填充工具输出不同的质量指标：

**IMPUTE2**：
- **info score**（通常输出为 .info 文件）：范围 0-1，衡量填充的"确定性"。0 = 完全不确定，1 = 完全确定。
- 推荐阈值：
  - info >= 0.8：高质量，适合精细定位和 GWAS
  - info 0.5-0.8：中等质量，适合基因组选择（GS 可以容忍一些误差）
  - info 0.3-0.5：低质量，仅适合探索性分析，不适合下游决策
  - info < 0.3：丢弃
- **注意**：IMPUTE2 的 info 衡量的是填充不确定性的内部估计，不是实际正确率。如果参考面板与目标群体匹配度差，info 可能虚高。

**Minimac4**：
- **R²**（通常输出为 .info 文件）：衡量填充的估算精度，范围 0-1。比 IMPUTE2 的 info 更保守。
- 推荐阈值：
  - R² >= 0.8：高质量（Minimac4 的 R² >= 0.8 通常比 IMPUTE2 info >= 0.8 更可靠）
  - R² 0.5-0.8：中等质量
  - R² 0.3-0.5：低质量
  - R² < 0.3：丢弃
- **Empirical R² vs estimated R²**：如果你有 masked validation set，计算经验 R²（填充基因型与真实基因型的相关系数）——这是唯一可靠的精度指标。

**Beagle 5**：
- **DR²**（dosage R²，在 .vcf.gz 的 FORMAT 字段）：范围 0-1，衡量等位基因剂量估算的可信度。
- **AF** 和 **AR2**（在 .vcf.gz 的 INFO 字段）：分别表示等位基因频率和等位基因 R²。
- 推荐阈值：
  - DR² >= 0.8：高质量
  - DR² 0.5-0.8：中等质量
  - DR² < 0.5：低质量，丢弃
- Beagle 的 DR² 在样本量小时（n < 50）可能系统性地偏低，因为内部参考面板的单倍型多样性不够。

**通用后填充 QC**（无论用哪种工具）：
1. 填充后重新检查 MAF：去掉 MAF < 0.01 的标记。注意，填充可能创造出 MAF 极低的"稀有变异"——它们在 LD 模式上没有真实的支持，只是算法的噪音。
2. 检查填充后的缺失率：虽然填充的目标是消灭缺失，但质量太低的填充位点应该重新设为缺失（`./.`），而不是保留低质量的填充值。
3. 检查 Ti/Tv 比率：填充后全基因组 Ti/Tv 应该接近 1.8-2.2（植物全基因组预期值）。如果 Ti/Tv < 1.5 或 > 3.0，说明填充引入了大量假阳性或系统偏差。

### 多倍体物种的填充策略

植物基因组学中多倍体的复杂性在填充中尤为突出。这里的策略取决于你的多倍体类型：

**异源多倍体（异源四倍体、异源六倍体）**：
- 关键原则：亚基因组是独立的遗传单元。在异源多倍体中，亚基因组间的重组率极低（严格二倍体化遗传），LD 只在亚基因组内部存在。
- 推荐策略：
  1. 将 SNP 按亚基因组拆分（需要 SNP 的亚基因组注释）
  2. 每个亚基因组独立进行填充（各亚基因组作为独立的"二倍体"数据集）
  3. 填充完成后合并亚基因组 VCF
- 分亚基因组填充比全基因组联合填充的精度显著更高（提升了 5-15% 的 R²），因为避免了亚基因组间假 LD 的干扰。
- 小麦（AABBDD）的具体方案：A 基因组 SNP → Beagle（A 参考面板），B 基因组 SNP → Beagle（B 参考面板），D 基因组 SNP → Beagle（D 参考面板）。
- 棉花（AADD）的具体方案：At 亚基因组和 Dt 亚基因组分别填充。

**同源多倍体（同源四倍体、同源六倍体）**：
- 同源多倍体的挑战更大，因为没有明确的亚基因组边界。同源染色体之间存在重组和 LD。
- Beagle 的多等位基因模型可以处理一些四倍体情况，但不能处理更高倍性的物种。
- 可行策略：
  1. 如果同源多倍体有较高的 preferential pairing（优先配对，即类似于异源多倍体的行为），可以尝试模拟亚基因组拆分
  2. 使用 `polyRAD` 或 `GWASpoly` 首先估算等位基因剂量（0/1/2/3/4），然后在这些剂量值上做填充——但这不是标准的 SNP 填充
  3. 对于没有已知亚基因组结构的同源多倍体（如高倍性观赏植物），坦言现状：填充精度将显著低于二倍体，报告时明确标注这些限制
- 马铃薯（4x）的特殊情况：栽培马铃薯是同源四倍体，但高度杂合。每个基因座最多 4 个等位基因。标准二倍体填充工具在此不适用。建议使用专门的马铃薯分析管线。

### 低密度到高密度填充：利用历史数据集

许多植物育种项目积累了数十年的低密度标记数据（如 384-SNP 芯片、SSR 标记）。这些"历史数据"包含宝贵的表型记录，但标记密度太低，无法做 GWAS 或基因组选择。

**从低密度到高密度填充的可行性**：
- 起点密度：< 500 标记 → 基本不可行。你需要至少 1000-2000 个标记才能让 LD 模式足够丰富以支持填充。
- 起点密度：1K-10K 标记 → 可行，但需要高质量的参考面板（内部 WGS 参考或公共面板）。
- 关键约束：参考面板中必须包含目标标记（即用于填充的"骨架"标记）。如果你用的是 384-SNP 芯片上的标记，而参考面板中没有这些标记，填充无法进行。

**历史 SSR/标记到 SNP 的过渡**：
- SSR 标记和 SNP 标记不在同一个坐标系中。不能直接用 SSR 做 SNP 填充。
- 过渡方案：选择一组有 SSR + SNP 双平台数据的"桥梁样本"（bridge samples），在桥梁样本中建立 SSR 等位基因与邻近 SNP 单倍型之间的关联。然后在没有 SNP 数据的历史样本中，通过 SSR 推断其 SNP 单倍型。
- 这种方法的不确定性很高，仅推荐用于群体结构推断等对精度要求较低的下游应用。

## 常见陷阱

### 填充精度虚高

IMPUTE2 的 info score 和 Minimac4 的 R² 只衡量"与参考面板的一致性"，不直接等于"填充正确率"。
- 参考面板与目标群体 LD 不一致时，info score 可能仍然很高
- 低频变异（MAF<0.01）的填充精度显著低于常见变异
- **最佳做法**：如果你有 5-10% 的已知基因型（如测序数据），mask 后验证
- 如果没有验证集，保守地过滤 R²<0.5 的标记（而不是通常的 0.3）

### 参考面板-目标群体不匹配

这是填充中最容易被忽视但影响最大的问题。

**何时参考面板不匹配**：
1. 参考面板是温带种质资源，你的目标是热带种质资源（或反之）。LD 结构差异极大。
2. 参考面板是栽培种，你的目标是野生近缘种。遗传分化 (Fst > 0.1) 导致 LD 模式不兼容。
3. 参考面板来自一个亚群（如 indica 水稻），你的目标群体包含其他亚群（如 japonica 或 aus）。
4. 参考面板的个体是现代育成品种，你的目标是个体是非改良地方品种。

**检测不匹配的方法**：
- 在填充前对参考面板和目标群体做 PCA。如果 PC1 明显分离了两个群体，这就是不匹配的信号。
- 计算参考群体和目标群体的 Fst。如果 Fst > 0.05，预期填充精度会下降；如果 Fst > 0.15，填充精度可能低于可接受的水平。
- 不匹配的典型症状：填充质量指标（R²/info）在某些染色体区域突然下降（这些区域的 LD 结构在参考和目标群体中差异最大）。

**缓解不匹配的策略**：
1. 如果参考面板的样本量足够大，筛选与目标群体遗传距离最近的参考个体作为子集参考面板。
2. 混合填充策略：在参考面板中加入部分目标群体个体（如果有 WGS 数据的话），使参考面板同时包含参考多样性和目标群体特异的单倍型。
3. 在报告中明确讨论参考面板与目标群体的遗传背景差异及其对填充精度的预期影响。审稿人会问这个问题。

### 多倍体的特殊挑战

- **异源多倍体**（小麦、油菜）：亚基因组间的同源交换（homeologous exchanges）会产生虚假信号。按亚基因组拆分填充更安全。
- **同源多倍体**（马铃薯、苜蓿）：剂量问题。二倍体工具假设 0/1/2 三种基因型，不适用于 0/1/2/3/4 的剂量。Beagle 的多等位基因模型可以部分解决。
- 报告填充结果时，明确标注多倍体带来的限制

### 群体分层导致的错误填充

如果目标群体有明显的亚群结构（如 PC1 解释 >10% 方差），直接全群体填充可能引入错误：
- 亚群 A 特有的单倍型被错误地填充到亚群 B
- 解决方案：按亚群分层填充，或使用包含亚群信息的参考面板
- 填充后检查：每个亚群的填充精度应相似。如果某亚群 R² 显著偏低，重新分层

### 着丝粒/异染色质区域

植物基因组通常有大片段的异染色质区域（尤其着丝粒附近）：
- 这些区域重组率极低，标记密度也低
- 填充精度通常很差（R² < 0.3 常见）
- 如果下游是 GWAS，考虑直接排除着丝粒区域的填充结果
- 在报告中单独标注这些区域

### 没有显著提升怎么办

如果填充后标记数没有大幅增加：
1. 检查参考面板与目标群体的匹配度
2. 确认输入数据质量（缺失率太高导致 Beagle 无法构建单倍型）
3. 可能你的标记集本身就只覆盖了基因组的一部分（如候选基因芯片）
4. 诚实地报告限制，不要假装填充成功了

## 填充 vs. 测序的经济学

在你决定走填充路线之前，考虑一下：
- 填充成本 = 计算资源 + 时间 + 精度损失
- 低深度全基因组测序（skim-seq, ~1X）成本在持续下降
- 对于某些场景（尤其是多倍体、非模式物种），直接测序可能更经济

这不是技术建议，而是策略提醒——和你的合作者讨论预算和精度权衡。
