# 基因型填充分析引导

## 这个分析解决什么问题

用户想将低密度基因型数据填充到高密度。典型表述：
- "我的芯片只有 10K SNP，想填充到测序级别"
- "不同批次用了不同芯片，怎么合并分析"
- "很多标记有缺失，怎么补全"

## 你需要问什么

1. **填充目的**: GWAS 精细定位 / 基因组选择 / 合并数据集？
2. **物种和参考面板**: 有公共参考面板吗（水稻3K/玉米HapMap）？
3. **当前标记密度**: 多少 SNP？<1000 几乎无法填充
4. **多倍体**: 异源多倍体需分亚基因组填充
5. **样本量**: 影响工具选择和执行时间

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **自交物种**: LD 衰减慢（水稻数百 Kb），低密度标记即可获得较好填充精度。Beagle 不依赖外部参考面板也能有效工作
- **异交物种**: LD 衰减快（玉米数 Kb），需要高密度参考面板或大样本量来进行准确填充
- **多倍体**: 多倍体的填充比二倍体复杂得多。IMPUTE2 不支持多倍体。Beagle 支持但精度受限于同源染色体的模糊性
- **种质资源库**: 植物育种中种质资源持续新增。考虑 single-sample calling + 增量填充（而非一次性 joint-calling + 整体填充）

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"低密度基因型数据、参考面板(公共或自有)、参考基因组完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"缺乏参考面板；或标记密度过低(<1000 SNP)"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"基因型数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
