# 植物环境组学：分析笔记

这份笔记帮助你像一个有经验的植物环境组学（Enviromics）科学家一样思考和做决策。

## 在开始之前：理解你的环境和实验设计

### 你有多少环境？

环境组学分析的第一步是盘点你的环境数量和质量：

- **env_count = 1**：单环境试验。跳过 G×E 分析，直接做单环境表型分析。没有环境变异可分解。
- **env_count = 2-4**：基本 G×E 设计。可以用 AMMI/GGE 双标图直观展示基因型×环境模式。FA 模型不适用（环境数太少无法估计协方差结构）。
- **env_count = 5-10**：中等规模 MET。可以尝试 FA(1) 或 FA(2) 模型。TPE 分析开始有意义。
- **env_count > 10**：大规模 MET。FA 模型优势明显。TPE 划分和育种区域规划成为核心任务。

### 你有哪些环境变量？

环境不仅仅是"地点+年份"的标签。真正的环境组学需要量化环境特征：

- **只有地点名和年份**：基本的环境分组。可以用环境均值作为隐式指数，但无法解释 G×E 的原因。
- **播种/收获日期**：可以计算生长季长度、积温（GDD）、累积降水。比仅有标签好很多。
- **每日天气数据（Tmax, Tmin, Precip）**：可以计算数十个环境协变量（W1201-W1213 等）。这是环境指标的"全基因组"级别数据。
- **土壤数据**：质地、pH、有机质、有效磷钾。与气候变量互补，共同解释 G×E。
- **管理措施**：播期、密度、灌溉、施肥。这些也是环境的一部分。

### 环境数据来源：植物研究的主要资源

获取高质量的环境数据是环境组学的第一步。以下是植物研究中主要的公共数据源：

**气候数据来源**：

| 数据源 | 覆盖范围 | 空间分辨率 | 时间范围 | 变量 | 获取方式 |
|--------|---------|-----------|---------|------|---------|
| NASA POWER | 全球 | 0.5° (~55 km) | 1981-至今 | Tmin/Tmax/Precip/Srad/RH/Wind | API / 网页下载 |
| WorldClim v2 | 全球 (陆地) | 30 arc-sec (~1 km) | 1970-2000 (气候均值) | Tmin/Tmax/Precip/Bio1-Bio19 | 网页下载 (GeoTIFF) |
| CHIRPS | 50°S-50°N | 0.05° (~5 km) | 1981-至今 | 降水 | 网页下载 |
| ERA5 (ECMWF) | 全球 | 0.25° (~28 km) | 1950-至今 | 多气候变量 | Copernicus CDS API |
| TerraClimate | 全球 (陆地) | 1/24° (~4 km) | 1958-至今 | 水热平衡变量 | 网页下载 (NetCDF) |

**土壤数据来源**：

| 数据源 | 覆盖范围 | 分辨率 | 变量 | 获取方式 |
|--------|---------|--------|------|---------|
| SoilGrids (ISRIC) | 全球 | 250m | pH/有机碳/砂粒/粉粒/粘粒/N/P/K | 网页 / API |
| HWSD (FAO) | 全球 | 1 km | 土壤类型/质地/pH/有机碳 | 网页下载 |
| gSSURGO (USDA) | 美国 | 10-30m | 详细土壤属性 | Web Soil Survey |
| ISRIC-WISE | 全球 | 土壤剖面点 | 化学/物理属性 | 网页下载 |

**如何使用这些数据**：
1. 根据你的环境位置（经纬度）提取气候和土壤数据
2. 对于多年多点试验：使用覆盖所有试验年份的数据，而不是气候均值（气候均值掩盖了年际变异）
3. NASA POWER 是获取每日天气数据的便捷入口（提供 API，R 包 `nasapower` 可直接调用）
4. WorldClim 用于长期气候特征（如判断哪个地点通常更热/更湿润）
5. SoilGrids 提供高分辨率土壤属性，对于解释营养胁迫相关的 G×E 特别有用
6. **关键注意事项**：这些数据源的分辨率不同——当你用 0.5° 分辨率的 NASA POWER 数据表示一个具体农田的环境时，存在空间尺度不匹配。在报告中讨论这一点。

### 环境指数构建方法

将原始气候变量转化为有生物学意义的环境指数是环境组学的核心任务。以下是主要方法：

**方法 1：直接环境变量法**
- 直接使用特定生长阶段的原始气候变量（如开花期平均温度、灌浆期累积降水）
- 优势：生物学可解释性强——你清楚地知道每个指数代表什么
- 劣势：需要预先知道"关键窗口期"是什么。如果关键窗口判断错误，指数失去意义
- 适用场景：对作物生理学有深入了解，能够预先指定关键生长阶段；或通过滑动窗口法探索后确定
- 典型变量：生长度日 (GDD)、累积降水、VPD（饱和水汽压差）、冷温时数、霜冻天数

**方法 2：PCA 综合指数**
- 对所有环境变量（标准化后）做 PCA，取前 2-3 个 PC 作为综合环境指数
- 优势：数据驱动，不依赖预先指定的关键窗口。通常 PC1 和 PC2 有清晰的生物学解释
- 劣势：PC 的解释可能因不同研究而不同（取决于输入的环境变量集），结果的可迁移性较差
- 适用场景：有充足的环境变量 (>= 5 个) 且不想预先限定关键窗口
- 实践中：PC1 通常代表水分/湿度梯度（降水、相对湿度、土壤水分等变量贡献大），PC2 通常代表温度梯度（日均温、最高温、最低温、辐射等变量贡献大）

**方法 3：生态生理模型指数**
- 使用作物模型（如 APSIM、DSSAT）或简化生理模型计算环境胁迫指数
- 例如：干旱胁迫指数 = 实际蒸散 / 潜在蒸散；高温胁迫 = 超过临界温度的天数 × 超过的度数
- 优势：最高生物学精度——直接量化植物"感受"到的环境胁迫
- 劣势：需要更多参数（土壤属性、管理信息），且模型校准需要时间和经验
- 适用场景：有详细管理记录和土壤数据的试验，且性状-环境关系是量化的、机械性的

**方法 4：ENVIROMICS 协变量 (W-stages)**
- 在育种社区中流行的方法：根据作物生长阶段（播种-出苗、出苗-开花、开花-成熟等），总结每个阶段的气候统计量
- 典型的 W 变量集（参考 Resende et al., 2020 "Enviromics in breeding"）：按阶段累计 > 30 个变量（平均温、降水和、辐射、GDD、极端温度天数、VPD 等，在不同阶段分别计算）
- 优势：系统化、可标准化，一次建立可以为所有后续分析提供环境协变量
- 劣势：计算工作量大（需要为每个环境提取每个阶段的每个变量）
- R 包 `EnvRtype` 提供了 W 变量计算的封装函数

**环境指数选择的一般原则**：
1. 首先用文献确定作物的关键生长阶段和关键胁迫类型
2. 构建初步的候选环境指数集（每种方法选 2-5 个候选指数）
3. 计算各指数与基因型表型的关联系数。选择与表型强相关的指数
4. 检查选择的指数之间的相关性。如果两个指数 r > 0.9，保留其中一个（避免冗余）
5. 最终的环境指数集通常包含 3-8 个指数，覆盖水分、温度和辐射梯度

### 你的基因型数量和结构如何？

- **genotype_count < 30**：统计推断受限。AMMI 模型可能不收敛。建议使用 Finlay-Wilkinson 回归等简单稳定性指标。
- **genotype_count = 30-100**：标准育种试验规模。AMMI/GGE 效果良好。FA(1) 可行。
- **genotype_count > 100**：适合 FA 模型。协方差结构可以较好地估计。反应规范模型可以拟合基因型特异斜率。

## 选择你的环境指数构建方法

### PCA 环境指数 — 当你有充足的气候变量

当你有 ≥3 个气候变量时，PCA 是首选方法：

1. 标准化所有气候变量（均值为 0、标准差为 1）
2. 运行 PCA，提取前 2-3 个主成分
3. PC1 通常代表水分梯度（降水/湿度变量贡献大）
4. PC2 通常代表温度梯度（积温/日均温贡献大）
5. PC1 + PC2 解释的方差 >60% 说明综合指数有效

**植物育种注意事项**：
- 不要无脑把所有气候变量扔进 PCA。按生物学意义分组：温度组、水分组、辐射组，分别做 PCA。
- 不同作物的关键气候窗口不同。小麦的关键期是抽穗-灌浆期，玉米是吐丝期前后。限定时间窗口比全生长季平均更有生物学意义。

### 直接使用原始变量 — 当气候数据有限

气候变量 <3 个时，直接用原始变量作为环境特征：
- 不要强行做 PCA（2 个变量的 PCA 没有降维意义）
- 单独分析每个变量与基因型表现的关联
- 用散点图展示基因型表型 vs 环境变量

## 选择你的 G×E 建模方法

### Factor-Analytic (FA) 模型 — 最全面的方法

FA 模型的核心优势是能够估计环境间遗传方差的异质性和遗传相关的不完全性。

**适合**：
- ≥5 个环境
- ≥50 个基因型
- 你关心环境间遗传相关的结构（哪些环境对基因型排序相似？）
- 育种值预测需要利用环境间遗传相关信息

**不适合**：
- 环境数太少（<5）：FA 模型需要足够的环境来估计因子载荷
- 基因型很少（<30）：遗传协方差矩阵估计不稳定

**FA 阶数选择**：
- FA(1)：最简单，假设一个公共因子。适合环境高度相似的情况。
- FA(2)：最常用。可以刻画较复杂的 G×E 模式。
- FA(k)：k 通常不超过环境数的 1/3。

### G×E 方差分量估计

理解 G×E 的效应大小对于解释其在总表型变异中的重要性至关重要。

**方差分量的估计方法**：

1. **混合线性模型 (Linear Mixed Model, LMM)**：
   - `y = μ + G + E + G×E + ε`，其中 G 和 G×E 为随机效应
   - 估计的方差分量：σ²_G（遗传方差）、σ²_E（环境方差）、σ²_{G×E}（互作方差）、σ²_ε（误差方差）
   - 总方差 σ²_total = σ²_G + σ²_E + σ²_{G×E} + σ²_ε
   - 每个分量占总方差的比例 = 遗传力 / G×E 相对重要性

2. **G×E 占总变异的比重（广义遗传力分解）**：
   - H² = σ²_G / (σ²_G + σ²_{G×E}/n_env + σ²_ε/(n_env × n_rep))
   - σ²_{G×E} 越大 → 广义遗传力越小 → 在多环境试验中预测基因型表现越不确定
   - 如果 σ²_{G×E} > σ²_G：G×E 效应大于基因型主效应，品种推荐必须区分环境类型

3. **G×E 与环境的关联**：
   - 如果有环境协变量（如 PC1、PC2 气候指数），可以将 G×E 分解为 G×Cov1 + G×Cov2 + residual G×E
   - 这回答了："G×E 中有多少是由已知的环境梯度（气候/土壤）解释的？"
   - 如果 G×Cov1 解释了 G×E 方差的 40% 以上，说明 PC1 代表的环境梯度是 G×E 的主要驱动因素

**在育种报告中报告方差分量**：
- 报告 σ²_G、σ²_{G×E}、σ²_ε 的估计值及其标准误
- 用条形图或饼图展示各分量的相对大小
- 比较不同性状的方差分量结构。产量通常有较大的 G×E 分量（σ²_{G×E} > σ²_G），而开花期通常 G×E 较小
- 讨论 G×E 大小对育种策略的影响：如果 σ²_{G×E} 很大，需要做 TPE 分区和分区测试

### AMMI/GGE 双标图 — 最直观的方法

**适合**：
- ≥2 个环境
- 你主要关注 G×E 模式的可视化解释
- 需要识别"哪个基因型在哪个环境最好"
- 育种家需要直观的决策工具

**不适合**：
- 需要正式的方差分量估计和育种值预测（用 FA 或混合模型）
- 有缺失数据（AMMI 要求平衡数据）

**AMMI vs GGE 的选择**：
- AMMI：先移除基因型主效应和環境主效应，再对残差（G×E）做 SVD。适合关注 G×E 模式本身。
- GGE：不移除基因型主效应，直接对 G+G×E 做 SVD。适合关注基因型评价和推荐。
- 实践中，GGE 在育种决策中更常用，因为它直接告诉你在某个环境中哪个基因型最好。

### 反应规范 (Reaction Norm) 模型 — 当你有环境指数

**适合**：
- 已有定量环境指数（PCA 得分或特定气候变量）
- 你想刻画基因型对环境梯度的连续响应
- 关注基因型的可塑性（斜率）和稳定性（截距变异）

**不适合**：
- 没有连贯的环境梯度（环境间没有可量化的连续差异）
- 反应规范假设线性关系，如果响应是高度非线性的需要谨慎

### 反应规范模型的深入

反应规范（Reaction Norm）将表型变异沿环境梯度可视化，每个基因型有自己的"响应线"。

**反应规范模型的 R 实现**（使用 lme4 或 ASReml）：
```r
# 随机斜率-随机截距模型
model <- lmer(y ~ env_index + (1 + env_index | genotype), data = met_data)
# 提取: 每个基因型的截距 (稳定部分) 和斜率 (可塑性/敏感性)
```

**关键输出解读**：
- **截距 (Intercept)**：在环境指数 = 0（通常是平均环境）时的预期表型。截距高的基因型在"平均环境"中表现好。
- **斜率 (Slope)**：表型对环境梯度的敏感性。斜率 = 1（平均敏感性），斜率 > 1（高可塑性——对环境变化非常敏感，环境好时表现极好但环境差时表现差），斜率 < 1（低可塑性——对环境变化不敏感，在多种环境中表现稳定但可能缺乏在最优环境中的高产潜力）。
- **斜率与截距的相关**：如果高截距的基因型也有高斜率（正相关），说明高产品种对环境变化更敏感。这在育种中是常见模式，但复杂化了品种推荐（高产 ≠ 稳定）。

**反应规范的非线性扩展**：
- 对于极端环境（非常干旱或非常高温），基因型的响应可能不是线性的
- 可以用多项式反应规范：`y ~ poly(env_index, 2) + (poly(env_index, 2) | genotype)`
- 或使用分段线性模型：对不同的环境指数区间使用不同的斜率
- 非线性反应规范在气候场景预测中尤其重要——如果未来的环境条件超出历史范围（如极端高温），线性外推可能严重错误

**植物反应规范的特定解读**：
- 对于产量性状：斜率代表"产量可塑性"。在多变气候区域（如半干旱区），低斜率的基因型更有利（稳定产量）。在水肥充足管理的核心产区，高斜率的基因型更有优势（高产潜力）。
- 对于品质性状（蛋白质含量、脂肪酸组成）：斜率代表"品质稳定性"。在食品加工中，品质稳定性往往比品质平均水平更重要——低斜率的基因型被优先考虑。
- 自交系 vs 杂交种：杂交种通常比自交系有更高的斜率（更大的可塑性），这既是优势（高产潜力）也是劣势（稳定性差）。在育种计划中平衡可塑性和稳定性是一个持续的挑战。

## TPE 分析：目标环境群体

TPE（Target Population of Environments）分析帮助育种家回答："我的育种目标环境是什么？"

### 为什么要做 TPE？

1. 育种资源有限，不能在所有环境中测试
2. 不同环境类型可能需要不同的品种
3. 理解 TPE 结构可以提高育种效率

### TPE 的正式定义和划分方法

**TPE 定义**："品种将被种植的所有未来环境的集合"。这不是简单的地点列表，而是一个统计分布——未来环境和其频率的集合。

**TPE 划分的三个层面**：
1. **空间维度**：品种将在哪些地理区域种植？（如："华北平原冬麦区"）
2. **时间维度**：未来年份的气候变异有多大？（历史气候变异 + 气候变化趋势）
3. **管理维度**：农民将采用什么样的管理措施？（灌溉/旱作、播期范围、施肥水平）

**TPE 划分的技术方法**：

**方法 1：基于气候/土壤的聚类分析**
- 输入：每个环境的气候（降水、温度、辐射等）和土壤（质地、pH、有机质等）特征
- 方法：层次聚类（Ward's method）或 K-means
- 输出：每个环境被分配到一个 TPE 组
- 验证：对每个 TPE 组，检查组内基因型排序是否相似（组内 G×E 应小），组间基因型排序是否不同（组间 G×E 应大）
- 优势：简单直接，基于物理环境特征
- 劣势：相似气候/土壤条件不一定意味着相似的基因型×环境互作（G×E 的复杂性超过物理环境变量）

**方法 2：基于遗传相关的聚类分析**
- 输入：FA 模型估计的环境间遗传相关矩阵 (r_g_ij)
- 方法：对遗传相关矩阵做聚类分析。遗传相关 r_g > 0.8 的两个环境可归为同一 TPE（基因型排序在这两个环境中高度相似）
- 输出：TPE 分组 + 各组内的环境间平均遗传相关矩阵
- 优势：直接基于遗传信息，与品种选择高度相关
- 劣势：需要足够的环境数（≥ 6）和基因型数（≥ 50）来稳定估计遗传相关矩阵

**TPE 划分后的育种决策**：
- 如果只有 1 个 TPE：在所有环境中品种排序相似 → 一个品种适合所有环境（育种简单）
- 如果有 2-3 个 TPE：需要 2-3 个品种（每个 TPE 最优品种可能不同）
- 如果有 > 5 个 TPE：可能意味着环境间无结构化差异，或试验环境/基因型不足以定义 TPE。需要更多数据或更审慎的解释

### 聚类方法

- **层次聚类**：基于气候/土壤变量。可以产生树状图，直观展示环境间的相似性。适合环境数较少（5-20）。
- **K-means**：更快的计算。需要预先指定 K。用轮廓系数（silhouette score）辅助选择最佳 K。
- **模型基础的聚类**：基于 FA 模型估计的环境间遗传相关矩阵，而非原始气候变量。理论上更有育种意义，因为直接利用的是遗传信息。

### TPE 分析后的行动

- 每个 TPE 聚类是否需要在育种计划中独立考虑？
- 每个 TPE 内是否需要独立的测试网络？
- 跨 TPE 的品种推荐策略是什么？

## 稳定性分析

稳定性分析不是附属品——它是环境组学分析的核心产出之一。

### 为什么需要多种稳定性指标？

不同的稳定性指标衡量不同的"稳定性"概念：

- **静态稳定性（Type 1）**：基因型在不同环境中表现一致。指标：环境方差（σ²）、CV。适合：你希望品种在所有环境中产量稳定，即使平均产量不是最高。
- **动态稳定性（Type 2）**：基因型对环境变化的响应与平均响应一致。指标：Finlay-Wilkinson 回归系数 b、Eberhart-Russell 参数。适合：你接受品种"高产环境更高产，低产环境也更低产"，但希望这种响应的可预测。
- **优越性指标**：基因型在所有或大多数环境中排名靠前。指标： superiority measure (Pi)、GGE 排名。适合：育种目标是在尽可能多的环境中表现优于对照。

### 常用稳定性指标一览

| 指标 | 类型 | 解释 | R 包 |
|------|------|------|------|
| CV (%) | 静态 | 变异系数，越小越稳定 | base |
| σ²_g | 静态 | 环境间方差 | base |
| b (Finlay-Wilkinson) | 动态 | 回归系数，≈1 为平均响应 | agricolae |
| S²d (Eberhart-Russell) | 动态 | 回归离差，越小越可预测 | agricolae |
| W (Wricke ecovalence) | 动态 | 对 G×E 的贡献 | agricolae |
| Pi (Superiority) | 优越性 | 与最佳基因型的平均距离 | metan |
| ASV (AMMI Stability Value) | 综合 | 基于 AMMI 得分的稳定性 | metan |
| WAASB | 综合 | 加权平均绝对 Scores | metan |
| GGE 排名 | 优越性 | 理想环境中的排名稳定性 | metan |
| MTSI (多性状稳定性) | 综合 | 同时考虑多性状的稳定性 | metan |

### 稳定性指标的交叉验证

不要只用一个指标。报告至少 3-4 个指标，计算它们之间的 Kendall W 一致性系数。如果多个稳定性指标给出相似的基因型排名，你的稳定性评估才是稳健的。

## 气候变化与育种

环境组学不仅分析历史环境——它还必须前瞻未来。气候变化正在改变作物品种的测试和部署环境。

### 气候变化情景在育种中的应用

**为什么需要考虑气候变化？**
- TPE 是基于历史气候数据定义的。但如果未来气候与历史气候不同，基于历史的 TPE 可能已经过时
- 品种育成通常需要 5-10 年，推广后种植 5-15 年。到品种"退役"时，气候可能已经显著变化
- 气候变化不是均匀的：某些区域变暖更快、某些区域降水模式变化更剧烈

**气候情景和工作流程**：

1. **获取未来气候预测**：
   - 使用 CMIP6 气候模型输出（可从 WorldClim 和 ISIMIP 获取）
   - 选择 2-3 个排放情景：SSP1-2.6（低排放，最乐观）、SSP2-4.5（中等排放，最可能）、SSP5-8.5（高排放，最悲观）
   - 时间范围：2050s（品种退役时）和 2080s（长期趋势）

2. **折算气候预测到环境层面**：
   - 提取试验地点在未来气候情景下的温度、降水预测
   - 对比历史气候（如 1990-2020 年）与未来气候（如 2040-2070 年）的差异
   - 量化每个地点的气候变化幅度：ΔTmax、ΔTmin、ΔPrecip、Δ极端事件频率

3. **TPE 迁移分析**：
   - 用历史数据定义 TPE 聚类
   - 将未来气候数据投影到 TPE 空间上
   - 如果一个地点在未来气候下从 TPE-1 转移到 TPE-2，该地点的最优品种可能改变
   - 量化"TPE 稳定性"：有多少地点/面积在不同气候情景下保持在同一 TPE 中

4. **育种目标气候韧性定义**：
   - 基于未来气候预测，确定目标性状：如未来更高温→耐热性更为重要；未来更干旱→耐旱性更为关键
   - 将"气候韧性"量化为：基因型在未来气候情景下的预测表型与历史气候下的预测表型之差
   - 识别"气候智能品种"（climate-smart varieties）：在历史和未来情景下都表现良好的基因型

**在报告中纳入气候变化分析**：
- 明确说明使用了哪个气候模型和情景
- 不要用单一模型/情景——气候模型间的不确定性很大。至少使用 2-3 个模型和 2 个情景（低排放大致乐观 + 高排放最坏情况）
- 用区间/范围而非精确数字报告气候变化的可能影响
- 讨论限制：气候模型在区域尺度上的不确定性、作物模型在极端条件下的鲁棒性、社会经济学变化未纳入技术分析

## 常见陷阱

### 混淆环境效应与 G×E

方差分析表显示显著的 G×E 效应并不总是意味着"真正的"G×E。显著 G×E 可能是因为：
- 尺度效应：基因型效应的方差在不良环境中被压缩
- 异常环境：某一年某一地点发生了极端气候事件
- 实验误差的空间异质性

在解读 G×E 之前，先检查各环境的 CV 和误差方差是否大致相等。

### 环境相关性膨胀

当使用多个气候变量作为环境协变量时，这些变量之间存在强相关（如最高温和最低温通常 r > 0.8）。这导致两个问题：

**问题 1：多重共线性**
- 如果两个环境变量 r > 0.9，它们在回归模型中提供的独立信息很少
- 模型参数估计的方差膨胀（VIF > 10 是危险信号）
- 解决方案：用 PCA 降维，或在全模型之前先检查协变量之间的相关性矩阵。对高度相关的变量对，保留其中一个。

**问题 2：虚假因果推断**
- 如果"高温"和"干旱"在数据中高度相关（因为热浪通常伴随干旱），你无法将 G×E 归因到温度或水分单独
- 方差分解中，高度相关的协变量之间的变异分配是任意的（取决于它们在模型中出现的顺序）
- 解决方案：(a) 使用条件分析控制其中一个变量；(b) 使用方差分解方法（如 `relaimpo` R 包中的平均顺序方法）；(c) 在讨论中诚实承认无法区分温度效应和水分效应

### 环境数量过少

这是许多植物育种研究中被迫接受的现实限制：

**为什么环境少是问题**：
- 2-3 个环境不足以可靠估计 G×E 方差分量。方差分量估计的标准误很大，置信区间可以包括 0 和非常大的值
- 2 个环境间估计的遗传相关 r_g 的置信区间通常非常宽（如 r_g = 0.7, 95% CI: 0.2-1.0）
- 如果这 2-3 个环境碰巧是"好"年份（风调雨顺），G×E 会被低估——你没有看到在逆境条件下可能出现的重大 G×E

**在报告中的处理**：
- 明确说明环境数量少及其对统计推断的限制
- 使用不确定性区间（标准误、置信区间）而非点估计
- 如果只有 2 个环境，不做正式的 TPE 分析。用描述性方法展示两个环境的差异，但不要声称发现了"TPE 结构"
- 如果历史数据在多个年份可用（即使是不同试验），考虑将其合并以增加环境样本量

### 气候数据的时间窗口错误

用整个生长季平均温度做环境指标通常是错误的。关键窗口才重要：
- 小麦：抽穗前 10 天的温度决定穗粒数
- 玉米：吐丝期前后 15 天的水分决定结实率
- 水稻：灌浆期温度影响品质

如果不确定关键窗口，先用滑动窗口法探索，而非假设全生长季。

### 忽略协方差结构

如果你使用混合模型做 BLUP 预测，假设环境间遗传相关等于 1（compound symmetry）可能会掩盖重要的 G×E 模式。FA 模型允许异质遗传方差和异质遗传相关，如果数据支持这个复杂度，应当使用。

### 把 TPE 聚类当做永久不变的

TPE 分析基于历史气候数据。气候变化会改变 TPE 的结构和边界。TPE 需要定期更新。报告中应注明：
- 数据覆盖的年份范围
- 是否检测到时间趋势
- 预测不同气候情景下 TPE 的可能变化

### 小环境样本导致的假阳性 G×E

当环境数量少时（< 5），一个异常环境（如某年发生冰雹导致局部减产）可以驱动"显著"的 G×E 信号。这个 G×E 是真实的（确实有基因型×环境差异），但它的生物学含义被这单一极端事件主导了。在报告结论中：
- 检查每个环境对 G×E 的贡献。如果 1 个环境贡献了 > 60% 的 G×E 方差，该 G×E 模式主要由这一个环境驱动
- 讨论这个"驱动环境"是否代表了未来育种中可能再次遇到的典型条件，还是纯粹偶然的极端事件
- 如果移除该异常环境后 G×E 不再显著，结论变成"本研究中 G×E 主要由环境 X 的极端事件驱动"，而非"这些基因型在广泛环境中表现出普遍的 G×E"
