# 候选基因关联分析：分析笔记

这份笔记帮助你像一个有经验的植物遗传学家一样思考候选基因关联分析中的各种决策。

## 在开始之前：候选基因关联 vs 全基因组 GWAS 的思维差异

这是最核心的区别——如果你带着 GWAS 思维做候选基因分析，你会犯系统性错误。

### 统计框架完全不同

GWAS 的默认假设是"我不知道哪些基因相关，让数据告诉我"。候选基因分析的前提是"我有理由相信这些基因与性状相关，我需要量化证据"。

这导致几个根本差异：

1. **显著性阈值**: GWAS 用 Bonferroni 校正（5e-8 或 0.05/总 SNP 数），因为需要控制全基因组范围的假阳性。候选基因分析检验次数少得多（几十到几百次），FDR 校正更合适，阈值更宽松。

2. **分析粒度**: GWAS 通常以单个 SNP 为单位，因为不知道哪些 SNP 落在基因内。候选基因分析可以——并应该——考虑基因级单倍型：将基因内的多个 SNP 组合成单倍型，分析单倍型与表型的关联。这提高了生物学解释性（你关心的是"这个基因的哪个等位版本与性状相关"，不是"这个 SNP 为什么显著"）。

3. **稀有变异的态度**: GWAS 中稀有变异（MAF < 0.05）通常被过滤掉，因为统计功效不足以在全基因组检验中存活。候选基因分析可以保留更低频率的变异（MAF > 0.01），因为检验次数少，假阳性风险可控，而功能等位变异在育种群体中往往频率很低。

4. **群体结构**: GWAS 中群体结构校正几乎是必须的（PCA + K 矩阵是标准配置）。候选基因分析中，如果候选基因来自同一亚群内的比较，且群体在亚群内遗传均匀，可以不做结构校正。这不是偷懒——是在恰当的情境下避免过度校正吃掉真信号。

### 什么时候候选基因分析比 GWAS 更合适

- 标记密度不足以做全基因组扫描（<500 SNP），但足以覆盖候选区域
- 你已经有很强的生物学先验（功能注释、QTL 精细定位结果、同源推断）
- 你处于验证阶段，不是在探索阶段
- 样本量太小（<100），GWAS 功效为零，但候选基因分析仍然可以检测中到大的效应

### 什么时候不应该做候选基因分析

- 你对性状的遗传基础一无所知，想做探索性发现 → GWAS
- 你想声明"全基因组最显著的位点" → 候选基因分析不能做这个声明
- 你的候选基因数量 >1000 → 检验次数太多，已经接近 GWAS 的规模，应该按 GWAS 的框架分析

## 单倍型分型策略

候选基因分析的一个核心方法论优势是：你可以构建基因级的单倍型。但如何分型取决于你的物种。

### 自交物种：直接观察法（Direct Observation）

水稻、大豆、小麦、大麦、番茄等自交物种的个体高度纯合。一个 accession 内的大多数位点是纯合的。

**方法**：
1. 提取候选基因区域内的所有 SNP
2. 对每个 accession，按纯合 SNP 的等位基因组合确定单倍型（因为位点是纯合的，phase 是已知的）
3. 去除杂合位点（通常是测序错误或少量残余杂合）
4. 统计每种单倍型的频率

**优势**：
- 不需要复杂的统计 phasing 算法
- 单倍型推断的准确性接近 100%
- 单倍型与表型的关联非常直观：单倍型 A 的材料 vs 单倍型 B 的材料，性状均值差多少？

**注意**：
- 如果 accession 是非常近期的杂交后代（如 F3 或 F4 代），仍可能有较多的杂合位点。检查 `observed_heterozygosity` 字段。如果 Ho > 0.05，需要考虑 phasing。
- 自交物种的 LD 衰减慢，一个基因内的所有 SNP 通常属于同一个 LD block。这意味着基因级单倍型是合理的分析单位。

### 异交物种：统计 phasing

玉米、黑麦、向日葵、多数果树等异交物种的个体杂合度高，不能直接观察单倍型。

**方法**：
1. 使用统计 phasing 工具（BEAGLE、SHAPEIT）
2. 对 phasing 后的单倍型进行关联分析
3. 需要报告 phasing 的不确定性（如 BEAGLE 的 posterior probability）

**关键警告**：
- 统计 phasing 的准确性依赖于参考面板的大小和 LD 模式。如果群体很小（<50）且没有参考面板，phasing 的质量会很差。
- 如果 phasing 质量不好，退回到单个 SNP 分析可能是更诚实的选择。
- 异交物种在进行基因级单倍型分析时，需要区分单倍型是来自母本还是父本（这又回到了 phasing 问题）。

## 稀有变异处理

候选基因分析中 MAF 阈值的选择比 GWAS 更灵活，但这不意味着可以毫无限制地保留所有变异。

### MAF 阈值选择逻辑

| 检验次数 (候选 SNPs 或单倍型) | 推荐 MAF 阈值 | 理由 |
|---|---|---|
| < 100 | 0.01 (或更低, 如 0.005) | 检验次数极少，假阳性风险低。可以保留功能性稀有等位变异 |
| 100-500 | 0.02-0.03 | 适度放宽，平衡检测能力和假阳性控制 |
| 500-1000 | 0.05 | 接近 GWAS 规模，用标准阈值 |
| > 1000 | 0.05 | 检验次数多，严格些更安全 |

### 稀有变异的特殊价值

在候选基因分析中，稀有变异（MAF 0.01-0.05）可能有特殊的生物学价值：

- **功能缺失等位变异**：很多已知功能基因的功能丧失突变在育种群体中频率很低（因为选择压力），但如果你关注的是性状退化的方向（如感病性），这些稀有变异非常关键。
- **抗性基因的稀有抗病等位基因**：抗病基因通常有多个等位变异，其中一些抗性等位变异在群体中频率很低但效应很大。
- **地方品种特异的适应等位变异**：来自特定地理区域的地方品种可能携带稀有的环境适应等位变异。

### 必须谨慎的情况

- 稀有变异（MAF < 0.01）的统计检验极其不稳定——如果你只有 3 个个体携带该等位变异，这 3 个个体的表型可能受其他因素影响。最诚实的做法是报告"观测到 X 个稀有等位变异，但因频率过低无法进行可靠的统计检验"。
- 不要仅凭稀有变异的效应量大就声称它是因果位点——大效应可能是由于小样本的极端值驱动。

## FDR vs Bonferroni：检验次数决定一切

这是候选基因分析中最容易犯错的决策点。许多研究者习惯性地使用 Bonferroni 0.05/n，但这对候选基因分析来说太保守了。

### 核心逻辑

| 检验次数 (n) | Bonferroni 阈值 (0.05/n) | FDR 阈值 | 推荐 | 理由 |
|---|---|---|---|---|
| 10 | 0.005 | BH-FDR 0.05 | FDR | Bonferroni 过于严格，可能漏掉真信号 |
| 100 | 0.0005 | BH-FDR 0.05 | FDR | FDR 允许 5% 的假阳性，候选分析中可以接受 |
| 500 | 0.0001 | BH-FDR 0.10 | FDR (0.10) | FDR 0.10 更宽松，适合候选筛选阶段 |
| 1000 | 5e-5 | BH-FDR 0.05 | Bonferroni | 检验次数多，接近 GWAS 规模，Bonferroni 更安全 |
| 5000 | 1e-5 | - | Bonferroni | 这已经是探索性规模了，应按 GWAS 标准 |

### FDR 的使用原则

1. **Benjamini-Hochberg (BH) 方法**是标准选择。假设检验之间是独立的或正相关的。
2. **FDR 0.05** 意味着预期有 5% 的显著结果是假阳性。在候选验证场景中，这通常是可以接受的。
3. **FDR 0.10** 意味着预期有 10% 的假阳性。适合需要"宁可多一些候选，不能漏掉真的"的筛选阶段。
4. **不要只用原始 p 值**。即使检验次数少，完全不校正是不可接受的。至少报告校正后的值，让用户自己判断。

### 植物领域常见做法

在水稻、玉米、小麦的候选基因关联分析论文中：
- 大多数论文使用 FDR < 0.10 或 FDR < 0.05
- 少数使用 Bonferroni（通常是因为同行评审要求）
- 有效的做法：报告原始 p 值 + FDR 校正 p 值 + Bonferroni 校正 p 值，讨论时以 FDR 为准

## 群体结构：什么时候需要校正

### 不需要校正的情况

候选基因分析中，以下情况可以不使用群体结构校正：

1. **候选基因来自同一遗传背景的群体**：例如，一个单一育种计划的 200 个品系，共享大部分遗传背景。在这些品系中观察到的表型差异更可能是由于少数功能等位变异的分离而非群体分层。
2. **候选基因数量少（<50）且亚群内比较**：如果所有比较都在同一亚群内进行（如仅 japonica 水稻），群体结构的影响很小。
3. **已经通过实验设计控制了结构**：如嵌套关联作图（NAM）群体或 MAGIC 群体，遗传结构本身就是设计的一部分，不需要额外的统计校正。

### 需要校正的情况

1. **混合亚群**：如果候选基因分析中包括来自不同亚群的样本（如 indica + japonica 水稻共分析），必须校正——否则你检测到的是亚群间的差异而非基因型-表型关联。
2. **多样化的自然种质集合**：如果样本来自全球各地的种质资源库，遗传结构复杂，需要 PCA 校正。
3. **GWAS 的后续验证**：如果你从 GWAS 中发现了候选基因，现在在同一群体中做验证，需要保持与 GWAS 相同的协变量结构以确保结果可比。

### 校正方法

- **PCA 协变量**（最常用）：将前 2-3 个 PC 加入 GLM。简单有效。
- **K 矩阵**（亲缘关系矩阵）：如果群体内存在复杂的系谱关系（如自交物种的育种群体），K 矩阵比 PCA 更精确。
- **不需要校正时加了校正**：如果你在不该加校正时加了校正，会损失统计功效——p 值会变得更大但不会产生假阳性。所以"不确定时加校正"是安全的选择，但会降低灵敏度。

## 多维排序：不只看 p 值

p 值最小的基因不一定是生物学上最重要的。一个完整的候选基因排序应该考虑：

### 排序维度

1. **统计显著性（权重: 25%）**
   - FDR 校正后的 p 值
   - 经过多重检验校正仍然显著是基本门槛

2. **效应量（权重: 25%）**
   - 对于连续表型：单倍型间性状均值差异（如单倍型 A 比 B 的千粒重高 3.2g）
   - 效应量大比效应量小更重要，即使 p 值不完全一致

3. **稳健性（权重: 15%）**
   - 如果有多环境数据：该单倍型-表型关联在不同环境中是否一致？
   - 如果进行了不同检验方法的交叉验证：是否在不同方法下都显著？

4. **功能注释相关性（权重: 20%）**
   - GO term 或 KEGG pathway 与性状生物学是否吻合？
   - 是否有已知的突变体表型支持？
   - 跨物种同源基因的功能信息（如拟南芥同源基因的突变体是否影响相似性状？）

5. **文献支持（权重: 15%）**
   - 该基因（或同源基因）是否在文献中被报道与相同或相似性状相关？
   - 即使是同一基因家族的其他成员有功能报道，也是弱支持

### 排序的实践

- 给出一个排序表，明确标注各维度的得分和总分
- 不要自动把排名第一的基因当作"因果基因"。在报告中写："基因 X 的综合得分最高，是最有希望的候选基因，但仍需功能验证。"
- 如果排序靠前的多个基因位于同一染色体区域，它们可能是 LD 拖带关系——需要说明这一点。

## 植物特化考量

### 自交物种的单倍型优势

自交物种（水稻、大豆、小麦、大麦）在候选基因关联分析中有一个独特的优势：个体的高度纯合性让单倍型分析非常简单和可靠。

- **读取即分型**：对于纯合自交系，基因内的所有纯合 SNP 组合直接就是单倍型。你不需要 BEAGLE/SHAPEIT 等 phasing 软件。
- **稳定的遗传背景**：每个 accession 代表一种固定的等位基因组合，可以在多年、多点被重复利用。这让你可以累积多年的表型数据，提高分析的可靠性。
- **功能性单倍型的发现**：因为自交物种的基因组是"冻结"的，自然选择和人工选择留下的单倍型"印记"更容易被识别。

### 候选基因选择来源（植物特有）

在植物研究中，候选基因的来源丰富多样：

1. **QTL 区间（最可靠）**：从精细定位的 QTL 区间中提取候选基因。优势是有独立的遗传证据支持。劣势是 QTL 区间通常包含数十到数百个基因，需要进一步筛选。

2. **BSA-seq 定位区间**：BSA（Bulked Segregant Analysis）结合高通量测序可以快速定位到 Mb 级别的区间。区间内的基因可作为候选。

3. **同源基因推断**：利用拟南芥和水稻的丰富功能基因组学数据，推断目标物种中的同源基因。需要注意：
   - 同源关系需要可靠（BLASTP e-value < 1e-10, identity > 40%）
   - 基因功能在进化中可能已分化——同源不保证同功能
   - 优先使用共线性支持的直系同源（ortholog）

4. **差异表达基因**：RNA-seq 差异表达分析发现的 DEGs 是好的候选来源。但需要注意：
   - 差异表达不等于因果关系——表达变化可能是下游效应
   - 结合 eQTL 分析可以提高置信度

5. **文献报道的功能基因家族**：如抗病基因的 NBS-LRR 家族、转录因子家族（NAC, WRKY, MYB 等）。家族成员在染色体上的位置通常成簇分布——候选区域中的家族成员优先考虑。

6. **GWAS 的候选区间**：从 GWAS 结果中根据 LD 衰减范围确定候选区间，区间内的基因可作为验证目标。

### 跨物种信息的使用原则

拟南芥是最重要的植物模式生物——但跨物种推断需要谨慎：

- **可靠的推断链**：
  1. 在目标物种中确定候选基因
  2. BLAST 找到拟南芥最佳同源基因（e-value < 1e-10）
  3. 查看拟南芥同源基因的突变体表型（TAIR 数据库）
  4. 如果表型一致 → 强有力的支持证据

- **不可靠的推断**：
  - 仅凭 GO 注释推断功能（IEA 证据代码）
  - 跨越太远的进化距离（如用拟南芥基因功能推断松树基因功能）
  - 忽略基因家族的功能冗余（如水稻中有 >1000 个激酶基因，单个成员的功能丧失可能没有表型）

## 常见陷阱

### 把候选基因分析结果当作全基因组发现

候选基因分析的结论必须限定在"候选集内"。你不能说"我们发现基因 X 是全基因组最重要的关联位点"——你没有测试全基因组。正确的表述是"在 X 个候选基因中，基因 Y 显示与性状最强的关联"。

### 忽略候选基因间的 LD 关系

如果候选基因在基因组上位置相近（如位于同一 QTL 区间），它们可能处于同一个 LD block。多个基因显著可能不是独立信号——只是一个因果变异在 LD 中拖带了周围基因。

**解决方法**：
- 检查候选基因的基因组位置
- 对于位于同一染色体区域的基因，做条件分析（conditional analysis）
- 报告时明确指出哪些显著基因可能是 LD 拖带

### 过度解读单倍型的生物学功能

"单倍型 A 的材料耐盐性显著高于单倍型 B"——这个结论是成立的。但"单倍型 A 导致耐盐性提高"——这需要因果关系证据。下一步可以是：
- 构建近等基因系（NIL）验证该基因的效应
- 转基因互补实验
- 基因编辑（CRISPR）敲除或等位基因替换

### 忽略表型数据的质量

候选基因分析的统计功效受表型数据质量影响很大。特别是：
- 单一环境无重复的表型 → 遗传力低，效应估计不可靠
- 极端偏态分布的表型 → 参数检验的假设不满足
- 表型测量误差大 → 掩盖真实关联
- 如果有多年多点的表型数据，优先使用 BLUP 值
