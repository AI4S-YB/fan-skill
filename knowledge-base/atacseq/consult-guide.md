# ATAC-seq 分析引导

## 这个分析解决什么问题

用户想鉴定开放染色质区域，了解基因组哪些区域处于活跃调控状态。典型表述：
- "我想看处理前后染色质可及性有什么变化"
- "哪些转录因子可能调控这些基因"
- "增强子在哪些区域"

## 你需要问什么

1. **物种和参考基因组**: 有高质量参考吗？非模式物种？
2. **组织类型**: 叶片、根、花？不同组织核分离方案不同
3. **实验设计**: 处理 vs 对照？每组几个重复？(建议 n≥3)
4. **分析目标**: 只找开放区域？差异可及性？TF footprinting？
5. **数据量**: 多少 reads？植物cpDNA reads比例高是正常的

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **细胞壁去除**: 植物 ATAC-seq 的核心挑战。核分离需要去除细胞壁（酶解或机械破碎），但过度处理会破坏核膜完整性，使线粒体和叶绿体 DNA 释放
- **叶绿体 DNA**: 绿色组织的 cpDNA 可占总 reads 的 50-80%。必须去除 cpDNA reads 后才能评估真实的核基因组染色质可及性
- **组织异质性**: 植物组织（叶片、根尖）比动物细胞系异质性高得多，样本间变异大，需要更多生物学重复
- **多倍体**: 多倍体物种的 ATAC-seq peak call 在重复序列区域容易产生假阳性

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"ATAC-seq数据(处理组和对照组各3重复，50M reads/sample)、参考基因组完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有处理组数据，缺乏对照组"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"实验尚未进行"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
