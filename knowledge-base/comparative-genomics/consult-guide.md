# 比较基因组学分析引导

## 这个分析解决什么问题

用户想比较不同物种的基因组。典型表述：
- "我想比较几个物种的基因组"
- "想做基因组进化分析"
- "想分析基因组的共线性"
- "想研究物种分化事件"

## 你需要问什么

1. **基因组数据**: 有多少个基因组需要比较？组装质量如何？
2. **基因注释**: 有基因注释吗？注释质量如何？
3. **分析目标**: 共线性分析? 基因家族进化? 基因组结构变异?
4. **系统发育背景**: 有物种的系统发育树吗？分化时间估计？
5. **外群选择**: 有合适的外群物种吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **多倍化历史**: 植物基因组常有多次WGD事件，比较分析需要考虑
- **基因组大小差异**: 植物基因组大小差异可达100倍，影响比对策略
- **重复序列**: 植物基因组高重复率影响共线性检测
- **染色体进化**: 植物染色体数目变化大，需要注意染色体融合/分裂事件

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"4个物种基因组组装和注释完整、外群基因组可用"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"有基因组组装但缺乏基因注释"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"目标物种基因组尚未测序"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
