# 基因家族分析引导

## 这个分析解决什么问题

用户想鉴定和分析一个基因家族。典型表述：
- "我想分析SPL基因家族"
- "我有目标基因的序列，想找同源基因"
- "想做基因家族的进化分析"
- "想分析基因家族的motif和结构"

## 你需要问什么

1. **目标基因**: 有目标基因的蛋白/DNA序列吗？已知基因ID或名称？
2. **参考物种**: 分析范围是什么物种？有参考基因组吗？
3. **比较物种**: 需要做进化分析吗？有多少比较物种的基因组可用？
4. **表达数据**: 有RNA-seq数据用于表达模式验证吗？
5. **分析深度**: 只做鉴定还是需要完整分析（共线性、Ka/Ks、顺式元件）？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **基因家族扩张**: 植物基因组中基因家族常通过全基因组复制(WGD)扩张，一个基因家族可能有数十甚至上百个成员
- **多倍体复杂性**: 多倍体物种的同源基因鉴定更复杂，需区分同源基因和旁系同源基因
- **组织特异性表达**: 植物基因家族成员常有组织特异性表达模式，表达分析很重要
- **启动子分析**: 顺式元件分析对理解基因家族功能调控很关键，PlantCARE是常用工具

## 对话注意事项

- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确
- 使用生物学语言解释推荐理由

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"参考基因组及注释、目标基因序列、比较物种基因组可用"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"缺乏表达数据用于验证；或比较物种基因组不完整"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"目标基因序列尚未获取"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
