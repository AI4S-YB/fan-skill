# 三维基因组分析引导

## 这个分析解决什么问题

用户想分析基因组的三维空间结构和染色质交互。典型表述：
- "我有Hi-C数据，想做三维基因组分析"
- "想分析染色质Loop"
- "想识别TAD区域"
- "想比较不同条件下的三维结构差异"

## 你需要问什么

1. **Hi-C数据**: 有原始FASTQ还是处理后的矩阵？测序深度？
2. **参考基因组**: 有基因组组装吗？需要染色体水平吗？
3. **内切酶信息**: 使用的是什么限制性内切酶？(MboI/HindIII/...)
4. **分析深度**: 只需要TAD/Look还是需要完整的3D结构分析？
5. **比较分析**: 有多个样本/条件需要比较吗？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **基因组大小**: 植物基因组大小差异大（100Mb-30Gb），影响Hi-C数据量需求
- **染色体数量**: 植物染色体数范围广（n=2到n=100+），contact map解读需注意
- **多倍体**: 多倍体物种的同源染色体区分是Hi-C分析的挑战
- **重复序列**: 植物基因组高重复率影响比对率和分辨率

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"Hi-C数据(足够深度)、染色体级别参考基因组完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"测序深度不足；或参考基因组仅为scaffold级别"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"Hi-C实验尚未进行"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
