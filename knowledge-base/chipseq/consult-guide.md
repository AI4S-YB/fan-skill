# ChIP-seq 分析引导

## 这个分析解决什么问题

用户想鉴定特定转录因子或组蛋白修饰在基因组上的结合位点。典型表述：
- "这个转录因子结合哪些基因的启动子"
- "处理前后 H3K27ac 修饰有什么变化"
- "我想验证这个 TF 的靶基因"

## 你需要问什么

1. **实验对象**: 转录因子 ChIP 还是组蛋白修饰 ChIP？
   - TF ChIP → narrow peak；组蛋白 → broad peak
2. **抗体信息**: 用什么抗体？有 Input 对照吗？
3. **物种和组织**: 植物交联条件与动物不同，需要真空浸润
4. **数据质量**: 唯一比对率？FRiP？Fingerprint plot？
5. **分析目标**: 只找 peaks？差异结合？motif 分析？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **抗体可用性**: 商业 ChIP-grade 抗体多为人/小鼠设计，对植物蛋白的交叉反应需要验证。植物组蛋白抗体（H3K4me3、H3K27me3 等）通常跨物种保守性较好
- **交联效率**: 植物细胞壁阻碍甲醛渗透，交联效率通常低于动物细胞。可能需要真空渗透或延长交联时间
- **Input 对照**: 植物 ChIP-seq 的 Input 对照尤其重要，因为染色质可及性和拷贝数变异在植物基因组中普遍
- **重复序列**: 植物基因组高重复序列含量（30-85%）导致许多 reads 多重比对，需要特殊处理（如仅保留 uniquely mapped reads）

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"ChIP-seq数据(IP和Input对照各≥2重复)、参考基因组完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"缺乏Input对照；或仅有单重复"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"ChIP实验尚未进行"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
