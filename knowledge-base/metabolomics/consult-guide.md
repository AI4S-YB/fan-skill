# 代谢组学分析引导

## 这个分析解决什么问题

用户想分析植物的代谢物组成及其在实验条件下的变化。典型表述：
- "处理组和对照组代谢物有什么不同"
- "哪些代谢物与抗病相关"
- "我想鉴定次生代谢物"

## 你需要问什么

1. **分析平台**: LC-MS / GC-MS / NMR？
2. **物种和组织**: 不同组织代谢谱差异极大
3. **实验设计**: 处理/对照？每组几个重复？
4. **分析目标**: 非靶向发现 / 靶向定量 / 差异代谢物？
5. **数据库**: 模式/非模式物种？非模式→代谢物鉴定困难

## 标准分析流程 (7模块完整性要求)

生成方案时必须覆盖以下7类模块：

### 模块1: 数据预处理与质控
- 峰检测 (XCMS centWave / MZmine)
- 峰对齐与合并
- 缺失值填充 (knn / min / half-min)
- 质量控制样本 (QC) 评估

### 模块2: 多元统计分析
- PCA无监督降维 (评估整体分布和批次效应)
- OPLS-DA两组判别 (VIP筛选)
- PLS-DA多组判别
- 置换检验验证模型显著性

### 模块3: 差异代谢物分析
- 单变量统计 (limma-trend / t-test)
- 多重检验校正 (BH / Bonferroni)
- VIP与p值联合筛选
- Fold change计算

### 模块4: 代谢物鉴定
- MS/MS谱库匹配 (SIRIUS+CSI:FingerID)
- 分子式推断
- MSI置信度等级标注 (Level 1-4)
- CANOPUS化合物类别预测

### 模块5: 通路与功能分析
- PlantCyc植物特异通路
- KEGG代谢通路映射
- 化学类别富集分析
- 通路可视化

### 模块6: 可视化
- 火山图 (log2FC vs -log10p)
- 热图聚类 (Z-score标准化)
- PCA/PLS-DA得分图
- 通路图标注

### 模块7: 机器学习与扩展分析 (可选)
- 随机森林特征选择
- ROC曲线生物标志物评估
- t-SNE/UMAP降维可视化

## 植物特定考量

- **次生代谢物**: 植物产生大量物种特异性的次生代谢物（黄酮、萜类、生物碱、芥子油苷等）。通用数据库的覆盖度有限，需要植物专用数据库（如 PlantCyc、KNApSAcK）
- **组织特异性**: 不同植物组织的代谢组差异巨大（叶 vs 根 vs 花 vs 果实），且受发育阶段、昼夜节律、环境条件的强烈影响
- **样品前处理**: 植物细胞壁和大量次生代谢物使代谢物提取比动物组织更复杂。液氮速冻 + 冷冻研磨是标准做法
- **质谱平台**: LC-MS 偏向次生代谢物（多酚、生物碱、脂质）、GC-MS 偏向初级代谢物（糖、氨基酸、有机酸）。非靶向分析推荐两种平台互补使用

## 对话注意事项

- 使用生物学语言而非工具名称解释推荐理由（如"这个分析找控制性状的基因组区域"而非"用 GAPIT 跑混合线性模型"）
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息——信息不足时推荐不准确

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"LC-MS原始数据、QC样本、样本metadata完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"缺乏QC样本，需补充"]
- **数据状态**: EMPTY — [说明数据不可用原因]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
