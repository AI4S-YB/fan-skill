# 细胞器基因组分析引导

## 这个分析解决什么问题

用户想分析叶绿体或线粒体基因组。典型表述：
- "我想组装叶绿体基因组"
- "我有线粒体基因组数据"
- "想分析细胞器基因组的结构"
- "想比较不同物种的细胞器基因组"

## 你需要问什么

1. **细胞器类型**: 叶绿体还是线粒体？还是两者都需要？
2. **数据类型**: 有专用的细胞器测序数据还是核基因组测序数据？
3. **分析目标**: 组装？注释？比较分析？
4. **物种信息**: 植物类型？是否有近缘参考基因组？
5. **分析深度**: 基本注释还是完整的比较基因组学分析？

## 相关信息收集后

收集够信息后 → 进入方法选择 (`rules.yaml`)

## 植物特定考量

- **叶绿体基因组**: 植物叶绿体基因组通常120-160kb，高度保守，包含四段结构（LSC、SSC、两个IR）
- **线粒体基因组**: 植物线粒体基因组大小变异极大（200kb-2.7Mb），结构复杂
- **母系遗传**: 大多数被子植物细胞器基因组为母系遗传
- **异质体**: 细胞器基因组可能存在异质体（heteroplasmy）

## 对话注意事项

- 使用生物学语言解释推荐理由
- 一次只问一个问题，根据用户的回答自然推进对话
- 如果用户已提供的信息覆盖了上述某个问题，跳过不要重问
- 确保在推荐分析前收集到足够信息

## 数据状态标签规范 (C4强制)

在生成方案的数据画像部分，必须显式标注数据状态：
- **数据状态**: FULL — [说明哪些数据完整可用，如"细胞器测序数据、近缘物种参考基因组、组装注释完整"]
- **数据状态**: PARTIAL — [说明哪些数据缺失，如"仅有总DNA数据，无细胞器富集；或缺乏近缘参考"]
- **数据状态**: EMPTY — [说明数据不可用原因，如"测序数据尚未生成"]

违例判定: 仅列出文件名/大小但无显式FULL/PARTIAL/EMPTY状态标签 → C4=0分
