# 植物 RNA-seq 差异表达分析：分析笔记

## 在开始之前

植物 RNA-seq 的核心挑战不是分析本身，而是实验设计。先问自己：

### 你有什么样的数据？

查看 data_profile.json 和 count matrix：
- **基因数**：植物基因组跨度大，草莓 ~25K, 水稻 ~35K, 小麦 ~100K（含同源基因）
- **样本数**：≥3 重复 → 正规分析；2 重复 → 可用但功效低；1 个 or 无重复 → 不能统计推断
- **同源基因**：小麦/棉花/油菜中同一基因在多个亚基因组各有拷贝，需要决定是合并还是分开分析

### 这个实验设计是什么？

- **简单两组比较**（处理 vs 对照）：DESeq2 标准流程
- **多组比较**（多个处理梯度/时间点）：DESeq2 + 所有 pairwise 对比
- **连续协变量**（如温度梯度、发育阶段）：limma-voom 更灵活
- **多因子设计**（基因型 × 处理 × 时间）：DESeq2 的 ~ genotype + treatment + genotype:treatment

### 你期望看到什么？

- **处理起始效应**：对初级胁迫响应 → 预期 5-15% 的 DEGs
- **全局转录重编程**（如某个处理 vs 对照的 >30% DEG）→ 处理强度很高
- **无 DEGs** → 实验可能确实无响应（正常结果），或功效太低（重复不够）

## 选择你的 DE 工具

### DESeq2 — 最稳健，默认选择

使用负二项分布 + 经验贝叶斯收缩。

**适合**：所有有重复的实验设计
**不适合**：仅在没有重复时才需要替代方案

植物 RNA-seq 中 DESeq2 几乎总是第一选择。它处理离散度估计好，对 outlier 稳健，shrinkage 降低了低表达基因的假阳性。

### edgeR — 低重复/无重复时考虑

**适合**：2 个重复 → edgeR 的 quasi-likelihood 帧框略有优势
**不适合**：≥3 重复时没有相对 DESeq2 的显著优势

植物实验中如果因为资源限制只有 2 重复，edgeR 值得考虑。但报告时诚实说明功效限制。

### limma-voom — 大样本/复杂设计

**适合**：30+ 样本，或包含连续协变量的模型
**不适合**：简单两组比较（DESeq2 更好）

对植物超大规模转录组数据（>50 样本来自多个组织/处理/发育阶段），limma-voom 效率远高于 DESeq2。

## 功能富集分析

### 植物的最优选择

1. **Plant Reactome**（水稻/玉米/拟南芥）→ 质量最高的通路注释
2. **AgriGO v2** → 覆盖更多农业物种
3. **MapMan/Mercator** → 非模式植物，在线注释后导出
4. **KEGG Plant** → 备选

### 常见误区

- 不要只看 GO 的 P 值——GO terms 之间有很强的层级依赖（parent-child 关系）
- DEGs 太少（<30）→ 任何富集分析都没有足够的统计功效
- 背景基因集应该是"所有表达的基因"，不是"基因组所有基因"

## WGCNA 共表达分析

### 什么时候值得做？

- 样本 ≥ 15 → WGCNA 开始有意义
- 有连续的性状数据 → 可以做模块-性状关联
- 有多个组织/发育阶段/处理 → 共表达模式往往比 DEG 更有生物学意义

### 植物 WGCNA 的特殊考虑

- 植物发育梯度（叶片年龄、果实成熟）→ 共表达模式往往非常清晰
- 不同组织（根/茎/叶/花）→ 组织特异性模块占主导
- 如果 90% 的基因都聚到一个"超级模块"→ minModuleSize 可能太小或 power 选择有问题

## 常见陷阱

### 同源基因的困扰

小麦的 TraesCS1A01G0001、TraesCS1B01G0001、TraesCS1D01G0001 → 三个亚基因组的同源拷贝。分开分析还是合并？

- **分开**：可以看到亚基因组特异性表达
- **合并（sum counts）**：只有一个"基因"进入 DEG 列表
- 如果不确定，默认分开

### 植物特有的"零膨胀"

某些组织的 RNA-seq 中 >50% 的基因 count 为 0 → 完全正常（基因不在此组织中表达）。DESeq2 的独立过滤会自动处理。

### 表型数据缺失

如果你的 DEG 列表很丰富但无法解释 → 检查是否有表型/metadata 可以帮助解释（组织类型、发育阶段、胁迫处理、基因型差异）。
